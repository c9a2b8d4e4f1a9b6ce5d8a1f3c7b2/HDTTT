Audit Report

## Title
Authenticated Users Can Bypass IP-Based Rate Limiting via `x-forwarded-for` Header Spoofing to Flood the Transaction Creation Endpoint

## Summary
`IpThrottlerGuard`, the sole global rate-limiting guard, derives its tracking key from the raw, user-controlled `x-forwarded-for` request header. An authenticated attacker can rotate this header on every request to obtain a fresh rate-limit bucket, bypassing the throttle entirely. No per-user limit exists on `POST /transactions`, so the attacker can create an unbounded number of database records, exhausting PostgreSQL storage and connection capacity.

## Finding Description

**Root cause — `IpThrottlerGuard` trusts a user-controlled header**

`getTracker` reads the client identifier directly from `req.headers['x-forwarded-for']` with no proxy-trust validation: [1](#0-0) 

This guard is wired as the sole global throttle via `APP_GUARD` in `api.module.ts`: [2](#0-1) 

`UserThrottlerModule` is not imported in `api.module.ts`, and `UserThrottlerGuard` (which tracks by `req.user.id`) is never applied to `TransactionsController`: [3](#0-2) 

**No per-user throttle on the transactions controller**

The controller's guard chain contains only authentication/verification guards — no rate-limiting guard: [4](#0-3) 

`UserThrottlerGuard` exists and correctly tracks by user ID, but is unused here: [5](#0-4) 

**Each `POST /transactions` call is expensive**

`createTransaction` delegates to `createTransactions`, which performs: a mirror-network client connection, cryptographic signature verification, a PostgreSQL `find` (duplicate check) + `save` (insert), a scheduler reminder registration, and a NATS status-update emission: [6](#0-5) [7](#0-6) 

There is no cap on how many active (non-terminal) transactions a single user may hold.

## Impact Explanation
An authenticated attacker with a single registered key can exhaust PostgreSQL disk space and connection pool capacity, and flood the NATS message bus, causing a full service outage for all other users. Recovery requires manual database cleanup and service restart.

## Likelihood Explanation
The precondition is a valid, verified user account with at least one registered key — the normal onboarding state for any legitimate user. The bypass technique (rotating the `x-forwarded-for` header) requires no special tooling; a simple `curl` loop or any HTTP client that sets custom headers is sufficient.

## Recommendation

1. **Do not trust `x-forwarded-for` unconditionally.** Use a trusted-proxy list (e.g., NestJS/Express `trust proxy` setting) so that `req.ip` is already the correct real IP, and remove the raw header fallback from `IpThrottlerGuard.getTracker`.

2. **Apply `UserThrottlerGuard` to `TransactionsController`.** The guard already exists and tracks by `req.user.id`, making it immune to header spoofing. Add it to the `@UseGuards` decorator on `createTransaction` or at the controller level.

3. **Enforce a per-user active-transaction cap** inside `createTransactions` by counting non-terminal transactions owned by the user before inserting a new one.

## Proof of Concept

```bash
JWT="<valid_jwt>"
i=0
while true; do
  i=$((i+1))
  curl -s -o /dev/null \
    -H "Authorization: Bearer $JWT" \
    -H "Content-Type: application/json" \
    -H "x-forwarded-for: 10.0.$((i/256)).$((i%256))" \
    -d '{"name":"flood","transactionBytes":"...","creatorKeyId":1,"signature":"...","mirrorNetwork":"testnet"}' \
    http://target/transactions &
done
```

Each iteration presents a new IP to `IpThrottlerGuard`, receiving a fresh empty bucket. The loop bypasses both the per-minute and per-second throttle buckets, inserting rows into PostgreSQL without bound until storage or connection limits are exhausted.

  async createTransactions(dtos: CreateTransactionDto[], user: User): Promise<Transaction[]> {
    if (dtos.length === 0) return [];

    await attachKeys(user, this.entityManager);

    const client = await getClientFromNetwork(dtos[0].mirrorNetwork);

    try {
      // Validate all DTOs upfront
      const validatedData = await Promise.all(
        dtos.map(dto => this.validateAndPrepareTransaction(dto, user, client)),
      );

      // Batch check for existing transactions
      const transactionIds = validatedData.map(v => v.transactionId);
      const existing = await this.repo.find({
        where: {
          transactionId: In(transactionIds),
          status: Not(
            In([
              TransactionStatus.CANCELED,
              TransactionStatus.REJECTED,
              TransactionStatus.ARCHIVED,
            ]),
          ),
        },
        select: ['transactionId'],
      });

      if (existing.length > 0) {
        throw new BadRequestException(
          `Transactions already exist: ${existing.map(t => t.transactionId).join(', ')}`,
        );
      }

      // Wrap database operations in transaction
      const savedTransactions = await this.entityManager.transaction(async (entityManager) => {
        const transactions = validatedData.map(data =>
          this.repo.create({
            name: data.name,
            type: data.type,
            description: data.description,
            transactionId: data.transactionId,
            transactionHash: data.transactionHash,
            transactionBytes: data.transactionBytes,
            unsignedTransactionBytes: data.unsignedTransactionBytes,
            status: TransactionStatus.WAITING_FOR_SIGNATURES,
            creatorKey: { id: data.creatorKeyId },
            signature: data.signature,
            mirrorNetwork: data.mirrorNetwork,
            validStart: data.validStart,
            isManual: data.isManual,
            cutoffAt: data.cutoffAt,
            publicKeys: data.publicKeys,
          }),
        );

        try {
          return await entityManager.save(Transaction, transactions);
        } catch (error) {
          throw new BadRequestException(ErrorCodes.FST);
        }
      });

      // Batch schedule reminders
      const reminderPromises = savedTransactions
        .map((tx, index) => {
          const dto = dtos[index];
          if (!dto.reminderMillisecondsBefore) return null;

          const remindAt = new Date(tx.validStart.getTime() - dto.reminderMillisecondsBefore);
          return this.schedulerService.addReminder(
            getTransactionSignReminderKey(tx.id),
            remindAt,
          );
        })
        .filter(Boolean);

      await Promise.all(reminderPromises);

      return savedTransactions;
    } catch (err) {
      // Preserve explicit BadRequestException, but annotate unexpected errors
      if (err instanceof BadRequestException) throw err;

      const PREFIX = 'An unexpected error occurred while creating transactions';
      const message = err instanceof Error && err.message ? `${PREFIX}: ${err.message}` : PREFIX;
      throw new BadRequestException(message);
    } finally {
      client.close();
    }
  }
