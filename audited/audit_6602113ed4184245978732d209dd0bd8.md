### Title
Unvalidated `mirrorNetwork` Field in Transaction Creation Enables Server-Side Request Forgery (SSRF)

### Summary
The `CreateTransactionDto.mirrorNetwork` field accepts any arbitrary string with no URL format or allowlist validation. This value is persisted to the database and later used by the backend to construct HTTP URLs for mirror node REST requests. An authenticated organization user can supply a crafted hostname, causing the backend server to issue HTTP GET requests to attacker-controlled endpoints.
