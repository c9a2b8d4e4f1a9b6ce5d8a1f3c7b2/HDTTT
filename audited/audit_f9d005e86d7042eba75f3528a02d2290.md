### Title
Hard-Coded Cryptographic Secrets in Committed `.env.test` File

### Summary
The repository contains a committed `.env.test` file with trivially weak, hard-coded values for `JWT_SECRET` and `OTP_SECRET`. These secrets govern authentication token signing and one-time password generation for the back-end API. Any attacker with read access to the repository can use these values to forge valid JWT tokens or predict OTPs, bypassing authentication entirely.

### Finding Description
The file `back-end/apps/api/.env.test` is committed directly to source control and contains production-class secret keys set to trivially guessable values:

```
JWT_SECRET=13123
OTP_SECRET=123
``` [1](#0-0) 

Unlike the `example.env` files (which are clearly labeled as templates), `.env.test` is a real environment file loaded during test runs. There is no `.gitignore` in the repository to prevent `.env.*` files from being tracked.

The `JWT_SECRET` is used by the NestJS API to sign and verify all JWT authentication tokens. The `OTP_SECRET` is used to generate one-time passwords for email-based verification flows. Both are consumed directly from environment variables at runtime.

Additionally, multiple `example.env` files across the project commit default credentials in plain text:

- `POSTGRES_PASSWORD=postgres` and `PGADMIN_DEFAULT_PASSWORD=pgadmin4` in `back-end/apps/api/example.env`
- `JWT_SECRET=some-very-secret-string` and `OTP_SECRET=some-very-secret-string-otp` in the same file [2](#0-1) [3](#0-2) 

The k6 performance test configuration also hard-codes default user passwords directly in source:

```ts
const DEFAULT_PASSWORD = 'Password123';
``` [4](#0-3) [5](#0-4) 

### Impact Explanation
An attacker who reads the repository (public or leaked) can:
1. Use `JWT_SECRET=13123` to sign arbitrary JWT tokens, impersonating any user including administrators, without knowing any password.
2. Use `OTP_SECRET=123` to generate valid OTPs, bypassing email-based verification.
3. Use the default `POSTGRES_PASSWORD=postgres` to connect directly to the database if the operator deployed without changing defaults.

**Impact: 4** — Full authentication bypass is possible; all organization data and transaction signing workflows are at risk.

### Likelihood Explanation
The `.env.test` file is committed to the repository and is directly readable. If the same secrets are reused in staging or production (a common mistake when copying config files), exploitation requires zero brute-force effort — the secret is already known. The default `postgres`/`pgadmin4` credentials are a well-known pattern that attackers routinely try.

**Likelihood: 3** — Reuse of test secrets in non-test environments is a common operational mistake; the values are trivially weak even if discovered independently.

### Recommendation
1. Remove `back-end/apps/api/.env.test` from version control and add `.env*` (excluding `*.example.env`) to `.gitignore`.
2. Rotate `JWT_SECRET` and `OTP_SECRET` in all environments immediately.
3. Replace hard-coded defaults in `example.env` files with placeholder strings (e.g., `CHANGE_ME_STRONG_RANDOM_SECRET`) that cannot be used as-is.
4. Replace hard-coded `DEFAULT_PASSWORD = 'Password123'` in `automation/k6/src/config/credentials.ts` and `automation/k6/src/config/constants.ts` with values sourced exclusively from environment variables, with no fallback default.
5. Use a secrets manager (e.g., HashiCorp Vault, AWS Secrets Manager) for all production credentials.

### Proof of Concept

**JWT Forgery using the leaked secret:**

```bash
# Install a JWT library
npm install -g jsonwebtoken

# Forge a token for any user ID using the committed secret
node -e "
const jwt = require('jsonwebtoken');
const token = jwt.sign({ sub: 1, email: 'admin@example.com', admin: true }, '13123', { expiresIn: '365d' });
console.log(token);
"

# Use the forged token against the API
curl -H "Authorization: Bearer <forged_token>" https://<target>/api/users
```

The secret `13123` is taken directly from the committed file: [6](#0-5) 

No credentials, network access, or prior authentication are required — only read access to the repository.
