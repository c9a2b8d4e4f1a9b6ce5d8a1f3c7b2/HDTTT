### Title
Critically Insufficient PBKDF2 Iteration Count (2560) Enables Offline Brute-Force of Encrypted Hedera Private Keys

### Summary
The `deriveKey` function in `front-end/src/main/utils/crypto.ts` uses only **2560 PBKDF2-SHA512 iterations** to derive the AES-256-GCM key that encrypts Hedera private keys stored in the local SQLite database. This constant is undocumented, has no rationale, and is ~82× below OWASP's current minimum recommendation of 210,000 iterations for PBKDF2-SHA512. An attacker who obtains the SQLite database file (via malware, stolen backup, or physical access) can brute-force the user's password at ~82× the speed of a properly configured KDF, recovering the raw Hedera private key and draining the associated account.

### Finding Description

**Root cause — hardcoded, undocumented, and critically weak KDF parameter:**

`front-end/src/main/utils/crypto.ts`, lines 5–9:
```typescript
export function deriveKey(password: string, salt: Buffer) {
  const iterations = 2560;   // ← no rationale, no documentation
  const keyLength = 32;
  return crypto.pbkdf2Sync(password, salt, iterations, keyLength, 'sha512');
}
```
No comment explains why `2560` was chosen, what security level it is intended to provide, or what assumptions are made about attacker capability or password entropy.

**Production code path — private key encryption:**

`front-end/src/main/services/localUser/keyPairs.ts` imports `encrypt`/`decrypt` directly from `@main/utils/crypto`:

```
keyPairs.ts line 5:  import { encrypt, decrypt } from '@main/utils/crypto';
keyPairs.ts line 76: keyPair.private_key = encrypt(keyPair.private_key, password);
keyPairs.ts line 148: return decrypt(keyPair?.private_key || '', password);
```

Every Hedera private key stored in the local SQLite database (the non-keychain path) is encrypted with AES-256-GCM using a key derived by `deriveKey` with 2560 iterations. The same weak `deriveKey` is also used in `organizationCredentials.ts` to encrypt organization credentials.

**Exploit flow:**

1. Attacker obtains the SQLite database file (e.g., via malware, stolen laptop backup, or physical access — all realistic for a desktop application).
2. Extracts the `private_key` blob from the `KeyPair` table. The blob format is `[64-byte salt | 16-byte IV | 16-byte GCM tag | ciphertext]` — fully self-contained.
3. Runs offline PBKDF2-SHA512 with 2560 iterations per password guess. On a modern GPU (RTX 4090, ~1 billion SHA-512 ops/sec), this yields approximately **390,000 password guesses/second**.
4. A 10-million-entry common-password dictionary is exhausted in **~25 seconds**. An 8-character alphanumeric brute-force completes in days.
5. Recovered plaintext is the raw Hedera ED25519 or ECDSA private key, which can be used to sign and submit transactions draining the account.

**Contrast with the migration path:** `dataMigration.ts` uses Argon2id with `memoryCost: 262144` (256 MB) and `timeCost: 3` for the same purpose — demonstrating the team is aware of proper KDF hardening but did not apply it to the primary key-storage path.

### Impact Explanation

- **Direct asset theft:** Recovery of the Hedera private key allows the attacker to sign arbitrary transactions, transferring all HBAR and tokens from the account with no recourse.
- **Permanent:** Once the private key is recovered, the account is permanently compromised unless the user rotates keys on-chain before the attacker acts.
- **Scope:** Affects every user who stores keys via the password path (i.e., users who do not use the OS keychain). The `useKeychain` flag is optional and not the default for all platforms.

### Likelihood Explanation

- **Attacker precondition:** Obtain the SQLite database file. For a desktop application, this is realistic via: (a) malware/RAT on the user's machine, (b) unencrypted backup (Time Machine, cloud sync), (c) physical access to an unlocked machine.
- **No privileged access required:** The database file is a plain SQLite file in the Electron app's user-data directory, readable by any process running as the same OS user.
- **Speed:** 2560 iterations makes common-password attacks trivial (~25 seconds for a 10M dictionary). Even moderately complex passwords (8–10 characters) are crackable within hours on commodity hardware.
- **No online rate-limiting applies:** This is a fully offline attack against the ciphertext blob.

### Recommendation

Replace the PBKDF2-based `deriveKey` with Argon2id (already used in `dataMigration.ts`) using parameters consistent with OWASP 2023 guidance:

```typescript
// Minimum: memoryCost=65536 (64MB), timeCost=3, parallelism=4
// Preferred: memoryCost=262144 (256MB), timeCost=3, parallelism=1
import * as argon2 from 'argon2';

export async function deriveKey(password: string, salt: Buffer): Promise<Buffer> {
  return argon2.hash(password, {
    type: argon2.argon2id,
    memoryCost: 65536,
    timeCost: 3,
    parallelism: 4,
    salt,
    raw: true,
    hashLength: 32,
  });
}
```

If PBKDF2 must be retained for compatibility, use a minimum of **210,000 iterations** (OWASP 2023 for PBKDF2-SHA512). Add a comment documenting the rationale and the security assumption (e.g., minimum password entropy, target offline attack resistance). Existing encrypted blobs must be re-encrypted on next login using the stronger KDF.

### Proof of Concept

**Setup:** Extract `private_key` column from the SQLite `KeyPair` table for a user who does not use the OS keychain.

**Attack script (pseudocode):**
```python
import hashlib, base64
from Crypto.Cipher import AES

blob = base64.b64decode(private_key_column_value)
salt, iv, tag, ciphertext = blob[:64], blob[64:80], blob[80:96], blob[96:]

for password in open('rockyou.txt'):
    key = hashlib.pbkdf2_hmac('sha512', password.strip().encode(), salt, 2560, 32)
    try:
        cipher = AES.new(key, AES.MODE_GCM, nonce=iv)
        plaintext = cipher.decrypt_and_verify(ciphertext, tag)
        print(f"Private key recovered: {plaintext.decode()}")
        break
    except ValueError:
        continue
```

**Expected outcome:** The `rockyou.txt` top-10M dictionary (covering the majority of real-world user passwords) is exhausted in under 30 seconds on a single GPU, yielding the plaintext Hedera private key.
