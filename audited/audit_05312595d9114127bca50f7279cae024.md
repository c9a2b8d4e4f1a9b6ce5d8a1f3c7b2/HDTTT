### Title
Unbounded Array Input to `uploadSignatureMap` Enables Resource Exhaustion via Disproportionate Per-Item CPU Work

### Summary
The `POST /transactions/:transactionId/signers` endpoint accepts an unbounded array of `UploadSignatureMapDto` objects with no size cap. Each element triggers multiple expensive CPU operations — at minimum two `SDKTransaction.fromBytes()` deserializations and nested cryptographic loops over the attacker-controlled `SignatureMap`. An authenticated user can submit a single request containing thousands of items, causing server CPU exhaustion proportional to the array length with no upper bound enforced anywhere in the stack.

### Finding Description

**Entry point — no array size limit:**

`signers.controller.ts` line 103 accepts `UploadSignatureMapDto | UploadSignatureMapDto[]` with no `@ArrayMaxSize()` or equivalent guard:

```typescript
async uploadSignatureMap(
  @Body() body: UploadSignatureMapDto | UploadSignatureMapDto[],
  ...
```

The `UploadSignatureMapDto` itself only carries `@IsDefined()`, `@IsNumber()`, `@IsNotEmptyObject()`, and `@IsSignatureMap()` — none of which constrain the outer array length. [1](#0-0) [2](#0-1) 

**Per-item expensive work — first deserialization in `validateTransactionStatus`:**

For every element in the array, `validateTransactionStatus` calls `SDKTransaction.fromBytes()`:

```typescript
const sdkTransaction = SDKTransaction.fromBytes(transaction.transactionBytes);
if (isExpired(sdkTransaction)) { ... }
``` [3](#0-2) 

**Per-item expensive work — second deserialization + nested cryptographic loop in `processTransactionSignatures`:**

A second `SDKTransaction.fromBytes()` is called, followed by a triple-nested loop over the attacker-controlled `SignatureMap`. For each unique public key found, `sdkTransaction.addSignature(publicKey, map)` is called, passing the **entire map** — the SDK internally re-iterates all nodes and transaction IDs in the map for each key:

```typescript
let sdkTransaction = SDKTransaction.fromBytes(transaction.transactionBytes); // 2nd deserialization

for (const nodeMap of map.values()) {
  for (const txMap of nodeMap.values()) {
    for (const publicKey of txMap.keys()) {
      // ...
      sdkTransaction = sdkTransaction.addSignature(publicKey, map); // full map re-iterated per key
    }
  }
}
``` [4](#0-3) 

**Analog to the external report:**

The external report describes `pop_accept` iterating over ALL addresses in `current_` — including those merely accessed but not modified in the current call frame. The analog here is two-fold:

1. The outer array is unbounded: the server iterates over ALL submitted items, performing full deserialization and crypto work for each, even items that will ultimately fail validation.
2. Inside `processTransactionSignatures`, `addSignature(publicKey, map)` passes the entire `SignatureMap` for each unique public key. The SDK re-scans all nodes × transaction IDs in the map per key, meaning work scales as O(nodes × txIds × uniqueKeys) per submitted item — all attacker-controlled dimensions.

The same pattern exists in `importSignatures` in `transactions.service.ts`, which also accepts an unbounded `UploadSignatureMapDto[]` array and performs per-item deserialization and signature validation. [5](#0-4) 

### Impact Explanation

A single authenticated HTTP request with N items causes at minimum 2N `SDKTransaction.fromBytes()` calls plus O(N × nodes × txIds × keys) cryptographic iterations. With N = 5,000 items and a `SignatureMap` containing 10 nodes × 10 transaction IDs × 5 keys, the server performs ≥10,000 deserializations and ≥2,500,000 map-scan iterations in one request. This exhausts CPU, degrades response times for all concurrent users, and can render the API service unavailable — a sustained condition achievable by a single attacker replaying the request.

### Likelihood Explanation

The attacker requires only a valid authenticated session (any registered user, no admin role). The attacker creates or references existing transaction IDs (they need not own them — the service fetches them by ID and only errors per-item, not early-exit), then crafts a single POST body with an arbitrarily large array. No special tooling is needed beyond a standard HTTP client. The two affected endpoints (`POST /transactions/:transactionId/signers` and `POST /transactions/import-signatures`) are both reachable by any verified user. [6](#0-5) 

### Recommendation

1. Add `@ArrayMaxSize(100)` (or a configurable limit) to the `uploadSignatureMap` controller parameter and enforce the same limit in `importSignatures`.
2. Add a size validator to `UploadSignatureMapDto` or in `transformAndValidateDto` to reject arrays exceeding the limit before any processing begins.
3. Optionally, add a structural size cap inside `@IsSignatureMap()` to bound the number of nodes, transaction IDs, and public keys within a single `SignatureMap` object, preventing the inner O(nodes × txIds × keys) amplification.

### Proof of Concept

1. Register as a normal user and obtain a JWT token.
2. Create or collect 5,000 valid transaction IDs in `WAITING_FOR_SIGNATURES` status.
3. Craft a `SignatureMap` with 10 node account IDs, 10 transaction IDs per node, and 5 public keys per transaction ID (500 entries total).
4. Send a single `POST /transactions/signers` request with a JSON body array of 5,000 objects, each containing one of the transaction IDs and the crafted `SignatureMap`.
5. Observe: the server performs ≥10,000 `SDKTransaction.fromBytes()` calls and ≥2,500,000 inner map iterations synchronously within the single request, spiking CPU and blocking the event loop for the duration.
6. Repeat the request at a low rate (e.g., one request every few seconds) to sustain service degradation without triggering rate limiting on request count.

  private async processTransactionSignatures(
    transaction: Transaction,
    map: SignatureMap,
    userKeyMap: Map<string, UserKey>,
    existingSignerIds: Set<number>
  ) {
    let sdkTransaction = SDKTransaction.fromBytes(transaction.transactionBytes);

    const userKeys: UserKey[] = [];
    const processedRawKeys = new Set<string>();

    // To explain what is going on here, we need to understand how sdkTransaction.addSignature works.
    // The addSignature method will go through each inner transaction, then go through the map
    // and pull the signatures for the supplied public key belonging to that inner transaction
    // (denoted by the node and transaction id), add the signatures to the inner transactions.
    // So we need to go through the map and get each unique publicKey and call addSignature one time
    // per key.
    for (const nodeMap of map.values()) {
      for (const txMap of nodeMap.values()) {
        for (const publicKey of txMap.keys()) {
          const raw = publicKey.toStringRaw();

          // Skip duplicates across node/tx maps, and already-processed keys
          if (processedRawKeys.has(raw)) continue;
          processedRawKeys.add(raw);

          // Look up key (raw first, then DER)
          let userKey = userKeyMap.get(raw);
          if (!userKey) {
            userKey = userKeyMap.get(publicKey.toStringDer());
          }
          if (!userKey) throw new Error(ErrorCodes.PNY);

          // Only add the signature once per unique key
          sdkTransaction = sdkTransaction.addSignature(publicKey, map);

          // Only return "new" signers (not already persisted)
          if (!existingSignerIds.has(userKey.id)) {
            userKeys.push(userKey);
          }
        }
      }
    }

    // Finally, compare the resulting transaction bytes to see if any signatures were actually added
    const isSameBytes = Buffer.from(sdkTransaction.toBytes()).equals(
      transaction.transactionBytes
    );

    return { sdkTransaction, userKeys, isSameBytes };
  }

  async importSignatures(
    dto: UploadSignatureMapDto[],
    user: User,
  ): Promise<SignatureImportResultDto[]> {
    type UpdateRecord = {
      id: number;
      transactionBytes: Buffer;
      transactionId: string;
      network: string;
    };

    const ids = dto.map(d => d.id);

    // Single batch query for all transactions
    const transactions = await this.entityManager.find(Transaction, {
      where: { id: In(ids) },
      relations: ['creatorKey', 'approvers', 'signers', 'observers'],
    });

    if (transactions.length === 0) {
      return ids.map(id => ({
        id,
        error: new BadRequestException(ErrorCodes.TNF).message,
      }));
    }

    // Create a map for quick lookup
    const transactionMap = new Map(transactions.map(t => [t.id, t]));

    const results = new Map<number, SignatureImportResultDto>();
    const updates = new Map<number, UpdateRecord>();

    for (const { id, signatureMap: map } of dto) {
      const transaction = transactionMap.get(id);

      try {
        /* Verify that the transaction exists and access is verified */
        if (!(await this.verifyAccess(transaction, user))) {
          throw new BadRequestException(ErrorCodes.TNF);
        }

        /* Checks if the transaction is canceled */
        if (
          transaction.status !== TransactionStatus.WAITING_FOR_SIGNATURES &&
          transaction.status !== TransactionStatus.WAITING_FOR_EXECUTION
        )
          throw new BadRequestException(ErrorCodes.TNRS);

        /* Checks if the transaction is expired */
        const sdkTransaction = SDKTransaction.fromBytes(transaction.transactionBytes);
        if (isExpired(sdkTransaction)) throw new BadRequestException(ErrorCodes.TE);

        /* Validates the signatures */
        const { data: publicKeys, error } = safe<PublicKey[]>(
          validateSignature.bind(this, sdkTransaction, map),
        );
        if (error) throw new BadRequestException(ErrorCodes.ISNMPN);

        for (const publicKey of publicKeys) {
          sdkTransaction.addSignature(publicKey, map);
        }

        transaction.transactionBytes = Buffer.from(sdkTransaction.toBytes());

        results.set(id, { id });
        updates.set(id, {
          id,
          transactionBytes: transaction.transactionBytes,
          transactionId: transaction.transactionId,
          network: transaction.mirrorNetwork,
        });
      } catch (error) {
        results.set(id, {
          id,
          error:
            (error instanceof BadRequestException)
              ? error.message
              : 'An unexpected error occurred while importing the signatures',
        });
      }
    }
