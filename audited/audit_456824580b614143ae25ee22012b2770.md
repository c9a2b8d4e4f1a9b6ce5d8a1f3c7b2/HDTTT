### Title
TOCTOU Race Condition in `bulkUpdateTransactions` Allows Signature Upload to Corrupt Already-Executed Transaction State

### Summary
The `SignersService.bulkUpdateTransactions` method performs a raw SQL `UPDATE` on `transactionBytes` without including a status guard in the `WHERE` clause. Transaction status is validated from an in-memory snapshot taken earlier in `loadTransactionData`, but between that check and the SQL execution, the transaction can transition to a terminal state (e.g., `EXECUTED`, `CANCELED`). This is the direct analog of the external report's missing `require(remainingMakerAmount > 0)` guard: an action is permitted on an already-completed state because the completion check is not atomic with the state mutation.
