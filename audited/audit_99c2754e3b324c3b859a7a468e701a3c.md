### Title
Cross-Network Replay of Approval Signatures Due to Missing Network Binding in `verifyTransactionBodyWithoutNodeAccountIdSignature`

### Summary
The approval signature schema in `approvers.service.ts` verifies signatures over transaction body bytes that explicitly strip the node account ID and include no network/ledger identifier (`mirrorNetwork`). A malicious organization member can craft a mainnet transaction whose body bytes are identical to a testnet transaction, wait for a victim approver to sign the testnet version, then replay that approval signature against the mainnet transaction — bypassing the approval threshold and causing unauthorized mainnet execution.

### Finding Description

**Root cause — the signed data omits all network context.**

`verifyTransactionBodyWithoutNodeAccountIdSignature` in `back-end/libs/common/src/utils/sdk/transaction.ts` builds the bytes to verify by calling `_makeTransactionBody(null)`, passing `null` for the node account ID: [1](#0-0) 

The resulting `bodyBytes` contain only: `transactionID` (payer account + valid-start timestamp), memo, fee, valid duration, and the operation payload. They contain **no ledger ID, no `mirrorNetwork` string, and no node account IDs**. Hedera's protobuf `TransactionBody` has no chain/ledger field.

This function is the sole verification step in `approveTransaction`: [2](#0-1) 

The `transaction.mirrorNetwork` field is stored in the database but is never included in the signed payload and is never checked against the approval signature.

**Exploit path (step-by-step):**

1. Attacker (a legitimate organization member who can create transactions) constructs a Hedera transaction body with a chosen payer account, valid-start timestamp, memo, fee, and operation — e.g., a large HBAR transfer on **mainnet**.
2. Attacker submits the same body bytes to the organization server twice: once with `mirrorNetwork = "testnet"` and once with `mirrorNetwork = "mainnet"` (using a slightly different valid-start to avoid the duplicate-transaction-ID guard, or exploiting the fact that the duplicate check is per `transactionId` string, not per network).
3. Attacker sets the victim user as an approver on **both** transactions.
4. Victim sees the testnet transaction, considers it low-risk, and approves it — producing signature `S` over `_makeTransactionBody(null)`.
5. Attacker submits `S` as the approval for the **mainnet** transaction via `POST /transactions/:id/approvers/approve`.
6. The server calls `verifyTransactionBodyWithoutNodeAccountIdSignature` with the mainnet transaction's bytes. Because the body bytes (minus node account ID) are identical, `S` verifies successfully.
7. The mainnet transaction's approval threshold is now satisfied; the chain service executes it on mainnet. [3](#0-2) 

### Impact Explanation

An attacker can cause a mainnet Hedera transaction to be approved and executed without the victim ever intending to approve a mainnet action. Depending on the transaction type this can result in:
- Unauthorized transfer of HBAR or tokens from the organization's accounts.
- Unauthorized account key updates, node operations, or file changes on mainnet.

The approval mechanism is the primary human-authorization gate before execution; bypassing it removes the last line of defense before funds move on-chain.

### Likelihood Explanation

The attacker must be a legitimate organization member with the ability to create transactions (no admin/owner keys required). This is the default role for any user in Organization Mode. The attack requires crafting two transactions with matching body bytes — straightforward since the attacker controls all fields of the transaction they submit. No cryptographic break is required; the vulnerability is purely in the missing domain separator.

### Recommendation

Include the `mirrorNetwork` (or a canonical ledger identifier) in the data that approvers sign. The simplest fix is to hash `mirrorNetwork || transactionBodyBytes` as the signed payload, and verify against the same composite on the server side. Alternatively, adopt a structured domain-separated signing scheme (analogous to EIP-712) that binds the signature to `{ network, transactionId, organizationServerId }`. [4](#0-3) 

### Proof of Concept

```
1. Register as a normal user on an organization server that manages both testnet and mainnet.

2. Build a TransactionBody proto with:
     payer    = 0.0.X (an account you control on both networks)
     validStart = T
     memo     = "test"
     data     = CryptoTransfer { 0.0.Y → 100 HBAR }

3. POST /transactions  { transactionBytes: <bytes>, mirrorNetwork: "testnet.mirrornode.hedera.com", ... }
   → server stores transaction A (testnet).

4. POST /transactions  { transactionBytes: <same bytes>, mirrorNetwork: "mainnet-public.mirrornode.hedera.com", ... }
   → server stores transaction B (mainnet).  (Use validStart = T+1 if duplicate-ID guard fires.)

5. POST /transactions/A/approvers  { userId: victim }
   POST /transactions/B/approvers  { userId: victim }

6. Victim approves transaction A (testnet):
   POST /transactions/A/approvers/approve  { userKeyId, signature: S, approved: true }

7. Attacker replays:
   POST /transactions/B/approvers/approve  { userKeyId, signature: S, approved: true }

8. Server calls verifyTransactionBodyWithoutNodeAccountIdSignature(txB_bytes, S, publicKey).
   Because _makeTransactionBody(null) produces identical bytes for A and B, S verifies → approved: true stored.

9. Transaction B (mainnet) threshold is met → chain service executes the 100 HBAR transfer on mainnet.
``` [5](#0-4) [1](#0-0)

/* Get transaction body bytes without node account id */
export const getTransactionBodyBytes = (transaction: SDKTransaction) => {
  // @ts-expect-error - _makeTransactionBody is a private method
  const transactionBody = transaction._makeTransactionBody(null);
  return proto.TransactionBody.encode(transactionBody).finish();
};

/* Verify the signature of the transaction body without node account id */
export const verifyTransactionBodyWithoutNodeAccountIdSignature = (
  transaction: SDKTransaction,
  signature: string | Buffer,
  publicKey: string | PublicKey,
) => {
  const bodyBytes = getTransactionBodyBytes(transaction);

  /* Deserialize Public Key */
  publicKey = publicKey instanceof PublicKey ? publicKey : PublicKey.fromString(publicKey);

  /* Deserialize Signature */
  signature = typeof signature === 'string' ? decode(signature) : signature;

  try {
    return publicKey.verify(bodyBytes, signature);
  } catch (err) {
    console.log(err);
    return false;
  }
};

  async approveTransaction(
    dto: ApproverChoiceDto,
    transactionId: number,
    user: User,
  ): Promise<boolean> {
    /* Get all the approvers */
    const approvers = await this.getVerifiedApproversByTransactionId(transactionId, user);

    /* If user is approver, filter the records that belongs to the user */
    const userApprovers = approvers.filter(a => a.userId === user.id);

    /* Check if the user is an approver */
    if (userApprovers.length === 0)
      throw new UnauthorizedException('You are not an approver of this transaction');

    /* Check if the user has already approved the transaction */
    if (userApprovers.every(a => a.signature)) throw new BadRequestException(ErrorCodes.TAP);

    /* Ensures the user keys are passed */
    await attachKeys(user, this.dataSource.manager);
    if (user.keys.length === 0) return false;

    const signatureKey = user.keys.find(key => key.id === dto.userKeyId);

    /* Gets the public key that the signature belongs to */
    const publicKey = PublicKey.fromString(signatureKey?.publicKey);

    /* Get the transaction body */
    const transaction = await this.dataSource.manager.findOne(Transaction, {
      where: { id: transactionId },
      relations: { creatorKey: true, observers: true },
    });

    /* Check if the transaction exists */
    if (!transaction) throw new BadRequestException(ErrorCodes.TNF);

    /* Checks if the transaction is requires approval */
    if (
      transaction.status !== TransactionStatus.WAITING_FOR_SIGNATURES &&
      transaction.status !== TransactionStatus.WAITING_FOR_EXECUTION
    )
      throw new BadRequestException(ErrorCodes.TNRA);

    const sdkTransaction = SDKTransaction.fromBytes(transaction.transactionBytes);

    /* Verify the signature matches the transaction */
    if (
      !verifyTransactionBodyWithoutNodeAccountIdSignature(sdkTransaction, dto.signature, publicKey)
    )
      throw new BadRequestException(ErrorCodes.SNMP);

    /* Update the approver with the signature */
    await this.dataSource.transaction(async transactionalEntityManager => {
      await transactionalEntityManager
        .createQueryBuilder()
        .update(TransactionApprover)
        .set({
          userKeyId: dto.userKeyId,
          signature: dto.signature,
          approved: dto.approved,
        })
        .whereInIds(userApprovers.map(a => a.id))
        .execute();
    });

    const notificationEvent = [{ entityId: transaction.id }];

    if (!dto.approved || userApprovers.every(a => a.approved)) {
      emitTransactionStatusUpdate(this.notificationsPublisher, notificationEvent);
    } else {
      emitTransactionUpdate(this.notificationsPublisher, notificationEvent);
    }

    return true;
