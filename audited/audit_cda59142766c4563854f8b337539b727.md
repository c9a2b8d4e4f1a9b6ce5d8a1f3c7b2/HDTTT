### Title
Division by Zero Yields `Infinity` in `getSignedTransactionsDimensions`, Causing an Infinite Loop DoS via Crafted Transaction Bytes

---

### Summary

In `back-end/libs/common/src/utils/sdk/transaction.ts`, the helper `getSignedTransactionsDimensions` computes `columns` by dividing `_signedTransactions.length` by `rowLength` (which equals `_nodeAccountIds.length`). In JavaScript, dividing a positive integer by zero produces `Infinity`, not a runtime error. The immediately following `for` loop iterates `col < columns`, which becomes `col < Infinity` — an infinite loop. Any organization-mode user who can submit crafted transaction bytes with zero node account IDs but at least one signed transaction entry can trigger this, permanently stalling the back-end worker that processes the request.

---

### Finding Description

**Root cause — `getSignedTransactionsDimensions`:**

```typescript
// back-end/libs/common/src/utils/sdk/transaction.ts  L188-210
const getSignedTransactionsDimensions = (transaction: SDKTransaction) => {
  const rowLength = transaction._nodeAccountIds.length;          // L189 — can be 0
  const columns = transaction._signedTransactions.length / rowLength; // L190 — 0-div → Infinity

  ...
  for (let col = 0; col < columns; col++) {                      // L200 — infinite when Infinity
    const bodyBytes = transaction._signedTransactions.get(col * rowLength).bodyBytes;
    ...
  }
``` [1](#0-0) 

JavaScript's numeric semantics:
- `positiveN / 0 → Infinity`
- `0 < Infinity → true` always
- `col++` never makes the condition false → the loop never terminates.

The only safe case is `0 / 0 → NaN`, where `0 < NaN` is `false` and the loop body is skipped entirely. The dangerous case is `_signedTransactions.length > 0` **and** `_nodeAccountIds.length === 0`.

**Call chain into the vulnerable function:**

`getSignedTransactionsDimensions` is called exclusively by `validateSignature`, which is exported from the same file:

```typescript
export const validateSignature = (transaction: SDKTransaction, signatureMap: SignatureMap) => {
  const { rowLength, nodeAccountIdRow, transactionIdCol } =
    getSignedTransactionsDimensions(transaction);   // triggers the infinite loop
``` [2](#0-1) 

`validateSignature` is part of the shared `@app/common` library consumed by the API service's signature-import pipeline (evidenced by `signature-import-result.dto.ts` in the API service and the signature-collation workflow described in the wiki). Organization-mode users submit raw transaction bytes and signature maps through the API; the API deserialises the bytes with `SDKTransaction.fromBytes` and passes the result to `validateSignature`.

**How to produce a zero-`rowLength` transaction:**

The Hedera SDK stores node account IDs in `_nodeAccountIds`. A transaction serialised before `freeze()` is called, or one whose protobuf body is hand-crafted to omit the node account ID list, will deserialise with `_nodeAccountIds.length === 0`. If the same bytes also carry at least one entry in `_signedTransactions` (e.g., a partial signature was attached before the node list was cleared), the precondition is met.

---

### Impact Explanation

When the infinite loop is entered, the Node.js event-loop iteration that is executing `validateSignature` never yields. Because Node.js is single-threaded per worker, the entire back-end API worker process is monopolised: no other requests are served, database connections time out, and the service becomes completely unresponsive until the process is killed and restarted. This is a **server-side CPU-burning denial of service** that requires no privileged credentials — only the ability to submit a crafted transaction to the organisation API.

---

### Likelihood Explanation

- **Attacker profile:** Any registered organisation user (no admin role required) who can reach the signature-import or transaction-submission endpoint.
- **Craft difficulty:** Low. The attacker needs to serialise a Hedera `Transaction` object that has at least one entry in `_signedTransactions` but an empty `_nodeAccountIds` list. This is achievable by calling `sign()` on a transaction before calling `freeze()` (which is what populates the node list), then serialising the bytes.
- **No rate-limiting defence:** A single malformed request is sufficient; the loop never exits, so the worker is permanently stalled after one request.

---

### Recommendation

1. **Guard against zero `rowLength` before the division:**
   ```typescript
   const getSignedTransactionsDimensions = (transaction: SDKTransaction) => {
     const rowLength = transaction._nodeAccountIds.length;
     if (rowLength === 0) {
       throw new Error('Transaction has no node account IDs; cannot compute dimensions');
     }
     const columns = transaction._signedTransactions.length / rowLength;
     ...
   ```
2. **Validate transaction structure before calling `validateSignature`:** Reject any deserialized transaction whose `_nodeAccountIds.length === 0` at the API boundary, before it reaches business logic.
3. **Add a unit test** that passes a transaction with `_nodeAccountIds.length === 0` to `validateSignature` and asserts it throws rather than hanging.

---

### Proof of Concept

```typescript
import { Transaction, PrivateKey } from '@hiero-ledger/sdk';
import { AccountCreateTransaction } from '@hiero-ledger/sdk';
import { validateSignature } from 'back-end/libs/common/src/utils/sdk/transaction';

// 1. Build a transaction WITHOUT freezing (no node account IDs assigned).
const tx = new AccountCreateTransaction();

// 2. Sign it — this populates _signedTransactions even though _nodeAccountIds is empty.
const key = PrivateKey.generateED25519();
await tx.sign(key);

// 3. Serialise and deserialise (simulating what the API does with user-supplied bytes).
const bytes = tx.toBytes();
const deserialized = Transaction.fromBytes(bytes);

// 4. Call validateSignature — enters infinite loop, worker hangs permanently.
validateSignature(deserialized, deserialized.getSignatures());
// ^ never returns; Node.js process is now stalled
```

export const validateSignature = (transaction: SDKTransaction, signatureMap: SignatureMap) => {
  const signerPublicKeys: PublicKey[] = [];

  const { rowLength, nodeAccountIdRow, transactionIdCol } =
    getSignedTransactionsDimensions(transaction);

  for (const [nodeAccountId, transactionIds] of signatureMap._map) {
    for (const [transactionId, publicKeys] of transactionIds._map) {
      for (const [publicKeyDer, signature] of publicKeys._map) {
        const publicKey = PublicKey.fromString(publicKeyDer);
        const publicKeyHex = publicKey.toStringRaw();

        const alreadySigned =
          transaction._signerPublicKeys.has(publicKeyHex) ||
          transaction._signerPublicKeys.has(publicKeyDer);

        if (!alreadySigned) {
          const row = nodeAccountIdRow[nodeAccountId];
          const col = transactionIdCol[transactionId];

          const bodyBytes = transaction._signedTransactions.get(col * rowLength + row).bodyBytes;

          const signatureValid = publicKey.verify(bodyBytes, signature);

          if (signatureValid) {
            signerPublicKeys.push(publicKey);
          } else {
            throw new Error('Invalid signature');
          }
        }
      }
    }
  }

  return signerPublicKeys;
};
```
