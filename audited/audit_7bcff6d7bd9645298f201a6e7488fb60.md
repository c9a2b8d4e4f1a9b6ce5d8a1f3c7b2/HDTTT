### Title
`cutoffAt` Can Be Set After `validStart`, Creating Temporally Inconsistent Transaction State

### Summary
The `CreateTransactionDto` in the back-end API accepts a `cutoffAt` field (the deadline for collecting signatures) without validating that it is earlier than the transaction's `validStart` (the execution time). Any authenticated user can submit a transaction where the signing cutoff is set equal to or after the execution time, inverting the intended temporal ordering and creating an inconsistent application state where the system may continue accepting or reminding signers for a transaction that has already executed or expired.

### Finding Description

**Vulnerability class**: Missing temporal ordering constraint (state transition invariant violation).

The `cutoffAt` field represents the deadline after which signatures should no longer be collected. The `validStart` field (embedded in `transactionBytes` and extracted server-side) represents when the Hedera transaction executes. The invariant `cutoffAt < validStart` must hold for the workflow to be coherent.

**Root cause — `CreateTransactionDto`**: [1](#0-0) 

`cutoffAt` is decorated only with `@IsDate()` and `@IsOptional()`. There is no cross-field validator enforcing `cutoffAt < validStart`.

**Root cause — `validateAndPrepareTransaction`**: [2](#0-1) 

The method validates signature, expiry, size, and node correctness, but performs **no check** that `dto.cutoffAt` is before `sdkTransaction.transactionId.validStart.toDate()`. The `cutoffAt` value is passed through directly: [3](#0-2) 

**Persistence — `createTransactions`**: [4](#0-3) 

The unchecked `cutoffAt` is saved directly to the database alongside `validStart`.

**Exploit path**:
1. Authenticated user constructs a valid Hedera transaction with `validStart = T`.
2. User submits `POST /transactions` with `cutoffAt = T + 1 hour` (after execution time).
3. Server validates signature, expiry, size, nodes — all pass.
4. Transaction is persisted with `cutoffAt > validStart`.
5. The application now holds a record where the signing deadline is after the execution deadline, violating the core workflow invariant.

### Impact Explanation

The application's transaction lifecycle assumes `cutoffAt < validStart`. When this is violated:
- The system may continue sending signature reminder notifications and accepting signature submissions for a transaction that has already executed or expired on the Hedera network.
- Any query or scheduler logic that uses `cutoffAt` to gate signature collection will operate on a logically impossible state.
- In the organization multi-signature workflow, approvers and signers may be prompted to act on a transaction that is no longer actionable, degrading trust and correctness of the collaborative signing process. [5](#0-4) 

### Likelihood Explanation

Any authenticated user of the organization back-end can reach `POST /transactions`. No admin or privileged role is required. The `cutoffAt` field is an optional, user-supplied date with no server-side ordering constraint. A malicious or misconfigured client can trivially supply an out-of-order value. The front-end does not enforce this constraint either, and the API is directly callable.

### Recommendation

Add a cross-field validation in `validateAndPrepareTransaction` (or as a custom class-validator decorator on `CreateTransactionDto`) that rejects any request where `cutoffAt >= validStart`:

```typescript
// In validateAndPrepareTransaction, after extracting validStart:
const validStart = sdkTransaction.transactionId.validStart.toDate();
if (dto.cutoffAt && dto.cutoffAt >= validStart) {
  throw new BadRequestException(
    'cutoffAt must be earlier than the transaction validStart'
  );
}
``` [6](#0-5) 

### Proof of Concept

1. Authenticate as any valid organization user and obtain a JWT.
2. Construct a Hedera `AccountCreateTransaction` with `validStart = now + 60s`.
3. Submit:
```http
POST /transactions
Authorization: Bearer <token>
Content-Type: application/json

{
  "name": "test",
  "description": "poc",
  "transactionBytes": "<hex of tx with validStart = now+60s>",
  "signature": "<valid creator signature>",
  "creatorKeyId": <id>,
  "mirrorNetwork": "testnet",
  "cutoffAt": "<now + 3600s>"   // ← after validStart
}
```
4. Server returns HTTP 201. The persisted transaction record has `cutoffAt > validStart`.
5. Query the transaction back — `cutoffAt` is stored as a future date beyond `validStart`, confirming the invariant is violated with no server-side rejection. [7](#0-6)

  private async validateAndPrepareTransaction(
    dto: CreateTransactionDto,
    user: User,
    client: Client,
  ) {
    const creatorKey = user.keys.find(key => key.id === dto.creatorKeyId);

    if (!creatorKey) {
      throw new BadRequestException(`Creator key ${dto.creatorKeyId} not found`);
    }

    const publicKey = PublicKey.fromString(creatorKey.publicKey);

    // Verify signature
    const validSignature = publicKey.verify(dto.transactionBytes, dto.signature);
    if (!validSignature) {
      throw new BadRequestException(ErrorCodes.SNMP);
    }

    // Parse SDK transaction
    const sdkTransaction = SDKTransaction.fromBytes(dto.transactionBytes);

    // Check the transaction is frozen, cannot require it to be frozen, breaks backwards compatibility
    if (!sdkTransaction.isFrozen()) {
      sdkTransaction.freezeWith(client);
    }

    // Check if expired
    if (isExpired(sdkTransaction)) {
      throw new BadRequestException(ErrorCodes.TE);
    }

    // Check size
    if (isTransactionBodyOverMaxSize(sdkTransaction)) {
      throw new BadRequestException(ErrorCodes.TOS);
    }

    // Check nodes
    const allowedNodes = getNodeAccountIdsFromClientNetwork(client);
    if (!isTransactionValidForNodes(sdkTransaction, allowedNodes)) {
      throw new BadRequestException(ErrorCodes.TNVN);
    }

    const transactionHash = await sdkTransaction.getTransactionHash();
    const transactionType = getTransactionTypeEnumValue(sdkTransaction);

    // Extract new keys if applicable
    let publicKeys: string[] | null = null;
    try {
      let keyToExtract: Key | null = null;

      if (transactionType === TransactionType.ACCOUNT_UPDATE) {
        keyToExtract = (sdkTransaction as AccountUpdateTransaction).key;
      } else if (transactionType === TransactionType.NODE_UPDATE) {
        keyToExtract = (sdkTransaction as NodeUpdateTransaction).adminKey;
      } else if (transactionType === TransactionType.NODE_CREATE) {
        keyToExtract = (sdkTransaction as NodeCreateTransaction).adminKey;
      }

      if (keyToExtract) {
        publicKeys = flattenKeyList(keyToExtract).map(pk => pk.toStringRaw());
      }
    } catch (error) {
      // Log but don't fail - publicKeys will remain null
      console.error(`Failed to extract public keys from transaction ${sdkTransaction.transactionId}:`, error);
    }

    return {
      name: dto.name,
      type: transactionType,
      description: dto.description,
      transactionId: sdkTransaction.transactionId.toString(),
      transactionHash: encodeUint8Array(transactionHash),
      transactionBytes: sdkTransaction.toBytes(),
      unsignedTransactionBytes: sdkTransaction.toBytes(),
      creatorKeyId: dto.creatorKeyId,
      signature: dto.signature,
      mirrorNetwork: dto.mirrorNetwork,
      validStart: sdkTransaction.transactionId.validStart.toDate(),
      isManual: dto.isManual,
      cutoffAt: dto.cutoffAt,
      publicKeys,
    };
  }
