### Title
Unhandled SDK Exception in `IsSignatureMap` Decorator Causes 500 Error on Malformed Public Key Input

### Summary

The `IsSignatureMap` decorator used to validate and transform the `signatureMap` field in `UploadSignatureMapDto` calls `PublicKey.fromString(publicKey)` and `decode(signature)` without any try/catch guard. When an authenticated user submits a malformed public key string or a non-string signature value, the Hiero SDK throws a generic `Error` (not a `BadRequestException`). Because the global exception filters only handle `BadRequestException` and `NotFoundException`, this unhandled exception propagates and returns a 500 Internal Server Error instead of a proper 400 Bad Request.

### Finding Description

In `back-end/libs/common/src/decorators/is-signature-map.decorator.ts`, the `Transform` callback iterates over the user-supplied signature map and calls:

```typescript
const decodedSignature = new Uint8Array(decode(signature));   // line 49
...
signatureMap.addSignature(
  AccountId.fromString(nodeAccountId),
  TransactionId.fromString(transactionId),
  PublicKey.fromString(publicKey),          // line 58 — no try/catch
  decodedSignature,
);
``` [1](#0-0) 

Two distinct crash paths exist:

**Path 1 — Invalid public key string:** `publicKey` is always a string (JavaScript object key), but it may be an invalid Hedera public key. `PublicKey.fromString(publicKey)` throws a generic SDK `Error`, not a `BadRequestException`.

**Path 2 — Non-string signature value:** `decode(signature)` calls `hexString.startsWith(...)`:

```typescript
export const decode = (hexString: string): Buffer =>
  Buffer.from(hexString.startsWith('0x') ? hexString.slice(2) : hexString, 'hex');
``` [2](#0-1) 

If `signature` is a JSON object, array, or number (all valid JSON), `hexString.startsWith` throws a `TypeError`.

Neither exception is a `BadRequestException`, so neither is caught by the global `BadRequestExceptionFilter`:

```typescript
@Catch(BadRequestException)
export class BadRequestExceptionFilter implements ExceptionFilter { ... }
``` [3](#0-2) 

The global setup registers only `NotFoundExceptionFilter` and `BadRequestExceptionFilter` — no catch-all filter:

```typescript
app.useGlobalFilters(new NotFoundExceptionFilter(), new BadRequestExceptionFilter());
``` [4](#0-3) 

The endpoint that triggers this path is `POST /transactions/:transactionId/signers`: [5](#0-4) 

The `UploadSignatureMapDto` applies `@IsSignatureMap()` to the `signatureMap` field: [6](#0-5) 

Contrast with the correct pattern used elsewhere in the codebase — `IsHederaPublicKeyConstraint` wraps `PublicKey.fromString` in try/catch: [7](#0-6) 

The `IsSignatureMap` decorator does not apply the same defensive pattern.

### Impact Explanation

Any authenticated user can send a POST request to `POST /transactions/:transactionId/signers` with a signature map containing an invalid public key string (e.g., `"notakey"`) or a non-string signature value (e.g., `{}`). The server returns a 500 Internal Server Error instead of a 400 Bad Request. This breaks the expected API contract, may expose internal error details in non-production environments, and disrupts the transaction signing workflow for legitimate users sharing the same session context.

### Likelihood Explanation

The endpoint requires a valid JWT (`JwtBlackListAuthGuard`, `JwtAuthGuard`, `VerifiedUserGuard`), so unauthenticated exploitation is not possible. However, any registered and verified user — including a malicious organization member — can trigger this with a single crafted HTTP request. The attack requires no special privileges beyond a valid session token.

### Recommendation

Wrap the entire body of the `Transform` callback in `is-signature-map.decorator.ts` in a try/catch that converts any non-`BadRequestException` into a `BadRequestException`:

```typescript
Transform(({ value }) => {
  try {
    // ... existing logic ...
    signatureMap.addSignature(
      AccountId.fromString(nodeAccountId),
      TransactionId.fromString(transactionId),
      PublicKey.fromString(publicKey),   // can throw SDK Error
      decodedSignature,
    );
  } catch (e) {
    if (e instanceof BadRequestException) throw e;
    throw new BadRequestException(ErrorCodes.ISNMP);
  }
  return signatureMap;
}, { toClassOnly: true })
```
