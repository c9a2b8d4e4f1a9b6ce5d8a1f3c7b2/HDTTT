### Title
Unguarded Clipboard Exposure of Full Recovery Phrase and Private Keys Allows Silent Exfiltration by Co-resident Malicious Applications

### Summary
The Hedera Transaction Tool desktop application writes the complete 24-word recovery phrase and decrypted private keys to the system clipboard in plaintext across three separate code paths, with no security warning, no data splitting, and no prompt to clear the clipboard. Any application running on the same machine with clipboard read access — or macOS Universal Clipboard sync — can silently exfiltrate these secrets, leading to full wallet compromise.

### Finding Description

Three independent clipboard write paths expose sensitive key material without any guard:

**Path 1 — Account Setup, Recovery Phrase Copy:**
`handleCopyRecoveryPhrase()` in `Generate.vue` writes the full 24-word mnemonic directly to the system clipboard with a single `navigator.clipboard.writeText()` call and shows only a success toast. [1](#0-0) 

**Path 2 — Migration Summary, Recovery Phrase Copy:**
`copyRecoveryPhrase()` in `Summary.vue` joins all mnemonic words with `, ` and writes the full phrase to clipboard. No warning is shown. [2](#0-1) 

**Path 3 — Settings Keys Tab, Private Key Copy:**
`handleCopy()` in `KeysTab.vue` is a generic clipboard writer called directly with the decrypted private key string. No warning is shown. [3](#0-2) 

It is invoked for the private key copy button here: [4](#0-3) 

In all three cases the full secret is placed into the OS-level clipboard as a single plaintext string. There is no data splitting (as recommended in the HAL-31 remediation), no expiry, and no user warning about clipboard risks.

### Impact Explanation

The clipboard is a shared OS resource readable by any application with the appropriate permission. On macOS, Universal Clipboard automatically syncs clipboard contents to all devices signed into the same iCloud account. A malicious application co-resident on the user's machine (e.g., a trojanized utility, browser extension with clipboard access, or any app granted clipboard permission) can silently read the full mnemonic or private key the moment the user clicks "Copy." Possession of the 24-word recovery phrase gives an attacker the ability to derive all keys and take full control of any associated Hedera accounts, including draining funds. Possession of a raw private key gives equivalent signing authority over the corresponding account.

**Impact: Critical** — direct theft of cryptographic key material enabling complete wallet takeover.

### Likelihood Explanation

The copy buttons are prominently placed in the normal user workflow (account setup, migration summary, and the settings keys tab). Users are expected to click them. On macOS, Universal Clipboard is enabled by default for all iCloud users, meaning the secret automatically propagates to every linked device without any additional attacker action. On any OS, clipboard-monitoring malware or a malicious Electron plugin can read the value synchronously. No privilege escalation or physical access is required — only a co-resident application with clipboard read permission, which is a standard OS capability granted to most desktop apps.

**Likelihood: Medium** — the attack requires a malicious co-resident app or iCloud device compromise, but the user action (clicking Copy) is routine and the clipboard is a well-known exfiltration channel.

### Recommendation

1. **Add a security warning dialog** before any clipboard write of sensitive material, explicitly stating that the clipboard is accessible to other applications and, on macOS, syncs via Universal Clipboard.
2. **Split the mnemonic** into multiple segments (e.g., two 12-word halves) that must be copied separately, preventing a single clipboard read from yielding the full secret — consistent with the HAL-31 remediation applied by InFlux Technologies.
3. **Auto-clear the clipboard** after a short timeout (e.g., 30–60 seconds) using a follow-up `navigator.clipboard.writeText('')` call.
4. **Prefer in-app display** (masked reveal with show/hide toggle) over clipboard copy for private keys, discouraging clipboard use entirely.

### Proof of Concept

1. Open the Hedera Transaction Tool and navigate to **Account Setup → Generate Recovery Phrase**.
2. Generate a 24-word phrase. Click the **Copy** button (`button-copy`).
3. From any co-resident application (e.g., a simple Node.js script using `pbpaste` on macOS, or `xclip -o` on Linux), read the clipboard immediately:
   ```
   pbpaste
   # Output: word1, word2, word3, ..., word24
   ```
4. The full mnemonic is available in plaintext with no delay, no warning having been shown, and no expiry.
5. Repeat via **Settings → Keys Tab**: reveal a private key, click the copy icon (`span-copy-private-key-{index}`), and read clipboard — the raw hex private key string is present.
6. On macOS with Universal Clipboard enabled, the secret is simultaneously available on all linked Apple devices without any further attacker action. [1](#0-0) [2](#0-1) [3](#0-2)
