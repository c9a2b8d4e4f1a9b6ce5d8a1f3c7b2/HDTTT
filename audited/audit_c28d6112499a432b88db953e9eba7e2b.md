### Title
Insufficient Balance Check Bypassed in Organization Mode for HBAR Transfer Transactions

### Summary
In `TransferHbar.vue`, the guard that prevents submission of a transfer transaction when the sender's balance is insufficient is conditionally applied only in local (personal) mode. When a user operates in organization mode (`selectedOrganization !== null`), the `anyTransfersExceedingBalance` check is skipped entirely, allowing the Sign & Submit button to remain enabled. The `preCreateAssert` function also contains no balance validation. As a result, an organization-mode user can submit a transfer transaction that exceeds the sender's HBAR balance, causing the transaction to fail on the Hedera network and consuming the transaction fee.

### Finding Description

**Root cause — `createDisabled` computed property:**

```javascript
// front-end/src/renderer/components/Transaction/Create/TransferHbar/TransferHbar.vue, lines 45-51
const createDisabled = computed(() => {
  return (
    !totalBalance.value.toBigNumber().isEqualTo(0) ||
    totalBalanceAdjustments.value > 10 ||
    totalBalanceAdjustments.value === 0 ||
    (user.selectedOrganization === null && anyTransfersExceedingBalance.value)
    //  ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
    //  Balance guard is ONLY active in local mode
  );
});
```

The condition `(user.selectedOrganization === null && anyTransfersExceedingBalance.value)` means:
- **Local mode** (`selectedOrganization === null`): button is disabled when any sender's debit exceeds their balance. ✓
- **Organization mode** (`selectedOrganization !== null`): the entire sub-expression evaluates to `false` regardless of balance, so the button stays enabled. ✗ [1](#0-0) 

**`preCreateAssert` also has no balance check:**

```javascript
// lines 101-113
const preCreateAssert = () => {
  if (totalBalanceAdjustments.value > 10) { ... }
  if (totalBalanceAdjustments.value === 0) { ... }
  if (!totalBalance.value.toBigNumber().isEqualTo(0)) { ... }
  // No check: if (anyTransfersExceedingBalance.value) { throw ... }
};
``` [2](#0-1) 

The `anyTransfersExceedingBalance` computed correctly detects when a sender's debit exceeds their mirror-node-reported balance, but this result is never enforced in organization mode: [3](#0-2) 

The child component `TransferHbarFormData.vue` does display an `"Insufficient balance for transfer"` error message in the UI, but this is purely cosmetic — it does not block submission in organization mode. [4](#0-3) 

### Impact Explanation

An organization-mode user who enters a sender debit amount exceeding the account's HBAR balance can click Sign & Submit without any blocking guard. The transaction is constructed, signed by all required organization signers, and submitted to the Hedera network. The network rejects it with `INSUFFICIENT_ACCOUNT_BALANCE`. The transaction fee (HBAR) is consumed and not refunded. In a multi-signature organization workflow, this also wastes the signing effort of every co-signer who participated before the on-chain failure.

**Impact category:** Financial loss (wasted HBAR fees) + operational disruption (failed multi-sig workflows).

### Likelihood Explanation

Any authenticated organization-mode user can trigger this with no special privileges. The steps are trivially reachable through the normal Transfer HBAR UI flow. The UI even shows the "Insufficient balance" warning label, confirming the user can see the problem yet still proceed. No attacker capability beyond a valid organization account is required.

### Recommendation

Remove the `user.selectedOrganization === null &&` condition from `createDisabled` so the balance guard applies in both modes:

```javascript
const createDisabled = computed(() => {
  return (
    !totalBalance.value.toBigNumber().isEqualTo(0) ||
    totalBalanceAdjustments.value > 10 ||
    totalBalanceAdjustments.value === 0 ||
    anyTransfersExceedingBalance.value   // enforce in ALL modes
  );
});
```

Additionally, add the same check to `preCreateAssert` as a defense-in-depth guard:

```javascript
if (anyTransfersExceedingBalance.value) {
  throw new Error('Insufficient balance for one or more sender accounts');
}
```

### Proof of Concept

1. Log in to the Hedera Transaction Tool and join an organization (organization mode active, `selectedOrganization !== null`).
2. Navigate to **Transactions → Create → Transfer HBAR**.
3. In the "From" card, enter an account ID whose HBAR balance is, e.g., 5 HBAR. Enter a debit amount of 1000 HBAR and click **Add Transfer**.
4. In the "To" card, enter any valid recipient and add a credit of 1000 HBAR.
5. Observe: the UI shows the "Insufficient balance for transfer" warning label, but the **Sign & Submit** button remains **enabled** (not disabled).
6. Click Sign & Submit. The transaction is constructed and submitted to the Hedera network.
7. The network returns `INSUFFICIENT_ACCOUNT_BALANCE`. The transaction fee is deducted from the payer account. The transfer does not occur.

In local mode, step 5 would result in the button being disabled, preventing submission entirely. [1](#0-0)

const anyTransfersExceedingBalance = computed(() => {
  let result = false;
  for (const transfer of data.transfers) {
    if (transfer.amount.isNegative()) {
      const accountInfo = accountInfos.value[transfer.accountId.toString()];
      if (
        accountInfo &&
        transfer.amount.negated().toBigNumber().isGreaterThan(accountInfo.balance.toBigNumber())
      ) {
        result = true;
        break;
      }
    }
  }
  return result;
});
```
