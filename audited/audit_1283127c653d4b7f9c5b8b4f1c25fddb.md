### Title
Unauthenticated-to-Any-User Access Control Bypass: Missing Authorization on Sensitive Read Endpoints Exposes All Organizational Transaction Data

### Summary
Two endpoints in the API service lack proper per-user authorization checks, allowing any authenticated (but otherwise unprivileged) organization member to enumerate all transaction groups and all historical transactions belonging to other users. The `GET /transaction-groups` endpoint is explicitly marked `/* TESTING ONLY */` in source but is deployed with no ownership or role filter, and `GET /transactions/history` similarly returns the full organization-wide transaction history to any caller.

### Finding Description

**Root Cause 1 — `GET /transaction-groups` (no user scope, labeled TESTING ONLY)**

In `transaction-groups.controller.ts`, the `getTransactionGroups` handler at lines 53–57 carries a `/* TESTING ONLY */` comment, accepts no `@GetUser()` parameter, and delegates directly to `transactionGroupsService.getTransactionGroups()` with no user argument: [1](#0-0) 

The controller class is protected only by `JwtBlackListAuthGuard`, `JwtAuthGuard`, and `VerifiedUserGuard` at the class level: [2](#0-1) 

No `AdminGuard` or ownership predicate is applied to this route. Any verified organization member can call `GET /transaction-groups` and receive every transaction group in the database, regardless of whether they created it or are a party to it.

**Root Cause 2 — `GET /transactions/history` (no user scope)**

In `transactions.controller.ts`, the `getHistoryTransactions` handler at lines 144–154 also accepts no `@GetUser()` parameter and passes no user identity to the service: [3](#0-2) 

Compare this with the adjacent `getTransactions` handler (line 119–132), which does accept `@GetUser() user: User` and passes it to the service for scoping: [4](#0-3) 

The asymmetry is the defect: executed, failed, and expired transactions — which may contain sensitive financial and governance data — are returned to any authenticated caller without ownership filtering.

**Exploit Path:**
1. Attacker registers or is invited as a low-privilege organization member (normal product flow).
2. Attacker authenticates via `POST /auth/login` and obtains a JWT.
3. Attacker calls `GET /transaction-groups` → receives all transaction groups in the organization.
4. Attacker calls `GET /transactions/history` → receives all completed/failed/expired transactions organization-wide.
5. Attacker iterates over returned IDs to correlate parties, amounts, and transaction types across the entire organization.

### Impact Explanation

Any verified organization member can read the complete transaction history and all transaction groups of the entire organization, including transactions they did not create, sign, or observe. Transaction records contain: transaction type (including privileged types such as `SystemDelete`, `NodeCreate`), valid-start timestamps, network identifiers, and group membership. This constitutes unauthorized cross-user/cross-tenant data disclosure of sensitive organizational governance and financial activity. The `/* TESTING ONLY */` comment on the `getTransactionGroups` route confirms the endpoint was never intended for production use, making its presence a clear deployment-time access control failure. [1](#0-0) 

### Likelihood Explanation

Likelihood is **high**. The attacker precondition is only a valid organization account — the normal onboarding path. No admin keys, no leaked credentials, and no privileged network access are required. The endpoints are standard REST GET routes discoverable via the Swagger UI (`/api` path, enabled by `setup-app.ts`). The `/* TESTING ONLY */` label indicates the route was intentionally left in and was never gated before shipping.

### Recommendation

1. **`GET /transaction-groups`**: Either remove the endpoint entirely (it is labeled testing-only) or add `@GetUser() user: User` and pass the user to `getTransactionGroups(user)` so the service can filter to groups the caller created or participates in. Apply `AdminGuard` if the intent is admin-only visibility.

2. **`GET /transactions/history`**: Add `@GetUser() user: User` to the handler signature and pass it to `getHistoryTransactions(user, ...)` so the service scopes results to transactions the caller is a creator, signer, approver, or observer of — matching the pattern already used by `getTransactions`. [4](#0-3) 

### Proof of Concept

```
# Step 1: Login as any low-privilege verified user
POST /auth/login
{ "email": "lowpriv@org.example", "password": "..." }
→ { "accessToken": "eyJ..." }

# Step 2: Dump all transaction groups (no ownership check)
GET /transaction-groups
Authorization: Bearer eyJ...
→ 200 OK — returns ALL transaction groups in the organization

# Step 3: Dump all historical transactions (no ownership check)
GET /transactions/history
Authorization: Bearer eyJ...
→ 200 OK — returns ALL executed/failed/expired transactions organization-wide
```

Expected (correct) behavior: both responses should be scoped to transactions/groups the authenticated user is a party to. Actual behavior: full organization-wide data is returned. [5](#0-4) [3](#0-2)

```typescript
@ApiTags('Transaction Groups')
@Controller('transaction-groups')
@UseGuards(JwtBlackListAuthGuard, JwtAuthGuard, VerifiedUserGuard)
export class TransactionGroupsController {
  constructor(private readonly transactionGroupsService: TransactionGroupsService) {}

  /* Submit a transaction group */
  @ApiOperation({
    summary: 'Create a transaction group',
    description:
      'Create a transaction group for the organization. ' +
      'The group contains group items that each point to a transaction ' +
      'that the organization is to approve, sign, and execute.',
  })
  @ApiResponse({
    status: 201,
    type: TransactionGroupDto,
  })
  @Post()
  @Serialize(TransactionGroupDto)
  createTransactionGroup(
    @GetUser() user: User,
    @Body() dto: CreateTransactionGroupDto,
  ): Promise<TransactionGroup> {
    return this.transactionGroupsService.createTransactionGroup(user, dto);
  }

  /* TESTING ONLY: Get all transactions groups */
  @Get()
  @Serialize(TransactionGroupDto)
  getTransactionGroups(): Promise<TransactionGroup[]> {
    return this.transactionGroupsService.getTransactionGroups();
  }
```
