### Title
Unbounded Recursive Approver Tree Creation Enables Authenticated DoS via Resource Exhaustion

### Summary
The `createTransactionApprovers` endpoint accepts an arbitrarily large and arbitrarily deep nested approver tree with no size or depth constraints. Because the recursive `createApprover` function issues multiple expensive database operations (including recursive CTEs) per node, any authenticated user who owns a transaction can exhaust database connections, CPU, and memory by submitting a single crafted request, rendering the API service unavailable.

### Finding Description

**Root cause — missing array size constraints on the DTO**

`CreateTransactionApproverDto` and `CreateTransactionApproversArrayDto` impose no upper bound on the number of elements: [1](#0-0) 

`approversArray` carries only `@IsArray()` + `@ValidateNested()`. The nested `approvers` field carries `@ArrayMinSize(1)` but no `@ArrayMaxSize()`. An attacker can therefore submit a payload with thousands of root-level entries, each containing thousands of nested children, with no rejection at the validation layer.

**Unbounded recursive processing**

`createTransactionApprovers` iterates over every element of `dto.approversArray` and, for each element, calls the inner async `createApprover` function recursively for every child in `dtoApprover.approvers`: [2](#0-1) 

**Per-node database cost**

For every single node in the tree, `createApprover` executes up to five separate database round-trips:

1. `this.isNode()` — a `COUNT` query against `transaction_approver`
2. `transactionalEntityManager.findOne(TransactionApprover, ...)` — parent lookup
3. `this.getRootNodeFromNode()` — a **recursive CTE** traversing the entire ancestor chain
4. `transactionalEntityManager.count(User, ...)` — user existence check
5. `this.getApproversByTransactionId()` — another **recursive CTE** over the full approver tree, followed by an `INSERT` [3](#0-2) 

All of this runs inside a single long-lived database transaction, holding a connection open for the entire duration.

**Entry point — reachable by any authenticated user**

The controller requires only JWT authentication (`JwtAuthGuard`, `VerifiedUserGuard`). Any registered user can create a transaction (making themselves the creator) and immediately call this endpoint: [4](#0-3) 

`getCreatorsTransaction` only checks that the requesting user is the transaction creator — a condition any user satisfies for their own transaction.

### Impact Explanation

A single malicious POST request containing a deeply nested or very wide approver tree (e.g., 500 root nodes each with 500 children = 250,000 nodes) causes:

- **Database connection exhaustion**: one long-held transaction connection per request; concurrent requests starve the pool.
- **CPU/memory exhaustion on the DB server**: each node triggers two recursive CTEs that re-scan the growing `transaction_approver` table.
- **API service unavailability**: the NestJS process blocks on awaited DB calls; concurrent legitimate requests time out.

The attack is permanent until the server is restarted or the connection pool recovers, and can be repeated continuously.

### Likelihood Explanation

- **Attacker precondition**: a valid account (self-registration is the normal onboarding flow).
- **No privileged access required**: any user can create a transaction and immediately call the approvers endpoint.
- **Single HTTP request**: no sustained effort needed; one crafted JSON body is sufficient.
- **No rate limiting observed** on this endpoint in the reviewed code.

### Recommendation

1. **Add `@ArrayMaxSize(N)` to both DTO fields** — enforce a hard cap (e.g., 50) on `approversArray` and on each nested `approvers` array:

```typescript
// create-transaction-approver.dto.ts
@IsArray()
@ArrayMinSize(1)
@ArrayMaxSize(50)          // add this
@IsOptional()
@ValidateNested({ each: true })
@Type(() => CreateTransactionApproverDto)
approvers?: CreateTransactionApproverDto[];

// CreateTransactionApproversArrayDto
@IsArray()
@ArrayMaxSize(50)          // add this
@ValidateNested({ each: true })
@Type(() => CreateTransactionApproverDto)
approversArray: CreateTransactionApproverDto[];
```

2. **Enforce a maximum nesting depth** inside `createApprover` by passing a depth counter and throwing if it exceeds a threshold (e.g., 5 levels).

3. **Apply API-level rate limiting** to the `POST /transactions/:transactionId/approvers` endpoint.

### Proof of Concept

**Preconditions**: attacker has a valid JWT (`ATTACKER_TOKEN`) and has created transaction with id `TX_ID`.

```bash
# Generate a payload: 200 root approvers, each with 200 children = 40,000 nodes
python3 -c "
import json, sys
child = {'userId': 1}
node  = {'threshold': 1, 'approvers': [child] * 200}
body  = {'approversArray': [node] * 200}
print(json.dumps(body))
" > payload.json

curl -X POST https://<server>/transactions/<TX_ID>/approvers \
  -H 'Authorization: Bearer <ATTACKER_TOKEN>' \
  -H 'Content-Type: application/json' \
  -d @payload.json
```

**Expected outcome**: the server holds a DB transaction open while executing ~200,000 recursive CTE queries. Concurrent legitimate requests to any endpoint that touches the database begin timing out. Repeating the request two or three times in parallel exhausts the connection pool entirely, returning 500 errors to all users.

      await this.dataSource.transaction(async transactionalEntityManager => {
        const createApprover = async (dtoApprover: CreateTransactionApproverDto) => {
          /* Validate Approver's DTO */
          this.validateApprover(dtoApprover);

          /* Check if the approver already exists */
          if (await this.isNode(dtoApprover, transactionId, transactionalEntityManager))
            throw new Error(this.APPROVER_ALREADY_EXISTS);

          /* Check if the parent approver exists and has threshold */
          if (typeof dtoApprover.listId === 'number') {
            const parent = await transactionalEntityManager.findOne(TransactionApprover, {
              where: { id: dtoApprover.listId },
            });

            if (!parent) throw new Error(this.PARENT_APPROVER_NOT_FOUND);

            /* Check if the root transaction is the same */
            const root = await this.getRootNodeFromNode(
              dtoApprover.listId,
              transactionalEntityManager,
            );
            if (root?.transactionId !== transactionId)
              throw new Error(this.ROOT_TRANSACTION_NOT_SAME);
          }

          /* Check if the user exists */
          if (typeof dtoApprover.userId === 'number') {
            const userCount = await transactionalEntityManager.count(User, {
              where: { id: dtoApprover.userId },
            });

            if (userCount === 0) throw new Error(this.USER_NOT_FOUND(dtoApprover.userId));
          }

          /* Check if there are sub approvers */
          if (
            typeof dtoApprover.userId === 'number' &&
            dtoApprover.approvers &&
            dtoApprover.approvers.length > 0
          )
            throw new Error(this.ONLY_USER_OR_TREE);

          /* Check if the approver has threshold when there are children */
          if (
            dtoApprover.approvers &&
            dtoApprover.approvers.length > 0 &&
            (dtoApprover.threshold === null || isNaN(dtoApprover.threshold))
          )
            throw new Error(this.THRESHOLD_REQUIRED);

          /* Check if the approver has children when there is threshold */
          if (
            typeof dtoApprover.threshold === 'number' &&
            (!dtoApprover.approvers || dtoApprover.approvers.length === 0)
          )
            throw new Error(this.CHILDREN_REQUIRED);

          /* Check if the approver threshold is less or equal to the number of approvers */
          if (
            dtoApprover.approvers &&
            (dtoApprover.threshold > dtoApprover.approvers.length || dtoApprover.threshold === 0)
          )
            throw new Error(this.THRESHOLD_LESS_OR_EQUAL_APPROVERS(dtoApprover.approvers.length));

          const data: DeepPartial<TransactionApprover> = {
            transactionId:
              dtoApprover.listId === null || isNaN(dtoApprover.listId) ? transactionId : null,
            listId: dtoApprover.listId,
            threshold:
              dtoApprover.threshold && dtoApprover.approvers ? dtoApprover.threshold : null,
            userId: dtoApprover.userId,
          };

          if (typeof dtoApprover.userId === 'number') {
            const userApproverRecords = await this.getApproversByTransactionId(
              transactionId,
              dtoApprover.userId,
              transactionalEntityManager,
            );

            if (userApproverRecords.length > 0) {
              data.signature = userApproverRecords[0].signature;
              data.userKeyId = userApproverRecords[0].userKeyId;
              data.approved = userApproverRecords[0].approved;
            }
          }

          /* Create approver */
          const approver = transactionalEntityManager.create(TransactionApprover, data);

          /* Insert approver */
          await transactionalEntityManager.insert(TransactionApprover, approver);
          approvers.push(approver);

          /* Continue creating the three */
          if (dtoApprover.approvers) {
            for (const nestedDtoApprover of dtoApprover.approvers) {
              const nestedApprover = { ...nestedDtoApprover, listId: approver.id };

              if (!nestedDtoApprover.approvers || nestedDtoApprover.approvers.length === 0) {
                nestedApprover.threshold = null;
              }

              await createApprover({ ...nestedDtoApprover, listId: approver.id });
            }
          }
        };

        for (const approver of dto.approversArray) {
          await createApprover(approver);
        }
      });

@Controller('transactions/:transactionId?/approvers')

  @ApiOperation({
    summary: 'Creates transaction approvers',
    description: 'Create transaction approvers for the given transaction with the provided data.',
  })
  @ApiResponse({
    status: 201,
    type: [TransactionApproverDto],
  })
  @Post()
  createTransactionApprovers(
    @GetUser() user: User,
    @Param('transactionId', ParseIntPipe) transactionId: number,
    @Body() body: CreateTransactionApproversArrayDto,
  ): Promise<TransactionApprover[]> {
    return this.approversService.createTransactionApprovers(user, transactionId, body);
