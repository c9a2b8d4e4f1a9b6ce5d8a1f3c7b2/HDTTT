### Title
Missing Cryptographic Signature Verification in `processTransactionSignatures` Allows Authenticated User to Corrupt Multi-Sig Transaction Bytes

### Summary
The `processTransactionSignatures` method in `signers.service.ts` accepts and stores signature bytes without cryptographically verifying them against the transaction body. An authenticated user can upload an invalid (non-verifiable) signature for their own registered public key, causing the transaction to be recorded as signed and potentially moved to execution, where it will fail on the Hedera network — wasting co-signers' legitimate signatures and potentially expiring the transaction window.

### Finding Description

**Root cause:** `processTransactionSignatures` in `back-end/apps/api/src/transactions/signers/signers.service.ts` (lines 217–266) only checks that the public key in the submitted `SignatureMap` belongs to the authenticated user via a `userKeyMap` lookup, then unconditionally calls `sdkTransaction.addSignature(publicKey, map)` — never calling `publicKey.verify(bodyBytes, signature)` to confirm the signature bytes are cryptographically valid. [1](#0-0) 

Contrast this with `importSignatures` in `transactions.service.ts`, which correctly calls `validateSignature(sdkTransaction, map)` — which internally calls `publicKey.verify(bodyBytes, signature)` — before accepting any signature: [2](#0-1) 

The `validateSignature` utility explicitly verifies each signature cryptographically: [3](#0-2) 

The two code paths diverge at the controller level: `POST /transactions/:id/signers` routes through `signers.service.ts` (no verification), while the `importSignatures` path routes through `transactions.service.ts` (with verification):

export const validateSignature = (transaction: SDKTransaction, signatureMap: SignatureMap) => {
  const signerPublicKeys: PublicKey[] = [];

  const { rowLength, nodeAccountIdRow, transactionIdCol } =
    getSignedTransactionsDimensions(transaction);

  for (const [nodeAccountId, transactionIds] of signatureMap._map) {
    for (const [transactionId, publicKeys] of transactionIds._map) {
      for (const [publicKeyDer, signature] of publicKeys._map) {
        const publicKey = PublicKey.fromString(publicKeyDer);
        const publicKeyHex = publicKey.toStringRaw();

        const alreadySigned =
          transaction._signerPublicKeys.has(publicKeyHex) ||
          transaction._signerPublicKeys.has(publicKeyDer);

        if (!alreadySigned) {
          const row = nodeAccountIdRow[nodeAccountId];
          const col = transactionIdCol[transactionId];

          const bodyBytes = transaction._signedTransactions.get(col * rowLength + row).bodyBytes;

          const signatureValid = publicKey.verify(bodyBytes, signature);

          if (signatureValid) {
            signerPublicKeys.push(publicKey);
          } else {
            throw new Error('Invalid signature');
          }
        }
      }
    }
  }

  return signerPublicKeys;
};
