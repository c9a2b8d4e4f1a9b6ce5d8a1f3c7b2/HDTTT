### Title
Unbounded Recursive Approver Tree in `createTransactionApprovers` Enables Single-Request Server-Side DoS

### Summary
The `POST /transactions/:transactionId/approvers` endpoint accepts an arbitrarily large, arbitrarily deep nested approver tree with no size or depth limit. The service processes each node recursively, executing 4–6 synchronous database queries per node inside a single database transaction. A verified user who owns any transaction can craft a single request with a massive nested tree to exhaust the database connection pool and cause a sustained denial of service for all other users.

### Finding Description

**Root cause — no array size bound on the DTO:**

`CreateTransactionApproversArrayDto.approversArray` and the nested `CreateTransactionApproverDto.approvers` field carry no `@ArrayMaxSize()` constraint. [1](#0-0) 

Both arrays are validated only for element type and minimum size (`@ArrayMinSize(1)` on the nested field), never for maximum size.

**Root cause — unbounded recursive processing with per-node DB queries:**

`createTransactionApprovers` iterates `dto.approversArray` and, for every element, calls the inner async `createApprover`. That function recursively calls itself for every element of `dtoApprover.approvers`: [2](#0-1) 

Each recursive invocation of `createApprover` issues the following sequential database operations before recursing:
1. `this.isNode()` → `COUNT` query
2. `transactionalEntityManager.findOne(TransactionApprover, ...)` → `SELECT`
3. `this.getRootNodeFromNode()` → recursive CTE `SELECT`
4. `transactionalEntityManager.count(User, ...)` → `COUNT`
5. `this.getApproversByTransactionId()` → recursive CTE `SELECT`
6. `transactionalEntityManager.insert(TransactionApprover, ...)` → `INSERT` [3](#0-2) 

All of this runs inside a single `dataSource.transaction()` call, holding a database connection open for the entire duration.

**Entry point — reachable by any verified user:**

The controller requires only `JwtBlackListAuthGuard`, `JwtAuthGuard`, and `VerifiedUserGuard`. No throttler guard is applied to this specific endpoint beyond the global user-level rate limiter (100 req/min, 10 req/sec). [4](#0-3) 

Any verified user can create a transaction (making themselves the creator) and immediately call this endpoint.

**Exploit path:**

1. Attacker registers and verifies an account (normal user flow).
2. Attacker creates a transaction — they become its creator.
3. Attacker sends one `POST /transactions/:id/approvers` with a payload like:
```json
{
  "approversArray": [
    {
      "threshold": 1,
      "approvers": [
        { "threshold": 1, "approvers": [
            { "threshold": 1, "approvers": [
                ... (hundreds of levels deep, each with many children)
            ]}
        ]}
      ]
    }
  ]
}
```
4. The server enters unbounded recursion, issuing thousands of sequential DB queries inside one held transaction.
5. The DB connection pool is exhausted; all other requests queue or fail.

### Impact Explanation

A single crafted HTTP request can hold a database connection open for an extended period while executing thousands of sequential queries. With a sufficiently large tree (e.g., 10 levels × 10 children = 10^10 nodes in the worst case, or even a flat array of thousands of entries), the server's database connection pool is exhausted, causing all concurrent requests from all users to fail or time out. This is a complete, sustained denial of service of the API service triggered by one authenticated request.

### Likelihood Explanation

The attacker precondition is minimal: a valid, verified account. Creating a transaction and calling the approvers endpoint are both normal, documented product flows. No privileged role, no leaked credentials, and no special knowledge beyond the public API schema are required. The crafted payload is trivially constructed from the documented DTO structure.

### Recommendation

1. **Add `@ArrayMaxSize(N)` to both DTO fields** — apply a reasonable cap (e.g., 50) to `approversArray` in `CreateTransactionApproversArrayDto` and to `approvers` in `CreateTransactionApproverDto`.
2. **Add a maximum recursion depth check** — track depth inside `createApprover` and throw if it exceeds a configured limit (e.g., 5 levels).
3. **Apply the `UserThrottlerGuard` to the approvers controller** — this limits the rate at which a single user can trigger the expensive path.

### Proof of Concept

**Preconditions:** Valid JWT for a verified user; a transaction ID owned by that user.

**Request:**
```
POST /transactions/1/approvers
Authorization: Bearer <valid_token>
Content-Type: application/json

{
  "approversArray": [
    { "threshold": 1, "approvers": [
      { "threshold": 1, "approvers": [
        { "threshold": 1, "approvers": [
          { "threshold": 1, "approvers": [
            { "threshold": 1, "approvers": [
              { "userId": 1 }, { "userId": 2 }, { "userId": 3 },
              { "userId": 4 }, { "userId": 5 }, { "userId": 6 },
              { "userId": 7 }, { "userId": 8 }, { "userId": 9 }
            ]}
          ]}
        ]}
      ]}
    ]}
  ]
}
```

**Expected outcome:** The server enters deep recursion, issuing hundreds of sequential DB queries inside a held transaction. Repeating this request a few times (within the 100 req/min rate limit) exhausts the DB connection pool, causing `503`/timeout responses for all concurrent users.

        const createApprover = async (dtoApprover: CreateTransactionApproverDto) => {
          /* Validate Approver's DTO */
          this.validateApprover(dtoApprover);

          /* Check if the approver already exists */
          if (await this.isNode(dtoApprover, transactionId, transactionalEntityManager))
            throw new Error(this.APPROVER_ALREADY_EXISTS);

          /* Check if the parent approver exists and has threshold */
          if (typeof dtoApprover.listId === 'number') {
            const parent = await transactionalEntityManager.findOne(TransactionApprover, {
              where: { id: dtoApprover.listId },
            });

            if (!parent) throw new Error(this.PARENT_APPROVER_NOT_FOUND);

            /* Check if the root transaction is the same */
            const root = await this.getRootNodeFromNode(
              dtoApprover.listId,
              transactionalEntityManager,
            );
            if (root?.transactionId !== transactionId)
              throw new Error(this.ROOT_TRANSACTION_NOT_SAME);
          }

          /* Check if the user exists */
          if (typeof dtoApprover.userId === 'number') {
            const userCount = await transactionalEntityManager.count(User, {
              where: { id: dtoApprover.userId },
            });

            if (userCount === 0) throw new Error(this.USER_NOT_FOUND(dtoApprover.userId));
          }

          /* Check if there are sub approvers */
          if (
            typeof dtoApprover.userId === 'number' &&
            dtoApprover.approvers &&
            dtoApprover.approvers.length > 0
          )
            throw new Error(this.ONLY_USER_OR_TREE);

          /* Check if the approver has threshold when there are children */
          if (
            dtoApprover.approvers &&
            dtoApprover.approvers.length > 0 &&
            (dtoApprover.threshold === null || isNaN(dtoApprover.threshold))
          )
            throw new Error(this.THRESHOLD_REQUIRED);

          /* Check if the approver has children when there is threshold */
          if (
            typeof dtoApprover.threshold === 'number' &&
            (!dtoApprover.approvers || dtoApprover.approvers.length === 0)
          )
            throw new Error(this.CHILDREN_REQUIRED);

          /* Check if the approver threshold is less or equal to the number of approvers */
          if (
            dtoApprover.approvers &&
            (dtoApprover.threshold > dtoApprover.approvers.length || dtoApprover.threshold === 0)
          )
            throw new Error(this.THRESHOLD_LESS_OR_EQUAL_APPROVERS(dtoApprover.approvers.length));

          const data: DeepPartial<TransactionApprover> = {
            transactionId:
              dtoApprover.listId === null || isNaN(dtoApprover.listId) ? transactionId : null,
            listId: dtoApprover.listId,
            threshold:
              dtoApprover.threshold && dtoApprover.approvers ? dtoApprover.threshold : null,
            userId: dtoApprover.userId,
          };

          if (typeof dtoApprover.userId === 'number') {
            const userApproverRecords = await this.getApproversByTransactionId(
              transactionId,
              dtoApprover.userId,
              transactionalEntityManager,
            );

            if (userApproverRecords.length > 0) {
              data.signature = userApproverRecords[0].signature;
              data.userKeyId = userApproverRecords[0].userKeyId;
              data.approved = userApproverRecords[0].approved;
            }
          }

          /* Create approver */
          const approver = transactionalEntityManager.create(TransactionApprover, data);

          /* Insert approver */
          await transactionalEntityManager.insert(TransactionApprover, approver);
          approvers.push(approver);

          /* Continue creating the three */
          if (dtoApprover.approvers) {
            for (const nestedDtoApprover of dtoApprover.approvers) {
              const nestedApprover = { ...nestedDtoApprover, listId: approver.id };

              if (!nestedDtoApprover.approvers || nestedDtoApprover.approvers.length === 0) {
                nestedApprover.threshold = null;
              }

              await createApprover({ ...nestedDtoApprover, listId: approver.id });
            }
          }
        };

        for (const approver of dto.approversArray) {
          await createApprover(approver);
        }
      });
