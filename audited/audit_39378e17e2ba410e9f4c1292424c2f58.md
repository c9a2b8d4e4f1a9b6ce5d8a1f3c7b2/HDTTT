### Title
`approveTransaction` never updates in-memory `userApprovers.approved` before the notification-type check, causing `emitTransactionStatusUpdate` to never fire on approval

### Summary

In `approvers.service.ts`, `approveTransaction` persists the approval to the database but then evaluates `userApprovers.every(a => a.approved)` against the **stale in-memory array** (fetched before the write). Because the in-memory objects are never updated, this condition is always `false` when `dto.approved` is `true`, so `emitTransactionStatusUpdate` — the event that triggers downstream status recalculation — is never emitted on an approval. Only `emitTransactionUpdate` (a lightweight notification) is sent. This is the direct analog of the QVSimpleStrategy bug: a state value is checked but never updated, causing a branch to always take the wrong path.

### Finding Description

`approveTransaction` in `back-end/apps/api/src/transactions/approvers/approvers.service.ts` follows this sequence:

1. Fetch `userApprovers` from the database (line 553–556).
2. Guard against double-approval using `userApprovers.every(a => a.signature)` (line 563) — this check is correct and uses the DB-persisted `signature` field.
3. Persist the approval to the database, setting `approved: dto.approved` for every record in `userApprovers` (lines 599–610).
4. **Decide which notification to emit** (lines 614–618):

```typescript
if (!dto.approved || userApprovers.every(a => a.approved)) {
  emitTransactionStatusUpdate(this.notificationsPublisher, notificationEvent);
} else {
  emitTransactionUpdate(this.notificationsPublisher, notificationEvent);
}
```

The problem is at step 4. The DB write at step 3 sets `approved = true` in the database, but the **in-memory `userApprovers` objects are never mutated**. Their `.approved` fields still hold the pre-write values (`null` or `false`). Therefore `userApprovers.every(a => a.approved)` is always `false` when `dto.approved` is `true`, and the `else` branch (`emitTransactionUpdate`) is always taken.

`emitTransactionStatusUpdate` is only reached when the user **rejects** (`!dto.approved === true`). It is never reached when the user approves. [1](#0-0) 

### Impact Explanation

`emitTransactionStatusUpdate` is the event that signals the chain/notification service to re-evaluate the transaction's overall status (e.g., to determine whether all required approvals have been collected and the transaction should advance from `WAITING_FOR_SIGNATURES` to `WAITING_FOR_EXECUTION`). Because this event is never emitted on approval, a transaction can collect every required approval and still remain permanently stuck in `WAITING_FOR_SIGNATURES`. The approval workflow is effectively broken for the approval path: approvals are recorded in the database but the system never acts on them. [2](#0-1) 

### Likelihood Explanation

This code path is exercised every time any approver calls `approveTransaction` with `dto.approved = true`. There are no conditions that could accidentally make `userApprovers.every(a => a.approved)` return `true` before the in-memory objects are updated (the pre-write values are `null` or `false`). The bug is deterministic and affects every approval action. [3](#0-2) 

### Recommendation

After the database write, update the in-memory `userApprovers` to reflect the persisted state before evaluating the notification condition:

```typescript
await this.dataSource.transaction(async transactionalEntityManager => {
  await transactionalEntityManager
    .createQueryBuilder()
    .update(TransactionApprover)
    .set({ userKeyId: dto.userKeyId, signature: dto.signature, approved: dto.approved })
    .whereInIds(userApprovers.map(a => a.id))
    .execute();
});

// ✅ Sync in-memory state with what was just written to the DB
userApprovers.forEach(a => { a.approved = dto.approved; });

if (!dto.approved || userApprovers.every(a => a.approved)) {
  emitTransactionStatusUpdate(this.notificationsPublisher, notificationEvent);
} else {
  emitTransactionUpdate(this.notificationsPublisher, notificationEvent);
}
```

### Proof of Concept

1. Creator creates a transaction requiring approval from User A.
2. User A calls `POST /transactions/:id/approvers` to register as approver.
3. User A calls `approveTransaction` with `dto.approved = true`.
4. The DB is updated: `approved = true`, `signature = <valid>`.
5. In memory, `userApprovers[0].approved` is still `null`.
6. `userApprovers.every(a => a.approved)` → `false`.
7. `emitTransactionUpdate` is called; `emitTransactionStatusUpdate` is **not** called.
8. The chain service never receives the status-update event and never recalculates the transaction status.
9. The transaction remains in `WAITING_FOR_SIGNATURES` indefinitely, even though all required approvals are present in the database. [4](#0-3) [5](#0-4)

  /* Approves a transaction */
  async approveTransaction(
    dto: ApproverChoiceDto,
    transactionId: number,
    user: User,
  ): Promise<boolean> {
    /* Get all the approvers */
    const approvers = await this.getVerifiedApproversByTransactionId(transactionId, user);

    /* If user is approver, filter the records that belongs to the user */
    const userApprovers = approvers.filter(a => a.userId === user.id);

    /* Check if the user is an approver */
    if (userApprovers.length === 0)
      throw new UnauthorizedException('You are not an approver of this transaction');

    /* Check if the user has already approved the transaction */
    if (userApprovers.every(a => a.signature)) throw new BadRequestException(ErrorCodes.TAP);

    /* Ensures the user keys are passed */
    await attachKeys(user, this.dataSource.manager);
    if (user.keys.length === 0) return false;

    const signatureKey = user.keys.find(key => key.id === dto.userKeyId);

    /* Gets the public key that the signature belongs to */
    const publicKey = PublicKey.fromString(signatureKey?.publicKey);

    /* Get the transaction body */
    const transaction = await this.dataSource.manager.findOne(Transaction, {
      where: { id: transactionId },
      relations: { creatorKey: true, observers: true },
    });

    /* Check if the transaction exists */
    if (!transaction) throw new BadRequestException(ErrorCodes.TNF);

    /* Checks if the transaction is requires approval */
    if (
      transaction.status !== TransactionStatus.WAITING_FOR_SIGNATURES &&
      transaction.status !== TransactionStatus.WAITING_FOR_EXECUTION
    )
      throw new BadRequestException(ErrorCodes.TNRA);

    const sdkTransaction = SDKTransaction.fromBytes(transaction.transactionBytes);

    /* Verify the signature matches the transaction */
    if (
      !verifyTransactionBodyWithoutNodeAccountIdSignature(sdkTransaction, dto.signature, publicKey)
    )
      throw new BadRequestException(ErrorCodes.SNMP);

    /* Update the approver with the signature */
    await this.dataSource.transaction(async transactionalEntityManager => {
      await transactionalEntityManager
        .createQueryBuilder()
        .update(TransactionApprover)
        .set({
          userKeyId: dto.userKeyId,
          signature: dto.signature,
          approved: dto.approved,
        })
        .whereInIds(userApprovers.map(a => a.id))
        .execute();
    });

    const notificationEvent = [{ entityId: transaction.id }];

    if (!dto.approved || userApprovers.every(a => a.approved)) {
      emitTransactionStatusUpdate(this.notificationsPublisher, notificationEvent);
    } else {
      emitTransactionUpdate(this.notificationsPublisher, notificationEvent);
    }

    return true;
  }
