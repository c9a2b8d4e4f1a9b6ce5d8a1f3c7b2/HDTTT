### Title
`Math.random()` Used to Generate Temporary Passwords, Enabling Predictable Credential Generation

### Summary
The backend API uses `Math.random()` — a non-cryptographically-secure PRNG — to generate temporary passwords for newly registered users. Because `Math.random()` in V8 (Node.js) uses the deterministic xorshift128+ algorithm, an attacker who can observe or infer the PRNG state can predict future temporary passwords and take over newly registered accounts before users change their credentials.

### Finding Description
In `back-end/apps/api/src/auth/auth.service.ts`, the `generatePassword()` method constructs a temporary password using `Math.random()`: [1](#0-0) 

```typescript
private generatePassword() {
  const getRandomLetters = (length: number) =>
    Array.from({ length }, () => String.fromCharCode(97 + Math.floor(Math.random() * 26))).join('');
  return `${getRandomLetters(5)}-${getRandomLetters(5)}`;
}
```

This password is then emailed to the user as their initial credential: [2](#0-1) 

The output space is only 26^10 ≈ 1.4×10^14 combinations of lowercase letters, but the effective entropy is far lower because `Math.random()` in V8 uses xorshift128+ — a 128-bit internal state that is **not seeded from a CSPRNG** and whose outputs are predictable once the state is recovered. Tools like `v8-randomness-predictor` can recover the full PRNG state from as few as 3 consecutive `Math.random()` outputs from the same process, allowing all future outputs to be predicted.

Attack path:
1. Attacker triggers any endpoint or side-channel that leaks `Math.random()` outputs from the same Node.js process (e.g., any feature using `Math.random()` in the same service — `cache-management.service.ts`, `email.service.ts`, `mirror-node.client.ts` all call `Math.random()`).
2. Attacker recovers the xorshift128+ state.
3. Admin registers a new user (or attacker triggers re-registration of an existing `UserStatus.NEW` user, which resets the password per lines 57–59).
4. Attacker predicts the 10-character temporary password before the victim changes it.
5. Attacker logs in as the victim and gains full account access. [3](#0-2) 

### Impact Explanation
A successful attack grants the attacker full authenticated access to a victim's account. Since the tool manages Hedera multi-signature transaction workflows, this allows the attacker to approve, sign, or manipulate transactions on behalf of the compromised user. Impact is **high** — unauthorized account takeover with direct effect on transaction integrity.

### Likelihood Explanation
The attack requires the attacker to observe multiple `Math.random()` outputs from the same process. The same API service calls `Math.random()` in at least three other locations (`cache-management.service.ts`, `email.service.ts`, `mirror-node.client.ts`), providing observable outputs. State recovery from V8's xorshift128+ is a well-documented, publicly available technique. The window of exploitation is the time between account creation and the user's first password change — a realistic window for a targeted attack. Likelihood is **medium**.

### Recommendation
Replace `Math.random()` with `crypto.randomBytes()` from Node.js's built-in `crypto` module, which is already imported and used elsewhere in the codebase (e.g., `front-end/src/main/utils/crypto.ts`):

```typescript
import { randomBytes } from 'crypto';

private generatePassword(): string {
  const getRandomLetters = (length: number) =>
    Array.from({ length }, () =>
      String.fromCharCode(97 + (randomBytes(1)[0] % 26))
    ).join('');
  return `${getRandomLetters(5)}-${getRandomLetters(5)}`;
}
```

Additionally, consider increasing password entropy (mixed case, digits, symbols) and enforcing a short expiry on temporary passwords.

### Proof of Concept
1. Register a new user via the admin endpoint, triggering `generatePassword()`.
2. From a separate request to any endpoint that internally calls `Math.random()` in the same process, collect 3+ floating-point outputs.
3. Use a V8 xorshift128+ state-recovery tool (e.g., `https://github.com/nicowillis/v8-randomness-predictor`) to reconstruct the PRNG state.
4. Replay the sequence of `Math.random()` calls that `generatePassword()` would make (10 calls: `Math.random()` × 10 for the two 5-letter groups).
5. Reconstruct the temporary password: `String.fromCharCode(97 + Math.floor(predicted_value * 26))` for each call.
6. Use the predicted password to authenticate as the newly registered user before they log in and change it.
