### Title
Overly Permissive CORS Configuration Violates Principle of Least Privilege

### Summary
The API service and WebSocket gateway both configure CORS with `origin: true` and `credentials: true`. In NestJS/Express, `origin: true` means "reflect the request `Origin` header verbatim," which is functionally equivalent to `Access-Control-Allow-Origin: *` but with credentials support enabled. This grants every origin on the internet credentialed cross-origin access to an API that manages Hedera private keys, transaction signing, and authentication — a direct analog to the external report's finding of superfluous permissions that unnecessarily expand the attack surface.
