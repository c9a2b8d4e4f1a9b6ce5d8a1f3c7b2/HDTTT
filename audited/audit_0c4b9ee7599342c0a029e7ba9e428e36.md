### Title
`approveTransaction` Inverted Conditional Causes Approvals to Never Emit Status-Update Notifications

### Summary

In `approvers.service.ts`, the `approveTransaction` function decides which notification event to emit based on a condition that uses stale pre-update data. Because `userApprovers.every(a => a.approved)` is provably always `false` at that point (due to an earlier guard), the condition collapses to `if (!dto.approved)`, meaning approvals **always** emit only a lightweight `emitTransactionUpdate` and **never** emit `emitTransactionStatusUpdate`. This is the direct analog of the external report's inverted ternary: the wrong branch fires for the wrong input.

### Finding Description

In `back-end/apps/api/src/transactions/approvers/approvers.service.ts`, the `approveTransaction` function:

1. Fetches `userApprovers` (stale, pre-update snapshot) at line 556.
2. Guards at line 563: throws `TAP` if `userApprovers.every(a => a.signature)` — i.e., if **all** approvers already have a signature.
3. Writes the new `{ signature, approved }` to the DB at lines 599–610.
4. Then at line 614 evaluates:

```typescript
if (!dto.approved || userApprovers.every(a => a.approved)) {
  emitTransactionStatusUpdate(this.notificationsPublisher, notificationEvent);
} else {
  emitTransactionUpdate(this.notificationsPublisher, notificationEvent);
}
``` [1](#0-0) [2](#0-1) 

**Root cause — dead second operand:**

`signature` and `approved` are always written together (line 603–607). Therefore, if `userApprovers.every(a => a.approved)` were `true`, then `userApprovers.every(a => a.signature)` would also be `true`, and the function would have already thrown at line 563. The second operand of the `||` is **unreachable dead code** — it can never be `true` when execution reaches line 614.

The condition therefore always reduces to:

```
if (!dto.approved)   →  emitTransactionStatusUpdate   // rejection only
else                 →  emitTransactionUpdate          // approval always
```

This is the inverted-conditional pattern: the heavier, correct event (`emitTransactionStatusUpdate`) fires only for rejections, while approvals always fire the lightweight `emitTransactionUpdate`. [3](#0-2) 

### Impact Explanation

`emitTransactionStatusUpdate` triggers `processTransactionStatusUpdateNotifications`, which:
- Deletes stale in-app indicator notifications and creates new ones
- Sends email notifications (e.g., `TRANSACTION_READY_FOR_EXECUTION`, `TRANSACTION_WAITING_FOR_SIGNATURES`)
- Pushes a `STATUS_UPDATE` WebSocket event to all affected clients [4](#0-3) 

`emitTransactionUpdate` triggers only `processTransactionUpdateNotifications`, which sends a lightweight `UPDATE` WebSocket ping with no email, no in-app notification creation, and no status recalculation. [5](#0-4) 

Concrete consequences when a user approves a transaction:
- **No email** is sent to signers, observers, or the creator indicating the approval occurred.
- **No in-app indicator** is created or updated for any party.
- The transaction's status-driven notification pipeline is silently skipped for every approval action.
- In an organization workflow requiring multiple approvals before execution, no participant is notified that the approval threshold has been met.

### Likelihood Explanation

This fires on **every approval** (`dto.approved = true`) submitted through the normal API flow. Any authenticated user who is a designated approver and submits an approval triggers the bug. No special privileges or race conditions are required — it is deterministic and 100% reproducible.

### Recommendation

Replace the stale-data second operand with a post-update re-fetch, or simply always emit `emitTransactionStatusUpdate` on any approval/rejection action, since both state changes are meaningful:

```typescript
// Always emit a full status update after any approval decision
emitTransactionStatusUpdate(this.notificationsPublisher, notificationEvent);
```

If the distinction between "status changed" and "just updated" is important, re-fetch the approvers from the DB after the write and evaluate the actual current state:

```diff
- if (!dto.approved || userApprovers.every(a => a.approved)) {
+ if (!dto.approved || dto.approved) {   // i.e., always
    emitTransactionStatusUpdate(this.notificationsPublisher, notificationEvent);
- } else {
-   emitTransactionUpdate(this.notificationsPublisher, notificationEvent);
  }
```

### Proof of Concept

1. Create a transaction in Organization Mode that requires at least one approver.
2. As the designated approver, call `POST /transactions/:id/approvers/approve` with `{ approved: true, ... }`.
3. Observe that `emitTransactionStatusUpdate` is **not** called (only `emitTransactionUpdate` is).
4. Confirm that no email notification is dispatched and no in-app indicator is created/updated for the creator, observers, or signers.
5. Repeat with `{ approved: false }` (rejection) and observe that `emitTransactionStatusUpdate` **is** called — confirming the branch inversion. [6](#0-5)

  /* Approves a transaction */
  async approveTransaction(
    dto: ApproverChoiceDto,
    transactionId: number,
    user: User,
  ): Promise<boolean> {
    /* Get all the approvers */
    const approvers = await this.getVerifiedApproversByTransactionId(transactionId, user);

    /* If user is approver, filter the records that belongs to the user */
    const userApprovers = approvers.filter(a => a.userId === user.id);

    /* Check if the user is an approver */
    if (userApprovers.length === 0)
      throw new UnauthorizedException('You are not an approver of this transaction');

    /* Check if the user has already approved the transaction */
    if (userApprovers.every(a => a.signature)) throw new BadRequestException(ErrorCodes.TAP);

    /* Ensures the user keys are passed */
    await attachKeys(user, this.dataSource.manager);
    if (user.keys.length === 0) return false;

    const signatureKey = user.keys.find(key => key.id === dto.userKeyId);

    /* Gets the public key that the signature belongs to */
    const publicKey = PublicKey.fromString(signatureKey?.publicKey);

    /* Get the transaction body */
    const transaction = await this.dataSource.manager.findOne(Transaction, {
      where: { id: transactionId },
      relations: { creatorKey: true, observers: true },
    });

    /* Check if the transaction exists */
    if (!transaction) throw new BadRequestException(ErrorCodes.TNF);

    /* Checks if the transaction is requires approval */
    if (
      transaction.status !== TransactionStatus.WAITING_FOR_SIGNATURES &&
      transaction.status !== TransactionStatus.WAITING_FOR_EXECUTION
    )
      throw new BadRequestException(ErrorCodes.TNRA);

    const sdkTransaction = SDKTransaction.fromBytes(transaction.transactionBytes);

    /* Verify the signature matches the transaction */
    if (
      !verifyTransactionBodyWithoutNodeAccountIdSignature(sdkTransaction, dto.signature, publicKey)
    )
      throw new BadRequestException(ErrorCodes.SNMP);

    /* Update the approver with the signature */
    await this.dataSource.transaction(async transactionalEntityManager => {
      await transactionalEntityManager
        .createQueryBuilder()
        .update(TransactionApprover)
        .set({
          userKeyId: dto.userKeyId,
          signature: dto.signature,
          approved: dto.approved,
        })
        .whereInIds(userApprovers.map(a => a.id))
        .execute();
    });

    const notificationEvent = [{ entityId: transaction.id }];

    if (!dto.approved || userApprovers.every(a => a.approved)) {
      emitTransactionStatusUpdate(this.notificationsPublisher, notificationEvent);
    } else {
      emitTransactionUpdate(this.notificationsPublisher, notificationEvent);
    }

    return true;
  }

  async processTransactionStatusUpdateNotifications(events: NotificationEventDto[]) {
    const ctx = await this.prepareEventContext(events, true);
    if (!ctx) return;

    const {
      cache,
      keyCache,
      transactionMap,
      approversMap,
      deletionNotifications,
      inAppNotifications,
      emailNotifications,
      inAppReceiverIds,
      emailReceiverIds,
      affectedUsers,
    } = ctx;

    // Process each event
    for (const { entityId: transactionId } of events) {
      const transaction = transactionMap.get(transactionId);
      if (!transaction) {
        console.warn(`Transaction ${transactionId} not found, skipping status-update notifications`);
        continue;
      }
      const approvers = approversMap.get(transactionId) || [];

      if (transaction.deletedAt && transaction.status !== TransactionStatus.CANCELED) {
        console.error(
          `Soft-deleted transaction ${transactionId} has unexpected status: ${transaction.status} (expected CANCELED)`
        );
        transaction.status = TransactionStatus.CANCELED;
      }

      const syncType = this.getInAppNotificationType(transaction.status);
      const emailType = this.getEmailNotificationType(transaction.status);

      // Single transaction for both notification types
      await this.entityManager.transaction(async entityManager => {
        await this.handleTransactionStatusUpdateNotifications(
          entityManager,
          transaction,
          approvers,
          syncType,
          emailType,
          cache,
          keyCache,
          deletionNotifications,
          inAppNotifications,
          inAppReceiverIds,
          emailNotifications,
          emailReceiverIds,
          affectedUsers,
          transactionId,
        );
      });
    }

    // Send all notifications in batch
    await this.sendDeletionNotifications(deletionNotifications);
    await this.sendInAppNotifications(inAppNotifications, inAppReceiverIds);
    await this.sendEmailNotifications(emailNotifications, emailReceiverIds);
    await this.sendNotifyClients(affectedUsers, TRANSACTION_EVENT_TYPE.STATUS_UPDATE);
  }

  async processTransactionUpdateNotifications(events: NotificationEventDto[]) {
    const ctx = await this.prepareEventContext(events);
    if (!ctx) return;

    const {
      keyCache,
      transactionMap,
      approversMap,
      affectedUsers,
    } = ctx;

    // Process each event
    for (const { entityId: transactionId } of events) {
      const transaction = transactionMap.get(transactionId);
      if (!transaction) continue;
      const approvers = approversMap.get(transactionId) || [];

      const syncType = this.getInAppNotificationType(transaction.status);

      if (syncType) {
        const receiverIds = await this.getNotificationReceiverIds(this.entityManager, transaction, syncType, approvers, keyCache);
        const groupId = transaction.groupItem?.groupId;
        receiverIds.forEach(id => {
          this.addAffectedUser(affectedUsers, id, transactionId, groupId);
        });
      }
    }

    await this.sendNotifyClients(affectedUsers, TRANSACTION_EVENT_TYPE.UPDATE);
