### Title
HTML Injection via `v-html` Rendering of Untrusted `groupItem.description` and `transactionMemo` in Transaction Group View

### Summary
In `front-end/src/renderer/pages/CreateTransactionGroup/CreateTransactionGroup.vue`, user-controlled fields `groupItem.description` and `transaction.transactionMemo` are rendered directly into the DOM using Vue's `v-html` directive without any HTML sanitization. In organization mode, a malicious member can craft a transaction with a description or memo containing arbitrary HTML (or JavaScript), which executes in the Electron renderer of any other member who opens the Create Transaction Group page and views that transaction's row.

### Finding Description
The vulnerable binding is at lines 421–429 of `CreateTransactionGroup.vue`:

```html
<div
  ...
  v-html="
    groupItem.type == 'Transfer Transaction'
      ? makeTransfer(index)
      : groupItem.description != ''
        ? groupItem.description
        : Transaction.fromBytes(groupItem.transactionBytes).transactionMemo
          ? Transaction.fromBytes(groupItem.transactionBytes).transactionMemo
          : createTransactionId(groupItem.payerAccountId, groupItem.validStart)
  "
></div>
``` [1](#0-0) 

Vue's `v-html` directive sets `innerHTML` directly, bypassing Vue's automatic HTML escaping that `{{ }}` mustache interpolation provides. Both branches that reach user-controlled data are unescaped:

1. **`groupItem.description`** — set by any organization member when creating a transaction via `TransactionInfoControls.vue` (up to 256 characters, no HTML encoding applied). [2](#0-1) 

2. **`Transaction.fromBytes(groupItem.transactionBytes).transactionMemo`** — the transaction memo embedded in the serialized transaction bytes, set by any member creating a transaction via `BaseTransaction.vue` (up to 100 characters, validated only for length). [3](#0-2) 

The description is stored in the backend PostgreSQL database and served to all organization members. When a victim opens the Create Transaction Group page and the group contains a transaction with a malicious description, the payload executes in their Electron renderer process.

### Impact Explanation
In an Electron application, HTML injection via `v-html` is equivalent to XSS. Even with `contextIsolation` enabled, a successful injection can:
- Steal sensitive data visible in the renderer (private key nicknames, account IDs, session tokens displayed on screen).
- Manipulate the DOM to display false transaction details to the victim at the moment of signing, causing them to sign a transaction they believe is different (directly analogous to the original report's misrepresentation concern).
- If `nodeIntegration` is enabled or `contextBridge` is misconfigured, escalate to arbitrary Node.js/OS code execution on the victim's machine.

The misrepresentation impact is concrete and immediate: a malicious description like `<b style="color:red">Transfer 1000 HBAR to attacker</b>` replaces the legitimate transaction summary in the group list, deceiving the signer about what they are about to approve.

### Likelihood Explanation
Any authenticated organization member can create a transaction with a crafted description or memo — no elevated privileges are required. The attack is triggered passively when any other member opens the Create Transaction Group page. In a multi-member organization (the primary use case of the tool), this is a realistic, low-effort attack.

### Recommendation
Replace `v-html` with safe text interpolation (`{{ }}`). Since the purpose of this cell is to display a short summary string, raw HTML rendering is unnecessary:

```html
<div ... >{{
  groupItem.type == 'Transfer Transaction'
    ? makeTransfer(index)
    : groupItem.description != ''
      ? groupItem.description
      : Transaction.fromBytes(groupItem.transactionBytes).transactionMemo
        || createTransactionId(groupItem.payerAccountId, groupItem.validStart)
}}</div>
```

If `makeTransfer` must return HTML markup (e.g., for bold formatting), sanitize its output with a library such as `DOMPurify` before passing it to `v-html`, and ensure `groupItem.description` and `transactionMemo` are never passed through `v-html`.

### Proof of Concept
1. Attacker (organization member) creates a transaction with description set to:
   ```
   <img src=x onerror="document.title='PWNED:'+document.cookie">
   ```
2. The transaction is saved to the backend and becomes visible to all organization members.
3. Victim (another organization member) opens **Create Transaction Group**, adds the attacker's transaction to the group.
4. The `v-html` binding at line 422 sets `innerHTML` to the unescaped description string.
5. The browser engine executes the `onerror` handler inside the Electron renderer, demonstrating arbitrary script execution. A real payload would exfiltrate key material or manipulate the signing UI. [4](#0-3)
