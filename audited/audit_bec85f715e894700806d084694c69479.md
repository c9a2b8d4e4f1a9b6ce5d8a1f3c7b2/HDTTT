### Title
Transaction Group Confirmation Modal Omits Per-Transaction `maxTransactionFee`, Enabling Fee Drain via Malicious Group Submission

### Summary
The "Confirm Transaction Group" modal in `TransactionGroupProcessor.vue` displays only transaction type and description for each group item — it never renders `maxTransactionFee` for any transaction. A user who imports a crafted transaction group (e.g., via CSV) or receives one through the organization workflow confirms and signs all transactions without ever seeing the fees they are authorizing. In contrast, the single-transaction confirmation in `ConfirmTransactionHandler.vue` does render `maxTransactionFee` (conditionally), making the group path the blind spot.

### Finding Description

**Root cause — `TransactionGroupProcessor.vue` template (lines 471–492):**

The group confirmation modal iterates over all group items and renders only transaction type and a description/memo/ID label. No `maxTransactionFee` field is extracted or displayed for any item. [1](#0-0) 

Compare this to the single-transaction path in `ConfirmTransactionHandler.vue`, which does render the fee (conditionally on `v-if="transaction?.maxTransactionFee"`): [2](#0-1) 

**Org signing path — `SignTransactionController.vue` (lines 111–120):**

The org-member signing flow uses `ActionController` with no `confirmTitle`, `confirmText`, or `dataTestid` props. Per `ActionController.vue` lines 48–54 and 117–124, when those props are absent `confirmationProps` is `null` and the controller skips the confirmation modal entirely, jumping straight to the password prompt. Org signers therefore receive zero transaction detail review — including no fee — before their key is used. [3](#0-2) [4](#0-3) [5](#0-4) 

**CSV import path — `ImportCSVController.vue`:**

Transaction groups can be imported from external CSV files. The CSV parser reads `transactionFee` from the file and builds group items. After import the user is shown the "Confirm Transaction Group" modal, which — as shown above — never displays the fee. [6](#0-5) 

**Exploit flow (personal mode, CSV vector):**

1. Attacker crafts a CSV transaction group with `maxTransactionFee` set to a large value (e.g., 1 000 HBAR per transaction) and a plausible description.
2. Victim imports the CSV via the normal "Import CSV" workflow.
3. Victim sees the "Confirm Transaction Group" modal: only transaction type and description are shown — no fee.
4. Victim clicks Confirm, enters their application password, and the group executes.
5. Each transaction charges up to the attacker-specified `maxTransactionFee` from the fee-payer account, draining it.

**Exploit flow (organization mode):**

1. Malicious org member (creator) submits a group with inflated `maxTransactionFee` targeting a shared fee-payer account.
2. Required signers click "Sign Transaction" on the transaction details page.
3. `SignTransactionController` → `ActionController` skips the confirmation dialog and immediately prompts for password.
4. Signer enters password; transaction is signed and eventually executed with the inflated fee.

### Impact Explanation

A user who confirms a transaction group or signs an org transaction has no in-dialog visibility into `maxTransactionFee`. A crafted group with per-transaction fees of hundreds of HBAR can drain the fee-payer account in a single confirmation click. On Hedera, `maxTransactionFee` is the ceiling the network may charge; setting it to the account balance effectively authorizes full drainage.

### Likelihood Explanation

The CSV import path requires the victim to open a file from an untrusted source — a realistic social-engineering step (e.g., a shared "template" file). The org path requires only a legitimate org membership (no privileged role). Both paths are reachable by a "malicious normal user abusing valid product/protocol flows" as defined in `RESEARCHER.md`. [7](#0-6) 

### Recommendation

1. **Group confirmation modal**: For every `groupItem`, decode `Transaction.fromBytes(groupItem.transactionBytes)` and render `maxTransactionFee` alongside the transaction type and description — mirroring the display already present in `ConfirmTransactionHandler.vue`.
2. **Org signing flow**: Pass `confirmTitle`, `confirmText`, and `dataTestid` to `ActionController` in `SignTransactionController.vue` so that a confirmation dialog (including fee) is shown before the password prompt.
3. **CSV import**: After parsing, surface the per-transaction fee in the import preview so users can review it before the group is added to the store.
