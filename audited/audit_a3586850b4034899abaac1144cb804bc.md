### Title
Hardcoded PBKDF2 Salt in `decryptLegacyMnemonic` Enables Precomputed Dictionary Attack Against BIP-39 Mnemonic Recovery File

### Summary
`decryptLegacyMnemonic` in `front-end/src/main/services/localUser/dataMigration.ts` uses a hardcoded, static salt `Buffer.from([1, 2, 3, 4, 5, 6, 7, 8])` when deriving the AES-256 decryption key via PBKDF2-SHA256. Because the salt is public and fixed, an attacker who obtains the `recovery.aes` file can precompute a PBKDF2 rainbow table for this exact salt and crack the user's password offline, recovering the BIP-39 mnemonic that controls all Hedera private keys.

### Finding Description
In `decryptLegacyMnemonic`, the key derivation is:

```typescript
// front-end/src/main/services/localUser/dataMigration.ts, line 108
const salt = Buffer.from([1, 2, 3, 4, 5, 6, 7, 8]);
const key = crypto.pbkdf2Sync(password, salt, iterationCount, keyLength, 'sha256');
``` [1](#0-0) 

The salt is a constant 8-byte sequence `[1, 2, 3, 4, 5, 6, 7, 8]` hardcoded in the source. The sole purpose of a salt in PBKDF2 is to prevent precomputed (rainbow table) attacks — a fixed, public salt completely defeats this protection.

The function decrypts the `recovery.aes` file located at `<Documents>/TransactionTools/Files/.System/recovery.aes`, which contains the user's BIP-39 mnemonic phrase. [2](#0-1) 

The encrypted file is read from a predictable, user-accessible path on disk: [3](#0-2) 

**Exploit path:**
1. Attacker obtains `recovery.aes` (via malware, cloud backup leak, physical access, or shared machine).
2. Salt is known: `[0x01, 0x02, 0x03, 0x04, 0x05, 0x06, 0x07, 0x08]`.
3. Attacker precomputes `PBKDF2-SHA256(candidate_password, [1,2,3,4,5,6,7,8], 65536, 32)` for a dictionary of common passwords — this table is reusable against **every** user of the legacy migration path.
4. Attacker tries each derived key against the AES-256-ECB ciphertext in `recovery.aes`.
5. Successful decryption yields the BIP-39 mnemonic.
6. Attacker derives all Hedera private keys from the mnemonic and drains all associated accounts.

Note: the non-legacy path (`decryptMnemonic`) correctly derives the salt from the user's token stored in `user.properties`, so it is **not** affected. [4](#0-3) 

### Impact Explanation
The BIP-39 mnemonic is the root secret for all Hedera private keys managed by the application. Recovering it gives the attacker complete, irrevocable control over every Hedera account derived from that mnemonic — enabling full asset theft. The hardcoded salt converts what should be a per-user brute-force problem into a one-time precomputation that applies to all legacy migration users simultaneously.

### Likelihood Explanation
The attacker must obtain the `recovery.aes` file. This is a local file at a predictable path, reachable via: malware/ransomware (common threat), cloud backup services that sync the Documents folder (e.g., iCloud, OneDrive, Google Drive — all enabled by default on many systems), physical access, or shared/compromised machines. The file is not protected by OS-level encryption beyond the user session. Once obtained, the precomputed table attack is entirely offline and requires no further interaction with the victim.

### Recommendation
Replace the hardcoded salt with a cryptographically random salt generated at encryption time and stored alongside the ciphertext (as the non-legacy `decryptMnemonic` path already does correctly):

```typescript
// At encryption time (old tool, not modifiable) — for the migration reader,
// document that the legacy format has no per-user salt and warn users.
// For any new encryption, use:
const salt = crypto.randomBytes(16);
```

Since the legacy `recovery.aes` format was produced by the old tool and cannot be re-encrypted, the minimum mitigation is to document the weakness prominently and encourage users to complete migration and delete the legacy file immediately after import. Additionally, enforce a strong password policy warning during the migration flow.

### Proof of Concept

```
# 1. Locate the legacy encrypted mnemonic
FILE=~/Documents/TransactionTools/Files/.System/recovery.aes

# 2. Salt is public: [0x01,0x02,0x03,0x04,0x05,0x06,0x07,0x08]

# 3. Precompute PBKDF2 keys for a password list (hashcat example)
#    hashcat mode 12000 = PBKDF2-HMAC-SHA256
#    Format: hash:salt:iterations:keylen
hashcat -m 12000 -a 0 \
  "$(base64 < $FILE):$(echo -n $SALT | base64):65536:32" \
  rockyou.txt

# 4. Use recovered key to decrypt recovery.aes with AES-256-ECB
#    Successful plaintext = BIP-39 mnemonic words
```

export async function decryptLegacyMnemonic(
  inputPath: string,
  password: string,
): Promise<string | null> {
  /* Read the encrypted data from the file */
  const data = await fs.promises.readFile(inputPath, { flag: 'r' });

  const iterationCount = 65536;
  const keyLength = 32; // 256 bits
  const salt = Buffer.from([1, 2, 3, 4, 5, 6, 7, 8]);

  const key = crypto.pbkdf2Sync(password, salt, iterationCount, keyLength, 'sha256');
  const header = Buffer.from('AES|256|CBC|PKCS5Padding|', 'utf-8');
  if (data.length < header.length || !header.equals(data.slice(0, header.length))) {
    const decipher = crypto.createDecipheriv('aes-256-ecb', key, null);
    const decrypted = Buffer.concat([decipher.update(data), decipher.final()]);
    return decrypted.toString('utf-8');
  }

  return null;
}

export async function decryptMnemonic(
  inputPath: string,
  token: string,
  password: string,
): Promise<string | null> {
  /* Read the encrypted data from the file */
  const data = await fs.promises.readFile(inputPath, { flag: 'r' });

  /* Get the salt from the token */
  const salt = getSalt(token);

  /* Generate the key from the password and the salt */
  const key = await generateArgon2id(password, salt);

  /* Get the header, auth tag, encrypted text, and IV from the data */
  const header = Buffer.from('AES|256|CBC|PKCS5Padding|', 'utf8');
  const authTag = data.subarray(data.length - AUTH_TAG_LENGTH);
  const encryptedText = data.subarray(header.length + IV_LENGTH, data.length - AUTH_TAG_LENGTH);
  const iv = data.subarray(header.length, header.length + IV_LENGTH);

  /* Create a decipher, set the auth tag */
  const decipher = crypto.createDecipheriv('aes-256-gcm', key, new Uint8Array(iv));
  decipher.setAuthTag(new Uint8Array(authTag));

  try {
    /* Decrypt the encrypted text */
    const decrypted = Buffer.concat([decipher.update(encryptedText), decipher.final()]);
    return decrypted.toString('utf-8');
  } catch (error) {
    logger.error('Error decrypting mnemonic', { error });
    return null;
  }
}
