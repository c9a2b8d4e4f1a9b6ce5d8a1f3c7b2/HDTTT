### Title
Unbounded Recursive Loop with Sequential DB Queries in `createTransactionApprovers` Enables Resource Exhaustion DoS

### Summary
The `createTransactionApprovers` function in `approvers.service.ts` accepts an arbitrarily large, arbitrarily deep nested approver tree from any authenticated transaction creator. For each node in the tree, it executes multiple sequential database queries inside a single long-running DB transaction. Because neither `approversArray` nor nested `approvers` arrays have a maximum-size constraint in the DTO, a single crafted request can trigger thousands of sequential DB queries, exhausting server memory, DB connection time, and causing request timeouts that degrade service for all users.

### Finding Description

**Root cause — no array size cap in the DTO:**

`CreateTransactionApproversArrayDto` and `CreateTransactionApproverDto` impose no upper bound on array length: [1](#0-0) 

`approversArray` has only `@IsArray()` + `@ValidateNested()`. Nested `approvers` has only `@ArrayMinSize(1)`. Neither has `@ArrayMaxSize()`.

**Unbounded recursive loop with sequential DB calls:**

`createTransactionApprovers` iterates over `dto.approversArray` and for every element calls the inner async `createApprover`, which recursively calls itself for every nested child: [2](#0-1) 

Per node, `createApprover` issues at minimum **4–6 sequential awaited DB queries**:
1. `this.isNode()` — a `COUNT` query
2. `transactionalEntityManager.findOne(TransactionApprover, ...)` — parent lookup
3. `this.getRootNodeFromNode(...)` — recursive CTE traversal
4. `transactionalEntityManager.count(User, ...)` — user existence check
5. `this.getApproversByTransactionId(...)` — recursive CTE
6. `transactionalEntityManager.insert(TransactionApprover, ...)` — insert

All of this runs inside a single DB transaction: [3](#0-2) 

**Entry point — normal authenticated user:**

The controller exposes this as a `POST /transactions/:transactionId/approvers` endpoint, guarded only by JWT auth and a check that the caller is the transaction creator: [4](#0-3) 

Any registered user who creates a transaction can immediately call this endpoint.

### Impact Explanation

A single crafted POST body with `approversArray` containing, e.g., 1 000 entries each with 10 nested children (10 000 total nodes) triggers ~50 000–60 000 sequential DB queries inside one DB transaction. This:

- Holds a DB connection and transaction lock for the entire duration, starving other requests of DB connections.
- Grows the in-memory `approvers[]` array without bound, risking OOM on the Node.js process.
- Causes the request to time out (or the DB transaction to time out), returning an error to the attacker while the damage (resource consumption) has already occurred.
- Degrades or crashes the API service for all concurrent users.

### Likelihood Explanation

Any authenticated user (no admin role required) who has created at least one transaction can trigger this. The API is reachable over the network. Crafting the payload requires only knowledge of the documented JSON schema (shown in `front-end/docs/api.md`). No special tooling is needed — a single `curl` or HTTP client call suffices.

### Recommendation

1. Add `@ArrayMaxSize(N)` (e.g., `N = 20`) to both `approversArray` in `CreateTransactionApproversArrayDto` and `approvers` in `CreateTransactionApproverDto`.
2. Enforce a maximum recursion depth (e.g., 5 levels) inside `createApprover` by passing and checking a depth counter.
3. Consider batching DB inserts rather than one `insert` per node.

### Proof of Concept

```http
POST /transactions/1/approvers
Authorization: Bearer <valid_jwt_of_transaction_creator>
Content-Type: application/json

{
  "approversArray": [
    { "userId": 1 },
    { "userId": 2 },
    ... (repeat 1000 times)
  ]
}
```

Or, for exponential amplification via nesting:

```json
{
  "approversArray": [{
    "threshold": 1,
    "approvers": [{
      "threshold": 1,
      "approvers": [{
        "threshold": 1,
        "approvers": [... 20 levels deep, each with 10 children ...]
      }]
    }]
  }]
}
```

Each node triggers 4–6 sequential DB queries inside the same open DB transaction. With 10 000 nodes, this is ~50 000 DB round-trips in a single request, exhausting the DB connection pool and server memory. [5](#0-4)

  async createTransactionApprovers(
    user: User,
    transactionId: number,
    dto: CreateTransactionApproversArrayDto,
  ): Promise<TransactionApprover[]> {
    await this.getCreatorsTransaction(transactionId, user);

    const approvers: TransactionApprover[] = [];

    try {
      await this.dataSource.transaction(async transactionalEntityManager => {
        const createApprover = async (dtoApprover: CreateTransactionApproverDto) => {
          /* Validate Approver's DTO */
          this.validateApprover(dtoApprover);

          /* Check if the approver already exists */
          if (await this.isNode(dtoApprover, transactionId, transactionalEntityManager))
            throw new Error(this.APPROVER_ALREADY_EXISTS);

          /* Check if the parent approver exists and has threshold */
          if (typeof dtoApprover.listId === 'number') {
            const parent = await transactionalEntityManager.findOne(TransactionApprover, {
              where: { id: dtoApprover.listId },
            });

            if (!parent) throw new Error(this.PARENT_APPROVER_NOT_FOUND);

            /* Check if the root transaction is the same */
            const root = await this.getRootNodeFromNode(
              dtoApprover.listId,
              transactionalEntityManager,
            );
            if (root?.transactionId !== transactionId)
              throw new Error(this.ROOT_TRANSACTION_NOT_SAME);
          }

          /* Check if the user exists */
          if (typeof dtoApprover.userId === 'number') {
            const userCount = await transactionalEntityManager.count(User, {
              where: { id: dtoApprover.userId },
            });

            if (userCount === 0) throw new Error(this.USER_NOT_FOUND(dtoApprover.userId));
          }

          /* Check if there are sub approvers */
          if (
            typeof dtoApprover.userId === 'number' &&
            dtoApprover.approvers &&
            dtoApprover.approvers.length > 0
          )
            throw new Error(this.ONLY_USER_OR_TREE);

          /* Check if the approver has threshold when there are children */
          if (
            dtoApprover.approvers &&
            dtoApprover.approvers.length > 0 &&
            (dtoApprover.threshold === null || isNaN(dtoApprover.threshold))
          )
            throw new Error(this.THRESHOLD_REQUIRED);

          /* Check if the approver has children when there is threshold */
          if (
            typeof dtoApprover.threshold === 'number' &&
            (!dtoApprover.approvers || dtoApprover.approvers.length === 0)
          )
            throw new Error(this.CHILDREN_REQUIRED);

          /* Check if the approver threshold is less or equal to the number of approvers */
          if (
            dtoApprover.approvers &&
            (dtoApprover.threshold > dtoApprover.approvers.length || dtoApprover.threshold === 0)
          )
            throw new Error(this.THRESHOLD_LESS_OR_EQUAL_APPROVERS(dtoApprover.approvers.length));

          const data: DeepPartial<TransactionApprover> = {
            transactionId:
              dtoApprover.listId === null || isNaN(dtoApprover.listId) ? transactionId : null,
            listId: dtoApprover.listId,
            threshold:
              dtoApprover.threshold && dtoApprover.approvers ? dtoApprover.threshold : null,
            userId: dtoApprover.userId,
          };

          if (typeof dtoApprover.userId === 'number') {
            const userApproverRecords = await this.getApproversByTransactionId(
              transactionId,
              dtoApprover.userId,
              transactionalEntityManager,
            );

            if (userApproverRecords.length > 0) {
              data.signature = userApproverRecords[0].signature;
              data.userKeyId = userApproverRecords[0].userKeyId;
              data.approved = userApproverRecords[0].approved;
            }
          }

          /* Create approver */
          const approver = transactionalEntityManager.create(TransactionApprover, data);

          /* Insert approver */
          await transactionalEntityManager.insert(TransactionApprover, approver);
          approvers.push(approver);

          /* Continue creating the three */
          if (dtoApprover.approvers) {
            for (const nestedDtoApprover of dtoApprover.approvers) {
              const nestedApprover = { ...nestedDtoApprover, listId: approver.id };

              if (!nestedDtoApprover.approvers || nestedDtoApprover.approvers.length === 0) {
                nestedApprover.threshold = null;
              }

              await createApprover({ ...nestedDtoApprover, listId: approver.id });
            }
          }
        };

        for (const approver of dto.approversArray) {
          await createApprover(approver);
        }
      });

      emitTransactionStatusUpdate(this.notificationsPublisher, [{ entityId: transactionId  }]);
    } catch (error) {
      throw new BadRequestException(error.message);
    }

    return approvers;
  }
