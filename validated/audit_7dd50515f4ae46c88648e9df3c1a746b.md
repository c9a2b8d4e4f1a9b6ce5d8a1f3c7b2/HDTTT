### Title
Missing Cryptographic Signature Verification in `uploadSignatureMaps` Allows Any Required Signer to Permanently Corrupt a Multi-Sig Transaction

---

### Summary

The `processTransactionSignatures` method in `signers.service.ts` checks that a submitted public key belongs to the authenticated user but **never cryptographically verifies** that the accompanying signature bytes are valid for that key and transaction body. An authenticated user who is a required signer can submit a properly-formatted but cryptographically invalid signature. The system accepts it, writes the corrupted bytes into the stored transaction, and marks the user as having signed. When the chain service later attempts to execute the transaction, the Hedera network rejects it with `INVALID_SIGNATURE`, permanently failing the transaction.

This is the direct analog of the external report: just as a malicious covenant committee member could return a garbage signature that gets blindly included in the unbonding transaction, a malicious required signer here can submit a garbage signature that gets blindly written into the stored transaction bytes.

---

### Finding Description

**Vulnerable path:** `POST /transactions/:transactionId/signers` → `SignersController.uploadSignatureMap` → `SignersService.uploadSignatureMaps` → `SignersService.processTransactionSignatures`

In `processTransactionSignatures` (signers.service.ts lines 217–267), the code:

1. Iterates over every public key in the submitted `SignatureMap`
2. Verifies the public key **belongs to the authenticated user** (ownership check only)
3. Calls `sdkTransaction.addSignature(publicKey, map)` — which blindly appends whatever bytes are in the map to the transaction, with no cryptographic check [1](#0-0) 

The `IsSignatureMap` DTO decorator only validates structure (node account ID format, transaction ID format, public key parsability, non-zero byte length). It does **not** call `publicKey.verify()`. [2](#0-1) 

**Contrast with the secure path:** `importSignatures` in `transactions.service.ts` (lines 545–549) calls `validateSignature` before `addSignature`, which cryptographically verifies each signature against the transaction body bytes before accepting it. [3](#0-2) 

The utility `verifyTransactionBodyWithoutNodeAccountIdSignature` exists in the codebase and is capable of performing this check, but it is not called in the signer upload path. [4](#0-3) 

---

### Impact Explanation

A required signer submits a 64-byte random hex string as their "signature" (passes the `IsSignatureMap` non-empty check). The server:

- Accepts the submission (public key ownership verified ✓)
- Writes the invalid signature into `transaction.transactionBytes` in the database
- Records the attacker as a signer in `TransactionSigner`
- If this was the last required signature, advances status to `WAITING_FOR_EXECUTION`

When the chain service calls `getValidatedSDKTransaction`, it checks `hasValidSignatureKey` against the set of signer public keys (not the cryptographic validity of the bytes), so it may pass that check. The transaction is then submitted to the Hedera network, which rejects it with `INVALID_SIGNATURE`. The transaction is permanently failed. [5](#0-4) 

The stored `transactionBytes` now contain the corrupted signature. There is no recovery path — the transaction cannot be re-signed because it is no longer in `WAITING_FOR_SIGNATURES` status.

---

### Likelihood Explanation

- The attacker must be an **authenticated user** with a **registered public key** that is a required signer for the target transaction. This is a realistic role in any multi-sig organization workflow.
- The attack requires only a single API call with a valid-format but random-bytes signature value.
- No special tooling is needed — any HTTP client can craft the payload.
- The attacker is not punished or flagged; the system records them as having signed.

---

### Recommendation

Before calling `sdkTransaction.addSignature(publicKey, map)` in `processTransactionSignatures`, cryptographically verify each signature using the existing `validateSignature` utility (as already done in `importSignatures`), or call `publicKey.verify(transactionBodyBytes, signatureBytes)` directly. Reject the entire upload if any signature in the map fails verification. [6](#0-5) 

---

### Proof of Concept

1. Register as a user with public key `PK_attacker` in an organization.
2. Obtain a transaction `T` that requires `PK_attacker` as one of its signers.
3. Craft a `POST /transactions/T.id/signers` request body:
```json
{
  "id": <T.id>,
  "signatureMap": {
    "0.0.3": {
      "<T.transactionId>": {
        "<PK_attacker_DER>": "aabbccdd...aabbccdd"
      }
    }
  }
}
```
where `aabbccdd...aabbccdd` is 64 bytes of random hex (valid format, cryptographically invalid).

4. The server responds `201` and records the attacker as a signer.
5. If `T` required only this signature, its status advances to `WAITING_FOR_EXECUTION`.
6. The chain service submits the transaction; Hedera returns `INVALID_SIGNATURE`; the transaction is permanently marked `FAILED`. [7](#0-6) [8](#0-7)

  private async processTransactionSignatures(
    transaction: Transaction,
    map: SignatureMap,
    userKeyMap: Map<string, UserKey>,
    existingSignerIds: Set<number>
  ) {
    let sdkTransaction = SDKTransaction.fromBytes(transaction.transactionBytes);

    const userKeys: UserKey[] = [];
    const processedRawKeys = new Set<string>();

    // To explain what is going on here, we need to understand how sdkTransaction.addSignature works.
    // The addSignature method will go through each inner transaction, then go through the map
    // and pull the signatures for the supplied public key belonging to that inner transaction
    // (denoted by the node and transaction id), add the signatures to the inner transactions.
    // So we need to go through the map and get each unique publicKey and call addSignature one time
    // per key.
    for (const nodeMap of map.values()) {
      for (const txMap of nodeMap.values()) {
        for (const publicKey of txMap.keys()) {
          const raw = publicKey.toStringRaw();

          // Skip duplicates across node/tx maps, and already-processed keys
          if (processedRawKeys.has(raw)) continue;
          processedRawKeys.add(raw);

          // Look up key (raw first, then DER)
          let userKey = userKeyMap.get(raw);
          if (!userKey) {
            userKey = userKeyMap.get(publicKey.toStringDer());
          }
          if (!userKey) throw new Error(ErrorCodes.PNY);

          // Only add the signature once per unique key
          sdkTransaction = sdkTransaction.addSignature(publicKey, map);

          // Only return "new" signers (not already persisted)
          if (!existingSignerIds.has(userKey.id)) {
            userKeys.push(userKey);
          }
        }
      }
    }

    // Finally, compare the resulting transaction bytes to see if any signatures were actually added
    const isSameBytes = Buffer.from(sdkTransaction.toBytes()).equals(
      transaction.transactionBytes
    );

    return { sdkTransaction, userKeys, isSameBytes };
  }

/* Verify the signature of the transaction body without node account id */
export const verifyTransactionBodyWithoutNodeAccountIdSignature = (
  transaction: SDKTransaction,
  signature: string | Buffer,
  publicKey: string | PublicKey,
) => {
  const bodyBytes = getTransactionBodyBytes(transaction);

  /* Deserialize Public Key */
  publicKey = publicKey instanceof PublicKey ? publicKey : PublicKey.fromString(publicKey);

  /* Deserialize Signature */
  signature = typeof signature === 'string' ? decode(signature) : signature;

  try {
    return publicKey.verify(bodyBytes, signature);
  } catch (err) {
    console.log(err);
    return false;
  }

  private async getValidatedSDKTransaction(
    transaction: Transaction,
  ): Promise<SDKTransaction> {
    /* Throws an error if the transaction is not found or in incorrect state */
    if (!transaction) throw new Error('Transaction not found');

    await this.validateTransactionStatus(transaction);

    /* Gets the SDK transaction from the transaction body */
    const sdkTransaction = SDKTransaction.fromBytes(transaction.transactionBytes);

    /* Gets the signature key */
    const signatureKey = await this.transactionSignatureService.computeSignatureKey(transaction);

    /* Checks if the transaction has valid signatureKey */
    if (!hasValidSignatureKey([...sdkTransaction._signerPublicKeys], signatureKey))
      throw new Error('Transaction has invalid signature.');

    return sdkTransaction;
