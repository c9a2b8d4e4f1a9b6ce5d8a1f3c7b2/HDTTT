Audit Report

## Title
Missing Validation of `mirrorNetwork` Parameter Enables SSRF via Arbitrary Network Identifier

## Summary
`CreateTransactionDto` accepts a `mirrorNetwork` field validated only as a non-empty string with no whitelist enforcement. Any authenticated user can supply an arbitrary hostname or IP address, causing the backend to initiate outbound gRPC connections (and subsequently HTTP connections) to attacker-controlled or internal infrastructure.

## Finding Description

**Root cause — no whitelist on `mirrorNetwork`:**

`CreateTransactionDto` decorates `mirrorNetwork` only with `@IsNotEmpty()` and `@IsString()`. No `@IsIn([...])` or `@IsEnum(...)` guard exists. [1](#0-0) 

**Code path that triggers the outbound gRPC connection:**

`createTransactions` calls `getClientFromNetwork(dtos[0].mirrorNetwork)` directly with the raw user-supplied value before any further validation. [2](#0-1) 

`getClientFromNetwork` falls through to the custom-network branch for any value that is not `mainnet`, `testnet`, `previewnet`, or `local-node`. [3](#0-2) 

`MirrorNetworkGRPC.fromBaseURL` appends `:443` to the raw user-supplied string and uses it as the gRPC mirror-node endpoint. [4](#0-3) 

The `AddressBookQuery().execute(client)` call at lines 42–44 of `client.ts` then initiates a live gRPC TCP connection to `${mirrorNetwork}:443`. [5](#0-4) 

**Secondary HTTP SSRF path:**

`MirrorNodeREST.fromBaseURL` constructs `https://${mirrorNetwork}` for any unrecognized value. [6](#0-5) 

`getMirrorNodeRESTURL` uses this directly to build the URL for `fetchAccountInfo` and `fetchNodeInfo`. [7](#0-6) 

This produces `https://<attacker-host>/api/v1/accounts/...` and `https://<attacker-host>/api/v1/network/nodes?...` — full HTTP SSRF to any URL the attacker embeds in `mirrorNetwork`. [8](#0-7) [9](#0-8) 

## Impact Explanation

- **gRPC SSRF (immediate, no extra infrastructure):** Any authenticated user can force the API server to open a gRPC TCP connection to an arbitrary `host:443`. This enables internal port scanning, probing of internal gRPC services, and exfiltration of the server's egress IP.
- **HTTP SSRF (requires attacker-controlled gRPC server):** If the attacker operates a gRPC mirror-node stub that returns a syntactically valid address book, the transaction is persisted with the attacker-controlled `mirrorNetwork`. The cache services subsequently issue HTTP GET requests to `https://<attacker-host>/api/v1/accounts/...`, enabling access to internal REST services, cloud metadata endpoints (e.g., `169.254.169.254`), and potential credential theft in cloud environments.

## Likelihood Explanation

The attacker only needs a valid authenticated session (normal user account). No admin or privileged role is required. The gRPC SSRF path requires zero additional infrastructure — the connection attempt is made unconditionally during `POST /transactions`, even if it ultimately fails. The HTTP SSRF path requires the attacker to run a minimal gRPC stub that returns a syntactically valid address book, which is low effort.

## Recommendation

Add a whitelist validator to `CreateTransactionDto.mirrorNetwork`. For example:

```typescript
import { IsIn, IsNotEmpty, IsString } from 'class-validator';

const ALLOWED_NETWORKS = ['mainnet', 'testnet', 'previewnet', 'local-node'];

@IsNotEmpty()
@IsString()
@IsIn(ALLOWED_NETWORKS)
mirrorNetwork: string;
```

If custom networks must be supported, enforce a strict allowlist via configuration (e.g., an environment variable or database-backed allowlist) and validate the supplied value against it before calling `getClientFromNetwork`.

## Proof of Concept

```
POST /transactions HTTP/1.1
Authorization: Bearer <valid_user_token>
Content-Type: application/json

{
  "name": "SSRF Test",
  "description": "SSRF via mirrorNetwork",
  "transactionBytes": "<valid_hex>",
  "creatorKeyId": 1,
  "signature": "<valid_hex>",
  "mirrorNetwork": "169.254.169.254"
}
```

The backend calls `getClientFromNetwork("169.254.169.254")`, which falls through to the custom-network branch, calls `MirrorNetworkGRPC.fromBaseURL("169.254.169.254")` returning `["169.254.169.254:443"]`, sets it as the gRPC mirror endpoint, and then executes `AddressBookQuery().execute(client)` — initiating a live TCP connection to `169.254.169.254:443`. If a valid address book is returned, the transaction is stored with `mirrorNetwork = "169.254.169.254"`, and subsequent cache refresh calls issue HTTP GET requests to `https://169.254.169.254/api/v1/accounts/...`.
