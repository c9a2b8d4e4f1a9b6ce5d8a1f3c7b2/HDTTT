### Title
Unauthenticated-Scope Data Exposure via Unguarded `getTransactionGroups` Endpoint and Missing Authorization on Comment Routes

### Summary
Two sets of endpoints in the NestJS API service expose organization-wide sensitive data to any authenticated user without ownership or role checks, directly analogous to the `WeightedFarmingPool` publicly-exposed management functions. The most severe is `GET /transaction-groups`, which returns every transaction group in the database with no user-scoping. A secondary issue exists in the comments controller, where any authenticated user can post to or read comments on any transaction they do not own or participate in.

---

### Finding Description

**Issue 1 — `GET /transaction-groups` returns all groups to any authenticated user**

The controller-level guard on `TransactionGroupsController` is `JwtBlackListAuthGuard, JwtAuthGuard, VerifiedUserGuard` — authentication only. The `getTransactionGroups` handler adds no further restriction:

```typescript
/* TESTING ONLY: Get all transactions groups */
@Get()
@Serialize(TransactionGroupDto)
getTransactionGroups(): Promise<TransactionGroup[]> {
  return this.transactionGroupsService.getTransactionGroups();
}
``` [1](#0-0) 

The service method performs a bare `find` with no `where` clause:

```typescript
getTransactionGroups(): Promise<TransactionGroup[]> {
  return this.dataSource.manager.find(TransactionGroup);
}
``` [2](#0-1) 

The comment "TESTING ONLY" confirms this was never intended for production use, yet it is reachable by any valid JWT holder.

Compare with the sibling `getTransactionGroup` (singular), which correctly scopes results to the requesting user via `getTransactionGroupItemsQuery(this.sqlBuilder, id, user)` and throws `UnauthorizedException` when the user has no items. [3](#0-2) 

---

**Issue 2 — `CommentsController` has no authorization on `createComment`, `getComments`, or `getCommentById`**

The controller carries only the class-level authentication guard. The `createComment` handler even has a developer TODO acknowledging the missing check:

```typescript
@Post()
//TODO need some sort of guard or check to ensure user can comment here
createComment(
  @GetUser() user: User,
  @Param('transactionId', ParseIntPipe) transactionId: number,
  @Body() dto: CreateCommentDto,
) {
  return this.commentsService.createComment(user, transactionId, dto);
}
``` [4](#0-3) 

`getComments` and `getCommentById` expose comments on any transaction to any authenticated user with no ownership or participation check: [5](#0-4) 

---

### Impact Explanation

- **Issue 1:** Any authenticated organization member can enumerate the IDs, metadata, and structure of every transaction group in the system — including those belonging to other users. Transaction groups contain financial transaction bytes, signing keys, and execution status. This is a full horizontal privilege escalation for read access.
- **Issue 2:** Any authenticated user can inject comments into transactions they have no role in (creator, signer, approver, observer), and read comments on any transaction. This pollutes audit trails and leaks transaction discussion data across organizational boundaries.

---

### Likelihood Explanation

Both endpoints are reachable with a standard authenticated HTTP request — no special tooling required. Any user who has registered and obtained a JWT can immediately call `GET /transaction-groups` or `POST /transactions/:id/comments`. The "TESTING ONLY" comment on Issue 1 indicates the endpoint was left in accidentally, making accidental or deliberate exploitation straightforward.

---

### Recommendation

**Issue 1:**
- Remove the `getTransactionGroups` endpoint entirely, or restrict it behind `AdminGuard` if an admin-scoped listing is genuinely needed.
- If a user-scoped listing is needed, add a `user` parameter and filter with a `where` clause (e.g., join through `TransactionGroupItem → Transaction → creatorKey → userId`).

**Issue 2:**
- Before allowing `createComment`, verify the requesting user is a creator, signer, approver, or observer of the target transaction (reuse the existing `getTransactionWithVerifiedAccess` pattern from `TransactionsService`).
- Apply the same check to `getComments` and `getCommentById`.

```

POST /auth/login

→ { "accessToken": "<JWT>" }

# Step 2: Dump all transaction groups (no ownership check)
GET /transaction-groups
Authorization: Bearer <JWT>
→ 200 OK — returns ALL TransactionGroup records in the database

# Step 3: Post a comment on a transaction the attacker does not own
POST /transactions/999/comments
Authorization: Bearer <JWT>
{ "content": "Injected comment on someone else's transaction" }
→ 201 Created — comment is persisted with no access check
```

```typescript
  async getTransactionGroup(user: User, id: number, full?: boolean): Promise<TransactionGroup> {
    const group = await this.dataSource.manager.findOne(TransactionGroup, {
      where: { id },
    });

    if (!group) {
      throw new BadRequestException(ErrorCodes.TNF);
    }

    const query = getTransactionGroupItemsQuery(this.sqlBuilder, id, user);

    const rows = await this.dataSource.manager.query(
      query.text,
      query.values,
    );

    group.groupItems = rows.map(row => {
      const creator = this.dataSource.manager.create(User, {
        id: row.tx_creator_key_user_id,
        email: row.tx_creator_email,
      });

      const creatorKey = this.dataSource.manager.create(UserKey, {
        id: row.tx_creator_key_id,
        userId: row.tx_creator_key_user_id,
        user: creator,
      });

      const transaction = this.dataSource.manager.create(Transaction, {
        id: row.tx_id,
        name: row.tx_name,
        type: row.tx_type,
        description: row.tx_description,
        transactionId: row.sdk_transaction_id,
        transactionHash: row.tx_transaction_hash,
        transactionBytes: row.tx_transaction_bytes,
        unsignedTransactionBytes: row.tx_unsigned_transaction_bytes,
        status: row.tx_status,
        statusCode: row.tx_status_code,
        creatorKeyId: row.tx_creator_key_id,
        creatorKey,
        signature: row.tx_signature,
        validStart: row.tx_valid_start,
        mirrorNetwork: row.tx_mirror_network,
        isManual: row.tx_is_manual,
        cutoffAt: row.tx_cutoff_at,
        createdAt: row.tx_created_at,
        executedAt: row.tx_executed_at,
        updatedAt: row.tx_updated_at,
      });

      return this.dataSource.manager.create(TransactionGroupItem, {
        seq: row.gi_seq,
        groupId: id,
        transactionId: row.tx_id,
        transaction,
      });
    });

    if (group.groupItems.length === 0) {
      throw new UnauthorizedException("You don't have permission to view this group.");
    }
