### Title
Unauthenticated Key Ownership Check in `importSignatures` Allows Any Authorized User to Inject Signatures from Keys They Do Not Own

---

### Summary

The `POST /transactions/signatures/import` endpoint (`importSignatures`) validates that submitted signatures are cryptographically valid but does **not** verify that the public keys in the submitted `SignatureMap` belong to the authenticated user. Any user with access to a transaction (including observers) can replay or inject a valid signature produced by a different user's private key, advancing the transaction state without the real key owner's consent.

---

### Finding Description

The `importSignatures` method in `transactions.service.ts` processes a caller-supplied `SignatureMap` through two checks:

1. **Access check** — `verifyAccess(transaction, user)` confirms the caller is a creator, observer, signer, or approver of the transaction.
2. **Cryptographic check** — `validateSignature(sdkTransaction, map)` confirms each `(publicKey, signature)` pair is mathematically valid over the transaction body bytes.

What is **absent** is a check that the public keys inside the `SignatureMap` are registered to the authenticated user. [1](#0-0) 

```typescript
const { data: publicKeys, error } = safe<PublicKey[]>(
  validateSignature.bind(this, sdkTransaction, map),
);
if (error) throw new BadRequestException(ErrorCodes.ISNMPN);

for (const publicKey of publicKeys) {
  sdkTransaction.addSignature(publicKey, map);
}
```

`validateSignature` only verifies the signature bytes are valid for the supplied public key — it does not consult the database to confirm that key is owned by the calling user. [2](#0-1) 

Contrast this with the **sibling endpoint** `uploadSignatureMaps` in `signers.service.ts`, which explicitly builds a `userKeyMap` from `user.keys` and throws `ErrorCodes.PNY` if any public key in the map is not found in that set: [3](#0-2) 

```typescript
// Build user key lookup once
const userKeyMap = new Map<string, UserKey>();
for (const key of user.keys) {
  userKeyMap.set(key.publicKey, key);
}
// ...
let userKey = userKeyMap.get(raw);
if (!userKey) userKey = userKeyMap.get(publicKey.toStringDer());
if (!userKey) throw new Error(ErrorCodes.PNY);   // ← enforced here
```

The `importSignatures` path has no equivalent guard.

The `verifyAccess` check that `importSignatures` does perform is broad: it returns `true` for observers, approvers, and anyone whose keys appear in the required-signer set — none of whom need to own the injected key. [4](#0-3) 

---

### Impact Explanation

An attacker who is an observer (or any other participant) on a transaction can:

1. Obtain a valid `SignatureMap` produced by another user (e.g., from a shared file, network interception, or a previously exported `.sigmap` file).
2. Call `POST /transactions/signatures/import` with that map.
3. The server accepts the map, adds the foreign signature to `transactionBytes`, and emits a `transactionStatusUpdate` event.

Because `transactionBytes` now carries the injected signature, the chain service may advance the transaction to `WAITING_FOR_EXECUTION` and submit it to the Hedera network — all without the real key owner ever consenting to this specific transaction state. Additionally, no `TransactionSigner` record is created, so the audit trail does not reflect who actually submitted the signature.

---

### Likelihood Explanation

- The endpoint is reachable by any authenticated, verified user who has observer or other access to a transaction — a low-privilege role.
- Valid `SignatureMap` files are routinely exported and shared in the tool's offline signing workflow, making them available to other participants.
- No special tooling is required; a standard HTTP client suffices.

---

### Recommendation

Inside `importSignatures`, after `validateSignature` returns the set of valid public keys, cross-reference each key against the authenticated user's registered keys (as `uploadSignatureMaps` already does). Reject the request if any key in the map is not owned by the caller:

```typescript
await attachKeys(user, this.entityManager);
const userKeyMap = new Map(user.keys.map(k => [k.publicKey, k]));

for (const publicKey of publicKeys) {
  const raw = publicKey.toStringRaw();
  const der  = publicKey.toStringDer();
  if (!userKeyMap.has(raw) && !userKeyMap.has(der)) {
    throw new BadRequestException(ErrorCodes.PNY);
  }
  sdkTransaction.addSignature(publicKey, map);
}
```

This mirrors the guard already present in `signers.service.ts` and closes the ownership gap.

---

### Proof of Concept

**Setup**: UserA and UserB both have access to transaction `#42`. UserA is a required signer; UserB is an observer. UserA exports a `.sigmap` file after signing offline.

**Attack**:

```http
POST /transactions/signatures/import
Authorization: Bearer <UserB_JWT>
Content-Type: application/json

[{
  "id": 42,
  "signatureMap": <UserA's exported SignatureMap>
}]
```

**Result**: The server calls `validateSignature` — the signature is valid (UserA really signed it). No ownership check is performed. `transactionBytes` is updated with UserA's signature attributed to UserB's API call. The `transactionStatusUpdate` event fires. If this was the last required signature, the transaction advances to `WAITING_FOR_EXECUTION` and is submitted to Hedera — without UserA's knowledge or consent for this particular invocation. export const validateSignature = (transaction: SDKTransaction, signatureMap: SignatureMap) => {
  const signerPublicKeys: PublicKey[] = [];

  const { rowLength, nodeAccountIdRow, transactionIdCol } =
    getSignedTransactionsDimensions(transaction);

  for (const [nodeAccountId, transactionIds] of signatureMap._map) {
    for (const [transactionId, publicKeys] of transactionIds._map) {
      for (const [publicKeyDer, signature] of publicKeys._map) {
        const publicKey = PublicKey.fromString(publicKeyDer);
        const publicKeyHex = publicKey.toStringRaw();

        const alreadySigned =
          transaction._signerPublicKeys.has(publicKeyHex) ||
          transaction._signerPublicKeys.has(publicKeyDer);

        if (!alreadySigned) {
          const row = nodeAccountIdRow[nodeAccountId];
          const col = transactionIdCol[transactionId];

          const bodyBytes = transaction._signedTransactions.get(col * rowLength + row).bodyBytes;

          const signatureValid = publicKey.verify(bodyBytes, signature);

          if (signatureValid) {
            signerPublicKeys.push(publicKey);
          } else {
            throw new Error('Invalid signature');
          }
        }
      }
    }
  }

  return signerPublicKeys;
};
