### Title
Unbounded Recursive Approver Tree Creation Causes Server-Side Denial of Service

### Summary
The `createTransactionApprovers` function in `approvers.service.ts` uses an unbounded recursive inner function `createApprover` to process nested approver DTOs supplied directly by the caller. No depth limit is enforced on the nesting level of the approver tree. A malicious authenticated user who owns any transaction can submit a deeply nested `approversArray` payload to exhaust the Node.js call stack or saturate the database connection pool, causing a server-side denial of service.

### Finding Description

In `back-end/apps/api/src/transactions/approvers/approvers.service.ts`, the `createTransactionApprovers` method defines an inner async function `createApprover` that calls itself recursively for every nested child approver in the incoming DTO: [1](#0-0) 

The recursive self-call occurs at line 348: [2](#0-1) 

There is no guard, counter, or maximum depth check anywhere in this path. Each recursive invocation also issues multiple database round-trips (`isNode`, `getRootNodeFromNode`, `count`, `insert`), so even moderate nesting depths (hundreds of levels) can saturate the PostgreSQL connection pool before a stack overflow is reached.

The same unbounded pattern appears in the PostgreSQL `WITH RECURSIVE` CTEs used throughout the service — `getApproversByTransactionId`, `getTransactionApproversById`, `getRootNodeFromNode`, and `removeNode` — all of which traverse the `transaction_approver` linked list without a depth cap: [3](#0-2) [4](#0-3) [5](#0-4) 

**Exploit path:**
1. Attacker registers as a normal user and creates any transaction (no special privilege required beyond being a registered user).
2. Attacker crafts a POST request to the approvers endpoint with a JSON body containing an `approversArray` whose `approvers` field is nested arbitrarily deep (e.g., 10,000 levels).
3. `createTransactionApprovers` calls `createApprover` recursively once per nesting level with no termination guard.
4. Node.js call stack overflows (`RangeError: Maximum call stack size exceeded`) or the database connection pool is exhausted by the flood of per-node queries.
5. The API process crashes or becomes unresponsive, affecting all users.

The only precondition checked before entering the recursive loop is that the caller is the creator of the target transaction: [6](#0-5) 

This is satisfied by any authenticated user who has created at least one transaction.

### Impact Explanation
A single crafted request can crash the Node.js API process or exhaust the database connection pool, causing a complete service outage for all users. Because the crash is triggered inside a database transaction, partial state may also be left in an inconsistent condition. Impact is **high**: full service unavailability with no self-recovery until the process is restarted.

### Likelihood Explanation
The attacker only needs a valid account and one transaction they created — both are normal, unprivileged product flows. No admin access, no leaked secrets, and no special timing is required. The attack is deterministic and repeatable. Likelihood is **high**.

### Recommendation
Enforce a maximum nesting depth on the incoming `CreateTransactionApproverDto` before any recursive processing begins. A depth of 5–10 levels is more than sufficient for any legitimate use case. Add a `depth` counter to `createApprover` and throw a `BadRequestException` when it exceeds the cap:

```typescript
const MAX_APPROVER_DEPTH = 10;

const createApprover = async (
  dtoApprover: CreateTransactionApproverDto,
  depth = 0,
) => {
  if (depth > MAX_APPROVER_DEPTH)
    throw new Error('Approver tree exceeds maximum allowed depth');
  // ...existing logic...
  for (const nested of dtoApprover.approvers ?? []) {
    await createApprover({ ...nested, listId: approver.id }, depth + 1);
  }
};
```

Additionally, add a DTO-level validation (e.g., a custom class-validator decorator) so the depth check is enforced at the HTTP layer before the service is even invoked.

### Proof of Concept

```
POST /api/transactions/{transactionId}/approvers
Authorization: Bearer <valid_user_token>
Content-Type: application/json

{
  "approversArray": [
    {
      "threshold": 1,
      "approvers": [
        {
          "threshold": 1,
          "approvers": [
            {
              "threshold": 1,
              "approvers": [
                // ... repeated 10,000 times
                { "userId": 1 }
              ]
            }
          ]
        }
      ]
    }
  ]
}
```

**Expected outcome:** The Node.js process throws `RangeError: Maximum call stack size exceeded` inside the database transaction, crashing the request handler. Depending on the NestJS exception filter configuration, this may propagate as an unhandled rejection and bring down the API worker process entirely.

          /* Validate Approver's DTO */
          this.validateApprover(dtoApprover);

          /* Check if the approver already exists */
          if (await this.isNode(dtoApprover, transactionId, transactionalEntityManager))
            throw new Error(this.APPROVER_ALREADY_EXISTS);

          /* Check if the parent approver exists and has threshold */
          if (typeof dtoApprover.listId === 'number') {
            const parent = await transactionalEntityManager.findOne(TransactionApprover, {
              where: { id: dtoApprover.listId },
            });

            if (!parent) throw new Error(this.PARENT_APPROVER_NOT_FOUND);

            /* Check if the root transaction is the same */
            const root = await this.getRootNodeFromNode(
              dtoApprover.listId,
              transactionalEntityManager,
            );
            if (root?.transactionId !== transactionId)
              throw new Error(this.ROOT_TRANSACTION_NOT_SAME);
          }

          /* Check if the user exists */
          if (typeof dtoApprover.userId === 'number') {
            const userCount = await transactionalEntityManager.count(User, {
              where: { id: dtoApprover.userId },
            });

            if (userCount === 0) throw new Error(this.USER_NOT_FOUND(dtoApprover.userId));
          }

          /* Check if there are sub approvers */
          if (
            typeof dtoApprover.userId === 'number' &&
            dtoApprover.approvers &&
            dtoApprover.approvers.length > 0
          )
            throw new Error(this.ONLY_USER_OR_TREE);

          /* Check if the approver has threshold when there are children */
          if (
            dtoApprover.approvers &&
            dtoApprover.approvers.length > 0 &&
            (dtoApprover.threshold === null || isNaN(dtoApprover.threshold))
          )
            throw new Error(this.THRESHOLD_REQUIRED);

          /* Check if the approver has children when there is threshold */
          if (
            typeof dtoApprover.threshold === 'number' &&
            (!dtoApprover.approvers || dtoApprover.approvers.length === 0)
          )
            throw new Error(this.CHILDREN_REQUIRED);

          /* Check if the approver threshold is less or equal to the number of approvers */
          if (
            dtoApprover.approvers &&
            (dtoApprover.threshold > dtoApprover.approvers.length || dtoApprover.threshold === 0)
          )
            throw new Error(this.THRESHOLD_LESS_OR_EQUAL_APPROVERS(dtoApprover.approvers.length));

          const data: DeepPartial<TransactionApprover> = {
            transactionId:
              dtoApprover.listId === null || isNaN(dtoApprover.listId) ? transactionId : null,
            listId: dtoApprover.listId,
            threshold:
              dtoApprover.threshold && dtoApprover.approvers ? dtoApprover.threshold : null,
            userId: dtoApprover.userId,
          };

          if (typeof dtoApprover.userId === 'number') {
            const userApproverRecords = await this.getApproversByTransactionId(
              transactionId,
              dtoApprover.userId,
              transactionalEntityManager,
            );

            if (userApproverRecords.length > 0) {
              data.signature = userApproverRecords[0].signature;
              data.userKeyId = userApproverRecords[0].userKeyId;
              data.approved = userApproverRecords[0].approved;
            }
          }

          /* Create approver */
          const approver = transactionalEntityManager.create(TransactionApprover, data);

          /* Insert approver */
          await transactionalEntityManager.insert(TransactionApprover, approver);
          approvers.push(approver);

          /* Continue creating the three */

              const nestedApprover = { ...nestedDtoApprover, listId: approver.id };

              if (!nestedDtoApprover.approvers || nestedDtoApprover.approvers.length === 0) {
                nestedApprover.threshold = null;
              }

              await createApprover({ ...nestedDtoApprover, listId: approver.id });
