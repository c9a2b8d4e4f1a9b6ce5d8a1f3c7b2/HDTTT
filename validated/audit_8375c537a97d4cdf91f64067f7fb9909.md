## Audit Report

## Title
Unbounded Recursive Approver Tree Enables Authenticated DoS via Excessive Database Operations

## Summary
The `POST /transactions/:id/approvers` endpoint accepts an arbitrarily large and deeply nested approver tree with no depth or size limit. The server-side handler processes this tree with a recursive async function that issues multiple sequential database queries per node, all inside a single open database transaction. Any verified user who has created a transaction can exploit this to exhaust the database connection pool and degrade server availability.

## Finding Description

**`CreateTransactionApproverDto` and `CreateTransactionApproversArrayDto` impose no upper bound on array sizes.**

In `back-end/apps/api/src/transactions/dto/create-transaction-approver.dto.ts`, the `approvers` field on `CreateTransactionApproverDto` carries only `@ArrayMinSize(1)` — no `@ArrayMaxSize()`. The outer wrapper `CreateTransactionApproversArrayDto.approversArray` has no size constraint at all. [1](#0-0) 

A repository-wide grep for `ArrayMaxSize` returns zero matches, confirming no such constraint exists anywhere in the codebase.

**The `createApprover` inner recursive function issues multiple sequential DB operations per node inside a single transaction.**

In `approvers.service.ts`, `createTransactionApprovers` opens a `this.dataSource.transaction(...)` and defines an inner async function `createApprover`. For every single node in the submitted tree, it sequentially executes:

1. `this.isNode(...)` — a `COUNT` query against `transaction_approver` (line 250)
2. `transactionalEntityManager.findOne(TransactionApprover, ...)` — parent lookup if `listId` is set (line 255)
3. `this.getRootNodeFromNode(...)` — a recursive SQL CTE traversal query (line 262)
4. `transactionalEntityManager.count(User, ...)` — user existence check (line 272)
5. `this.getApproversByTransactionId(...)` — another recursive SQL CTE query (line 319)
6. `transactionalEntityManager.insert(TransactionApprover, ...)` — the write (line 336)

Then it recurses into every child node (lines 340–350), all within the same open transaction. [2](#0-1) 

The `getApproversByTransactionId` and `getRootNodeFromNode` methods both issue recursive SQL CTEs: [3](#0-2) [4](#0-3) 

**The database connection pool is configured at 2–3 connections in all deployment configurations.** [5](#0-4) [6](#0-5) 

**Observer endpoint note:** `CreateTransactionObserversDto.userIds` also lacks `@ArrayMaxSize()`, but the observer service uses a single batch `repo.save(observers)` call rather than per-item inserts, making it a lower-severity concern than the approver endpoint. [7](#0-6) [8](#0-7) 

## Impact Explanation

A single malicious POST request with a large flat `approversArray` (e.g., 1,000 entries each with `threshold: 1, approvers: [{userId: X}]`) triggers O(N) to O(N²) database round-trips within a single open PostgreSQL transaction. With a pool of only 2–3 connections, even a single such request can hold all available connections for an extended period. Two or three concurrent requests fully exhaust the pool, causing all other API operations — including legitimate transaction signing and execution — to queue or fail with connection timeout errors. This is a server-wide denial of service.

## Likelihood Explanation

Any registered and verified user can create a transaction and immediately become its creator, granting them the right to call `POST /transactions/:id/approvers`. The user throttler allows 100 requests per minute and 10 per second — sufficient to sustain the attack with a single account. [9](#0-8) 

The controller applies only authentication and verification guards, with no additional rate limiting or payload size restriction on this endpoint. [10](#0-9) 

No admin privilege is required. The attack payload is a trivially constructed JSON object.

## Recommendation

1. **Add `@ArrayMaxSize(N)` to both `CreateTransactionApproversArrayDto.approversArray` and `CreateTransactionApproverDto.approvers`** — choose a reasonable maximum (e.g., 20–50) that covers legitimate use cases. [11](#0-10) 

2. **Add a maximum recursion depth guard** inside `createApprover` — track depth and throw if it exceeds a configured limit (e.g., 5).

3. **Add `@ArrayMaxSize(N)` to `CreateTransactionObserversDto.userIds`** as a secondary hardening measure. [7](#0-6) 

4. **Apply a stricter per-endpoint throttle** on `POST /transactions/:id/approvers` using `@Throttle(...)` to limit how frequently this expensive operation can be called.

5. **Consider increasing `POSTGRES_MAX_POOL_SIZE`** in production deployments, though this is a mitigation, not a fix.

## Proof of Concept

```http
POST /transactions/1/approvers
Authorization: Bearer <valid_token>
Content-Type: application/json

{
  "approversArray": [
    { "threshold": 1, "approvers": [{ "userId": 1 }] },
    { "threshold": 1, "approvers": [{ "userId": 1 }] },
    ... (repeat 1000 times)
  ]
}
```

Each of the 1,000 entries triggers at minimum 3 DB queries (`isNode` count, `count(User)`, `getApproversByTransactionId` CTE, plus the insert), all within a single open transaction. With `POSTGRES_MAX_POOL_SIZE=3`, two concurrent requests of this shape exhaust the connection pool. Legitimate requests then fail or queue indefinitely until the long-running transactions complete.

        const createApprover = async (dtoApprover: CreateTransactionApproverDto) => {

          this.validateApprover(dtoApprover);

          /* Check if the approver already exists */
          if (await this.isNode(dtoApprover, transactionId, transactionalEntityManager))
            throw new Error(this.APPROVER_ALREADY_EXISTS);

          /* Check if the parent approver exists and has threshold */
          if (typeof dtoApprover.listId === 'number') {
            const parent = await transactionalEntityManager.findOne(TransactionApprover, {
              where: { id: dtoApprover.listId },
            });

            if (!parent) throw new Error(this.PARENT_APPROVER_NOT_FOUND);

            /* Check if the root transaction is the same */
            const root = await this.getRootNodeFromNode(
              dtoApprover.listId,
              transactionalEntityManager,
            );
            if (root?.transactionId !== transactionId)
              throw new Error(this.ROOT_TRANSACTION_NOT_SAME);
          }

          /* Check if the user exists */
          if (typeof dtoApprover.userId === 'number') {
            const userCount = await transactionalEntityManager.count(User, {
              where: { id: dtoApprover.userId },
            });

            if (userCount === 0) throw new Error(this.USER_NOT_FOUND(dtoApprover.userId));
          }

          /* Check if there are sub approvers */
          if (
            typeof dtoApprover.userId === 'number' &&
            dtoApprover.approvers &&
            dtoApprover.approvers.length > 0
          )
            throw new Error(this.ONLY_USER_OR_TREE);

          /* Check if the approver has threshold when there are children */
          if (
            dtoApprover.approvers &&
            dtoApprover.approvers.length > 0 &&
            (dtoApprover.threshold === null || isNaN(dtoApprover.threshold))
          )
            throw new Error(this.THRESHOLD_REQUIRED);

          /* Check if the approver has children when there is threshold */
          if (
            typeof dtoApprover.threshold === 'number' &&
            (!dtoApprover.approvers || dtoApprover.approvers.length === 0)
          )
            throw new Error(this.CHILDREN_REQUIRED);

          /* Check if the approver threshold is less or equal to the number of approvers */
          if (
            dtoApprover.approvers &&
            (dtoApprover.threshold > dtoApprover.approvers.length || dtoApprover.threshold === 0)
          )
            throw new Error(this.THRESHOLD_LESS_OR_EQUAL_APPROVERS(dtoApprover.approvers.length));

          const data: DeepPartial<TransactionApprover> = {
            transactionId:
              dtoApprover.listId === null || isNaN(dtoApprover.listId) ? transactionId : null,
            listId: dtoApprover.listId,
            threshold:
              dtoApprover.threshold && dtoApprover.approvers ? dtoApprover.threshold : null,
            userId: dtoApprover.userId,
          };

          if (typeof dtoApprover.userId === 'number') {
            const userApproverRecords = await this.getApproversByTransactionId(
              transactionId,
              dtoApprover.userId,
              transactionalEntityManager,
            );

            if (userApproverRecords.length > 0) {
              data.signature = userApproverRecords[0].signature;
              data.userKeyId = userApproverRecords[0].userKeyId;
              data.approved = userApproverRecords[0].approved;
            }
          }

          /* Create approver */
          const approver = transactionalEntityManager.create(TransactionApprover, data);

          /* Insert approver */
          await transactionalEntityManager.insert(TransactionApprover, approver);
          approvers.push(approver);

          /* Continue creating the three */
          if (dtoApprover.approvers) {
            for (const nestedDtoApprover of dtoApprover.approvers) {
              const nestedApprover = { ...nestedDtoApprover, listId: approver.id };

              if (!nestedDtoApprover.approvers || nestedDtoApprover.approvers.length === 0) {
                nestedApprover.threshold = null;
              }

              await createApprover({ ...nestedDtoApprover, listId: approver.id });
            }
          }
        };

        for (const approver of dto.approversArray) {
          await createApprover(approver);
