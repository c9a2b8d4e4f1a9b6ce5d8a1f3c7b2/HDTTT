### Title
`isApproved` Incorrect Rejection Logic Causes False "Rejected" Status for Threshold Approver Nodes

### Summary
The `isApproved` function in the frontend uses a falsy-fallback pattern `(approver.threshold || rejections.length)` to evaluate whether a threshold approver node is definitively rejected. This produces an incorrect result whenever `rejections.length >= threshold` but there are still enough remaining approvers to satisfy the threshold. The function returns `false` (rejected) prematurely, misleading all participants into believing the transaction is dead when it is still actionable.

### Finding Description

**Root cause — incorrect sentinel/fallback in comparison logic**

In `front-end/src/renderer/utils/sdk/index.ts`, the `isApproved` function evaluates a threshold approver node:

```typescript
