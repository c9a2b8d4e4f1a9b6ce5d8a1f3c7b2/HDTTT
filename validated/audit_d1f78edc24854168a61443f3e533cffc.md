### Title
Unbounded Sequential External Calls in `getTransactionsToSign` Enable Application-Level DoS

### Summary
`getTransactionsToSign` in `transactions.service.ts` fetches **all** non-terminal transactions from the database with no query-level limit, then sequentially `await`s an external mirror-node call (`userKeysToSign` → `computeSignatureKey`) for every transaction before pagination is applied. Because there is no per-user cap on transaction creation, any authenticated user can inflate the global transaction set and cause every subsequent call to this endpoint — by any user — to time out or severely degrade.

### Finding Description

**Root cause — no query limit before the loop:**

In `getTransactionsToSign` the `repo.find` call carries no `take` constraint:

```typescript
const transactions = await this.repo.find({
  where: whereForUser,   // status NOT IN terminal set — no user filter, no limit
  relations: ['groupItem'],
  order,
});
``` [1](#0-0) 

Every non-terminal transaction in the entire system is loaded into memory.

**Sequential external calls inside the loop:**

```typescript
for (const transaction of transactions) {
  try {
    const keysToSign = await this.userKeysToSign(transaction, user);
    if (keysToSign.length > 0) result.push({ transaction, keysToSign });
  } catch (error) {
    console.log(error);
  }
}
``` [2](#0-1) 

`userKeysToSign` calls `transactionSignatureService.computeSignatureKey`, which resolves account/node keys via mirror-node HTTP calls. Each iteration is a sequential `await`; N transactions produce N sequential round-trips.
