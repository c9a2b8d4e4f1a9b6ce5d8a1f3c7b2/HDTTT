### Title
Cross-Network Signature Replay via Missing Ledger-ID Binding in Signature Verification

### Summary
The signature verification logic in `validateSignature` and `processTransactionSignatures` does not bind accepted signatures to a specific Hedera network (ledger ID). Because Hedera `TransactionBody` protobuf does not include a ledger/chain ID, a cryptographically valid signature produced for a testnet transaction is equally valid for a mainnet transaction with identical bytes. A malicious organization member can replay testnet signatures on mainnet transactions, satisfying signing thresholds without the signers' explicit consent for mainnet.

---

### Finding Description

**Root cause — `validateSignature` is network-blind**

`validateSignature` in `back-end/libs/common/src/utils/sdk/transaction.ts` verifies each signature by calling:

```ts
const signatureValid = publicKey.verify(bodyBytes, signature);
``` [1](#0-0) 

`bodyBytes` is the raw Hedera `TransactionBody` protobuf. The Hedera `TransactionBody` proto contains the payer account, node account IDs, valid start, and transaction content — but **no ledger ID or chain ID**. A signature over these bytes is therefore valid on any Hedera network that presents the same bytes.

**`importSignatures` accepts any cryptographically valid signature with no network check**

`importSignatures` in `back-end/apps/api/src/transactions/transactions.service.ts` calls `validateSignature` and then `sdkTransaction.addSignature(publicKey, map)` for every key that passes. The only guards are: user access, transaction status, and expiry. The stored `mirrorNetwork` field is never compared against the network for which the signature was actually produced. [2](#0-1) 

**`processTransactionSignatures` (signer upload path) has the same gap**

`processTransactionSignatures` in `back-end/apps/api/src/transactions/signers/signers.service.ts` calls `sdkTransaction.addSignature(publicKey, map)` with no network validation either. [3](#0-2) 

**`mirrorNetwork` is metadata only — never cryptographically bound**

When a transaction is created, `mirrorNetwork` is taken verbatim from the client DTO and stored as a plain metadata column. It is never embedded in the signed bytes and is never checked during signature ingestion. [4](#0-3) 

---

### Impact Explanation

A malicious organization member (Bob) who can:
1. Read the testnet transaction bytes (stored in the DB, retrievable via the API for any transaction Bob has access to), and
2. Create or access a mainnet transaction with identical bytes (same payer account, same valid-start, same node IDs — all of which overlap between networks for council accounts `0.0.2`, `0.0.50`, `0.0.55`),

can POST Alice's testnet signature map to the mainnet transaction's `/transactions/{id}/signers` endpoint or call `importSignatures`. The backend accepts it because `validateSignature` only checks `publicKey.verify(bodyBytes, sig)` — which passes identically on both networks.

If the replayed signature satisfies the required signing threshold, the chain service will execute the mainnet transaction (HBAR transfer, `FileUpdateTransaction` on a critical system file, `FreezeTransaction`, `NodeUpdateTransaction`, etc.) without Alice ever consenting to sign for mainnet. [5](#0-4) 

---

### Likelihood Explanation

- **Attacker profile**: malicious registered organization member — no admin/privileged keys required.
- **Preconditions**: (a) the organization tests transactions on testnet before mainnet (standard practice); (b) council/shared accounts (`0.0.2`, `0.0.50`, `0.0.55`) are used as payers on both networks; (c) the attacker can create or observe a mainnet transaction whose bytes match a testnet transaction that has already been signed.
- **Barrier**: the valid-start timestamp must match. This is achievable by copying the testnet transaction bytes directly into a new mainnet transaction record — a one-API-call operation for any organization member who can create transactions. [6](#0-5) 

---

### Recommendation

1. **Short term**: When ingesting signatures (both `importSignatures` and `uploadSignatureMaps`), retrieve the `mirrorNetwork` stored with the transaction and compare it against the network encoded in the submitted signature map's transaction ID context. Reject any signature map whose embedded transaction ID was not frozen against the stored `mirrorNetwork`.

2. **Medium term**: At transaction creation time, freeze the transaction with the correct `Client` (which sets the `LedgerId` on the SDK object). Store the `LedgerId` alongside the transaction bytes and enforce it during signature ingestion.

3. **Long term**: Document the organizational risk of using identical transaction bytes across testnet and mainnet, and enforce distinct payer accounts or valid-start timestamps per network to eliminate byte-level collisions.

---

### Proof of Concept

```
# Step 1 – Create a testnet transaction (payer 0.0.2, validStart T, node 0.0.3)
POST /transactions  { mirrorNetwork: "testnet", transactionBytes: <TX_BYTES>, ... }
→ { id: 10, mirrorNetwork: "testnet" }

# Step 2 – Create an identical mainnet transaction with the same TX_BYTES
POST /transactions  { mirrorNetwork: "mainnet", transactionBytes: <TX_BYTES>, ... }
→ { id: 20, mirrorNetwork: "mainnet" }

# Step 3 – Alice signs the testnet transaction (id=10) via the tool
POST /transactions/10/signers  { signatureMap: <ALICE_SIG_MAP> }
→ 201 OK

# Step 4 – Bob (malicious insider) extracts Alice's signature from the testnet
#           transaction bytes and replays it against the mainnet transaction (id=20)
POST /transactions/20/signers  { signatureMap: <ALICE_SIG_MAP> }
→ 201 OK   ← accepted; validateSignature passes because TX_BYTES are identical

# Result: mainnet transaction id=20 now carries Alice's signature.
# If threshold is met, the chain service executes it on mainnet.
```

export const validateSignature = (transaction: SDKTransaction, signatureMap: SignatureMap) => {
  const signerPublicKeys: PublicKey[] = [];

  const { rowLength, nodeAccountIdRow, transactionIdCol } =
    getSignedTransactionsDimensions(transaction);

  for (const [nodeAccountId, transactionIds] of signatureMap._map) {
    for (const [transactionId, publicKeys] of transactionIds._map) {
      for (const [publicKeyDer, signature] of publicKeys._map) {
        const publicKey = PublicKey.fromString(publicKeyDer);
        const publicKeyHex = publicKey.toStringRaw();

        const alreadySigned =
          transaction._signerPublicKeys.has(publicKeyHex) ||
          transaction._signerPublicKeys.has(publicKeyDer);

        if (!alreadySigned) {
          const row = nodeAccountIdRow[nodeAccountId];
          const col = transactionIdCol[transactionId];

          const bodyBytes = transaction._signedTransactions.get(col * rowLength + row).bodyBytes;

          const signatureValid = publicKey.verify(bodyBytes, signature);

          if (signatureValid) {
            signerPublicKeys.push(publicKey);
          } else {
            throw new Error('Invalid signature');
          }
        }
      }
    }
  }

  return signerPublicKeys;
};

  private async validateAndProcessSignatures(
    dto: UploadSignatureMapDto[],
    user: User,
    transactionMap: Map<number, Transaction>,
    signersByTransaction: Map<number, Set<number>>
  ) {
    // Build user key lookup once
    const userKeyMap = new Map<string, UserKey>();
    for (const key of user.keys) {
      userKeyMap.set(key.publicKey, key);
    }

    return Promise.all(
      dto.map(async ({ id, signatureMap: map }) => {
        try {
          const transaction = transactionMap.get(id);
          if (!transaction) return { id, error: ErrorCodes.TNF };

          // Validate transaction status
          const statusError = this.validateTransactionStatus(transaction);
          if (statusError) return { id, error: statusError };

          // Process signatures
          const { sdkTransaction, userKeys, isSameBytes } = await this.processTransactionSignatures(
            transaction,
            map,
            userKeyMap,
            signersByTransaction.get(id) || new Set()
          );

          return {
            id,
            transaction,
            sdkTransaction,
            userKeys,
            isSameBytes,
            error: null,
          };
        } catch (err) {
          console.error(`[TX ${id}] Error:`, err.message);
          return { id, error: err.message };
        }
      })
    );
  }

  private async processTransactionSignatures(
    transaction: Transaction,
    map: SignatureMap,
    userKeyMap: Map<string, UserKey>,
    existingSignerIds: Set<number>
  ) {
    let sdkTransaction = SDKTransaction.fromBytes(transaction.transactionBytes);

    const userKeys: UserKey[] = [];
    const processedRawKeys = new Set<string>();

    // To explain what is going on here, we need to understand how sdkTransaction.addSignature works.
    // The addSignature method will go through each inner transaction, then go through the map
    // and pull the signatures for the supplied public key belonging to that inner transaction
    // (denoted by the node and transaction id), add the signatures to the inner transactions.
    // So we need to go through the map and get each unique publicKey and call addSignature one time
    // per key.
    for (const nodeMap of map.values()) {
      for (const txMap of nodeMap.values()) {
        for (const publicKey of txMap.keys()) {
          const raw = publicKey.toStringRaw();

          // Skip duplicates across node/tx maps, and already-processed keys
          if (processedRawKeys.has(raw)) continue;
          processedRawKeys.add(raw);

          // Look up key (raw first, then DER)
          let userKey = userKeyMap.get(raw);
          if (!userKey) {
            userKey = userKeyMap.get(publicKey.toStringDer());
          }
          if (!userKey) throw new Error(ErrorCodes.PNY);

          // Only add the signature once per unique key
          sdkTransaction = sdkTransaction.addSignature(publicKey, map);

          // Only return "new" signers (not already persisted)
          if (!existingSignerIds.has(userKey.id)) {
            userKeys.push(userKey);
          }
        }
      }
    }

    // Finally, compare the resulting transaction bytes to see if any signatures were actually added
    const isSameBytes = Buffer.from(sdkTransaction.toBytes()).equals(
      transaction.transactionBytes
    );

    return { sdkTransaction, userKeys, isSameBytes };
  }
