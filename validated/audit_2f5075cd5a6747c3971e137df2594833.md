## Audit Report

## Title
Unguarded Clipboard Exposure of Full Recovery Phrase and Private Keys Allows Silent Exfiltration by Co-resident Malicious Applications

## Summary
The Hedera Transaction Tool desktop application writes the complete 24-word recovery phrase and decrypted private keys to the system clipboard in plaintext across three independent code paths, with no security warning, no clipboard-clearing timer, and no data splitting. Any co-resident application with clipboard read access can silently exfiltrate these secrets the moment the user clicks "Copy."

## Finding Description

**Path 1 — Account Setup, Recovery Phrase Copy (`Generate.vue`):**
`handleCopyRecoveryPhrase()` joins all 24 mnemonic words with `', '` and writes the result directly to the system clipboard via `navigator.clipboard.writeText()`. Only a success toast is shown; no warning about clipboard risks is presented. [1](#0-0) 

**Path 2 — Migration Summary, Recovery Phrase Copy (`Summary.vue`):**
`copyRecoveryPhrase()` joins `user.recoveryPhrase?.words` with `', '` and writes the full phrase to the clipboard. No warning is shown. [2](#0-1) 

**Path 3 — Settings Keys Tab, Private Key Copy (`KeysTab.vue`):**
`handleCopy(text, message)` is a generic clipboard writer that calls `navigator.clipboard.writeText(text)` with no guards. It is invoked directly with the decrypted private key string from the `decryptedKeys` array. [3](#0-2) [6](#0-5) 

In all three cases the full secret is placed into the OS-level clipboard as a single plaintext string with no expiry, no user warning, and no data splitting.

## Impact Explanation
The clipboard is a shared OS resource readable by any application with the appropriate permission. On macOS, Universal Clipboard automatically syncs clipboard contents to all devices signed into the same iCloud account without any additional attacker action. A malicious application co-resident on the user's machine (trojanized utility, browser extension with clipboard access, or any app granted clipboard permission) can silently read the full mnemonic or private key the moment the user clicks "Copy."

Possession of the 24-word recovery phrase gives an attacker the ability to derive all keys and take full control of any associated Hedera accounts, including draining funds. Possession of a raw private key gives equivalent signing authority over the corresponding account.

**Impact: Critical** — direct theft of cryptographic key material enabling complete wallet takeover.

## Likelihood Explanation
The copy buttons are prominently placed in the normal user workflow (account setup, migration summary, and the settings keys tab). Users are expected to click them. On macOS, Universal Clipboard is enabled by default for all iCloud users, meaning the secret automatically propagates to every linked device. On any OS, clipboard-monitoring malware or a malicious Electron plugin can read the value synchronously. No privilege escalation or physical access is required.

**Likelihood: Medium** — the attack requires a malicious co-resident app or a linked iCloud device, but the user action (clicking Copy) is routine and the clipboard is a well-known exfiltration channel.

## Recommendation
1. **Pre-copy warning dialog**: Before writing sensitive key material to the clipboard, display a modal warning the user that the data will be placed in the system clipboard and may be accessible to other applications.
2. **Clipboard clearing timer**: After writing to the clipboard, schedule an automatic clipboard clear (e.g., after 30–60 seconds) using `navigator.clipboard.writeText('')`.
3. **Data splitting (HAL-31 pattern)**: For the recovery phrase, consider splitting the phrase across multiple copy operations or requiring the user to copy individual words, reducing the value of a single clipboard intercept.
4. **Electron clipboard isolation**: Evaluate whether Electron's `safeStorage` or IPC-based clipboard handling can be used to limit clipboard exposure to the renderer process.

## Proof of Concept
1. Open the Hedera Transaction Tool and navigate to Account Setup → Generate Recovery Phrase.
2. Click "Generate" to produce a 24-word mnemonic.
3. Click the "Copy" button — `handleCopyRecoveryPhrase()` executes `navigator.clipboard.writeText(words.value.join(', '))`.
4. From any co-resident application (e.g., a simple Node.js script using `clipboardy`, or a Python script using `pyperclip`), read the clipboard contents immediately after step 3.
5. The full 24-word mnemonic is returned in plaintext with no delay or expiry.
6. Repeat steps 1–5 for the Migration Summary "Copy" button (`copyRecoveryPhrase()` in `Summary.vue`) and the Settings → Keys tab private key copy icon (`handleCopy()` in `KeysTab.vue`) to confirm all three paths.
