### Title
Sensitive Cryptographic Material (Mnemonic Phrase and Private Key) Written to System Clipboard Without Safeguards

### Summary
The Hedera Transaction Tool allows users to copy their full 24-word mnemonic recovery phrase and decrypted private keys directly to the system clipboard via `navigator.clipboard.writeText()`. No warning is displayed, no auto-clear timer is set, and no partial masking is applied. Any process on the same machine with clipboard read access (malware, clipboard manager, browser extension) can silently exfiltrate the complete secret, leading to full account compromise.

### Finding Description

**Location 1 — Full mnemonic phrase copy (account setup flow)**

`front-end/src/renderer/pages/AccountSetup/components/Generate.vue`, `handleCopyRecoveryPhrase` at line 92–94:

```js
const handleCopyRecoveryPhrase = () => {
  navigator.clipboard.writeText(words.value.join(', '));
  toastManager.success('Recovery phrase copied');
};
```

This function is bound to a "Copy" button rendered unconditionally whenever `words.filter(w => w).length !== 0` (line 185–191). The entire 24-word BIP-39 mnemonic is written to the OS clipboard as a comma-separated string. No warning is shown, no clipboard clear is scheduled, and no partial masking is applied. [1](#0-0) [2](#0-1) 

**Location 2 — Decrypted private key copy (Settings → Keys tab)**

`front-end/src/renderer/pages/Settings/components/KeysTab/KeysTab.vue`, `handleCopy` at line 134–137, invoked for the private key at lines 362–369:

```js
const handleCopy = (text: string, message: string) => {
  navigator.clipboard.writeText(text);
  toastManager.success(message);
};
```

After the user decrypts a key pair (via `handleShowPrivateKey` → `decrypt` → `decryptPrivateKey`), a copy icon appears next to the revealed private key. Clicking it calls `handleCopy(decryptedKeys.find(...).decrypted, 'Private Key copied successfully')`, writing the raw private key string to the clipboard. Again, no warning, no auto-clear, no masking. [3](#0-2) [4](#0-3) 

**Root cause:** Both code paths call `navigator.clipboard.writeText()` with the complete secret and take no further action to limit exposure time or warn the user.

### Impact Explanation

The mnemonic phrase is the root secret from which all key pairs are derived (`restorePrivateKey` in `keyPairService.ts` lines 111–131). Possession of the mnemonic gives an attacker the ability to re-derive every private key at every index, enabling complete and permanent takeover of all associated Hedera accounts. The private key itself, when copied, directly authorizes transaction signing. Either secret being captured from the clipboard results in irreversible asset loss with no recovery path for the victim. [5](#0-4) 

### Likelihood Explanation

The attack requires a co-resident process on the victim's machine capable of reading clipboard contents — a capability present in clipboard manager utilities (common on desktop systems), browser extensions with clipboard permissions, and any malware with user-level privileges. No physical access is required; no privileged OS access is required. The copy buttons are part of the normal, documented user workflow (account setup and key management), so the user is likely to use them. The clipboard is not cleared after any interval, so the window of exposure is unbounded.

### Recommendation

1. **Remove or gate the copy buttons for secrets.** Do not provide a one-click copy for the full mnemonic or raw private key. If copy is retained for UX reasons, copy only a partial value (e.g., first 4 words) so the complete secret is never in the clipboard.
2. **Auto-clear the clipboard.** If a full copy is kept, schedule `navigator.clipboard.writeText('')` after a short interval (e.g., 30 seconds) immediately after writing.
3. **Display a security warning.** Before writing to the clipboard, show a modal warning the user that clipboard contents may be accessible to other applications and advising them to clear it after use.
4. **Prefer manual transcription UX.** Display the mnemonic in a read-only, selectable-text field and instruct users to write it down, rather than providing a copy button.

### Proof of Concept

1. Launch the Hedera Transaction Tool and begin account setup.
2. Click **Generate** to produce a new 24-word mnemonic. The phrase is displayed.
3. Click the **Copy** button (rendered at `Generate.vue` line 185–191).
4. `handleCopyRecoveryPhrase` executes `navigator.clipboard.writeText(words.value.join(', '))` — the full mnemonic is now in the OS clipboard.
5. From any co-resident process (e.g., `xclip -o`, a clipboard manager, or a browser extension with `clipboardRead` permission), read the clipboard: the complete 24-word mnemonic is returned in plaintext.
6. Use the mnemonic with `restorePrivateKey(words, '', index, 'ED25519')` to derive private keys for any index, then sign and submit Hedera transactions as the victim.

The same steps apply to the private key copy in **Settings → Keys tab**: decrypt a key, click the copy icon, and the raw private key string is in the clipboard indefinitely.

```typescript
export const restorePrivateKey = async (
  words: string[],
  passphrase: string,
  keyIndex: number,
  type: 'ECDSA' | 'ED25519',
) => {
  if (words.length !== 24) {
    throw Error('Invalid Recovery Phrase');
  }

  const mnemonic = await Mnemonic.fromWords(words);

  switch (type) {
    case 'ED25519':
      return mnemonic.toStandardEd25519PrivateKey(passphrase, keyIndex);
    case 'ECDSA':
      return mnemonic.toStandardECDSAsecp256k1PrivateKey(passphrase, keyIndex);
    default:
      throw new Error('Key type not supported');
  }
};
```
