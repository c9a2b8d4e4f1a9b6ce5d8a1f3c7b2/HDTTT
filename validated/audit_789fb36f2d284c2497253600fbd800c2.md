### Title
Stored XSS via Unsanitized `v-html` Rendering of Transaction Memo and Description in Organization Mode

### Summary
`CreateTransactionGroup.vue` renders `groupItem.description`, `Transaction.fromBytes(groupItem.transactionBytes).transactionMemo`, and the output of `makeTransfer()` directly into the DOM via Vue's `v-html` directive without any HTML sanitization. In organization mode, a malicious organization member can craft a Hedera transaction with an HTML/JavaScript payload in the `transactionMemo` field (or description), store it on the shared backend, and have it execute in any other organization member's Electron renderer when they open the Create Transaction Group page.

### Finding Description

In `CreateTransactionGroup.vue`, lines 421–429, the following `v-html` binding renders untrusted data directly into the DOM:

```html
v-html="
  groupItem.type == 'Transfer Transaction'
    ? makeTransfer(index)
    : groupItem.description != ''
      ? groupItem.description
      : Transaction.fromBytes(groupItem.transactionBytes).transactionMemo
        ? Transaction.fromBytes(groupItem.transactionBytes).transactionMemo
        : createTransactionId(groupItem.payerAccountId, groupItem.validStart)
"
``` [1](#0-0) 

Three distinct injection surfaces exist:

1. **`groupItem.description`** — a user-supplied string stored in the organization backend, rendered raw via `v-html`.
2. **`Transaction.fromBytes(groupItem.transactionBytes).transactionMemo`** — the `transactionMemo` field is extracted from serialized Hedera transaction bytes. Any organization member can craft a transaction with an arbitrary memo string. These bytes are stored in the backend and synced to all members.
3. **`makeTransfer(index)`** — calls `formatHbarTransfers` on transfer transaction data; if that utility returns any attacker-influenced string, it is also injected unsanitized.

None of these values are passed through an HTML sanitizer (e.g., DOMPurify) before being set as `innerHTML` by Vue's `v-html`.

The analog to the external report is direct: the external report shows `simulationResult` items passed to `text()` (which renders Markdown/control characters) without sanitization. Here, `transactionMemo` and `description` are passed to `v-html` (which renders raw HTML) without sanitization.

A second, lower-severity instance exists in `AppStepper.vue` where `item.bubbleLabel` is also rendered via `v-html` without sanitization:

```javascript
return `<i class="bi bi-${item.bubbleIcon}"></i>`;
// and item.bubbleLabel is also rendered via v-html
``` [2](#0-1) [3](#0-2) 

### Impact Explanation

This is a **stored cross-site scripting (XSS)** vulnerability in an Electron desktop application. The impact depends on the Electron security configuration:

- If `nodeIntegration` is enabled in the renderer, XSS escalates to **full remote code execution** on the victim's machine (arbitrary shell commands via `require('child_process')`).
- Even with `nodeIntegration` disabled and `contextIsolation` enabled, the attacker can steal session tokens, private keys cached in the renderer's memory/stores, or manipulate the UI to trick the victim into signing malicious transactions.

The Hedera Transaction Tool manages cryptographic keys and signs financial transactions on the Hedera network. Compromise of the renderer process can directly lead to **unauthorized transaction signing and asset theft**.

### Likelihood Explanation

The attacker only needs to be a **normal organization member** — no privileged access is required. Creating a Hedera transaction with a crafted `transactionMemo` (e.g., `<img src=x onerror=alert(1)>`) is a standard SDK operation. The payload is stored in the organization backend and automatically rendered for every other member who opens the Create Transaction Group page. This is a realistic, low-barrier attack.

### Recommendation

Replace `v-html` with safe text interpolation (`{{ }}`) wherever the content is not intentionally trusted HTML. For cases where HTML rendering is genuinely needed (e.g., `makeTransfer`), sanitize the output with a library such as DOMPurify before passing it to `v-html`:

```javascript
import DOMPurify from 'dompurify';
// ...
v-html="DOMPurify.sanitize(makeTransfer(index))"
```

For `transactionMemo` and `description`, these are plain-text fields and should use `{{ }}` interpolation, not `v-html`.

### Proof of Concept

1. Attacker joins an organization as a normal member.
2. Attacker creates a Hedera `TransferTransaction` (or any transaction type) with `transactionMemo` set to:
   ```
   <img src=x onerror="alert(document.cookie)">
   ```
3. Attacker adds this transaction to a transaction group and saves it to the organization backend.
4. Victim (another organization member) opens the **Create Transaction Group** page in their Electron client.
5. The `v-html` binding at line 426 of `CreateTransactionGroup.vue` renders the memo as raw HTML.
6. The `onerror` handler fires, executing arbitrary JavaScript in the victim's Electron renderer process. [4](#0-3)
