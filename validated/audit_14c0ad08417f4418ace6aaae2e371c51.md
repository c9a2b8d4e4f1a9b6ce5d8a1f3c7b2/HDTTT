Audit Report

## Title
Any Authenticated User Can Retrieve All Historical Transactions Without Authorization

## Summary
`GET /transactions/history` in `TransactionsController` accepts no user identity and delegates to `getHistoryTransactions` in `TransactionsService`, which applies no user-scoped filter. Any holder of a valid JWT can retrieve the complete historical transaction record of the entire organization.

## Finding Description

**Root cause — controller passes no user to the service:**

`getHistoryTransactions` in the controller declares no `@GetUser()` parameter and forwards no user identity to the service. [1](#0-0) 

**Root cause — service applies no user-scoped WHERE clause:**

`getHistoryTransactions` in the service filters only on `status` (via `getHistoryStatusWhere`). There is no condition on `signers.userId`, `observers.userId`, `creatorKey.userId`, or any other user-identity field. [2](#0-1) 

**Contrast with `getTransactions`**, which correctly scopes results to the requesting user across three relations: [3](#0-2) 

**Class-level guards only verify JWT validity**, not which data is returned: [4](#0-3) 

**The endpoint is actively used by the front-end** — it carries no `/* NO LONGER USED BY FRONT-END */` comment, unlike the sibling endpoints at lines 110, 157, and 199: [5](#0-4) [6](#0-5) 

## Impact Explanation
Any registered user can enumerate the full operational history of the organization: transaction types, statuses (`EXECUTED`, `FAILED`, `EXPIRED`, `CANCELED`, `ARCHIVED`), `transactionId`, `mirrorNetwork`, `validStart`, `executedAt`, and group membership. For an organization managing treasury or node operations on Hedera, this constitutes a material confidential data breach. No further exploitation steps are required beyond a single authenticated HTTP GET.

## Likelihood Explanation
Likelihood is high. The only precondition is a valid JWT — the lowest possible privilege level. The endpoint is actively consumed by the front-end `History.vue` page, meaning it is reachable in normal application flow. No special role, no leaked credentials, and no internal network access are required.

## Recommendation
Pass the authenticated user into `getHistoryTransactions` and add user-scoped `WHERE` conditions mirroring those in `getTransactions`:

```typescript
// Controller
getHistoryTransactions(
  @GetUser() user: User,          // add this
  @PaginationParams() paginationParams: Pagination,
  ...
) {
  return this.transactionsService.getHistoryTransactions(user, paginationParams, filter, sort);
}

// Service
async getHistoryTransactions(user: User, ...) {
  const whereForUser = [
    { ...statusWhere, signers: { userId: user.id } },
    { ...statusWhere, observers: { userId: user.id } },
    { ...statusWhere, creatorKey: { userId: user.id } },
  ];
  // also include approver sub-query as in getTransactions
}
```
