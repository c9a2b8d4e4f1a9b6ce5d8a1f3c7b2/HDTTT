Audit Report

## Title
Missing Array Bounds Check in CSV Row Parser Causes Silent Partial Import and Uninformative Failure

## Summary
`handleImportCsv()` in `ImportCSVController.vue` accesses `rowInfo[1]` on every transaction data row without first verifying the parsed row contains at least two columns. A malformed row causes a `TypeError` that is caught by a generic outer handler which shows only `"Failed to import CSV file"` and does **not** call `transactionGroup.clearGroup()`, leaving the group in a silently partial state that the user may subsequently submit.

## Finding Description

**Root cause — unguarded indexed access:**

The CSV row is parsed via a regex match that returns `[]` on a non-matching row: [1](#0-0) 

In the `default` branch (transaction data rows), `rowInfo[1]` is immediately dereferenced with `.replace()`: [2](#0-1) 

There is no `rowInfo.length >= 2` guard. If a data row has fewer than two comma-separated fields, `rowInfo[1]` is `undefined` and `undefined.replace(...)` throws:
```
TypeError: Cannot read properties of undefined (reading 'replace')
```

**Failure path — generic catch, no group cleanup:**

The `TypeError` propagates to the outer `try/catch`, which only shows a toast and logs — `transactionGroup.clearGroup()` is never called: [3](#0-2) 

By contrast, `clearGroup()` **is** called in the receiver-account-lookup failure path: [4](#0-3) 

The `finally` block then fires `props.callback()` unconditionally: [5](#0-4) 

**The only existing bounds check is for the optional index-3 field:** [6](#0-5) 

This confirms the developer applied the pattern for the optional field but not for the mandatory fields at indices 1 and 2.

**Header rows are also unguarded** — `rowInfo[1]` is read for `sender account`, `fee payer account`, `sending time`, `transaction fee`, `transaction valid duration`, and `memo` without length checks: [7](#0-6) 

**Partial group state persists:** `handleOnFileChanged` in the parent clears the group before import begins: [8](#0-7) 

Any rows successfully processed before the bad row are already committed via `transactionGroup.addGroupItem()` and remain in the store after the error.

## Impact Explanation

A user who imports a CSV where any transaction data row is missing the amount column will:

1. Receive only the generic toast `"Failed to import CSV file"` with no indication of which row failed or that the group is incomplete.
2. Have a transaction group that is **silently partially populated** — all rows processed before the bad row are already in `transactionGroup.groupItems`.
3. If the user does not notice the partial state and proceeds to click **Sign & Submit** or **Save Group**, only a subset of the intended HBAR transfers will be executed on the Hedera network, causing unintended financial outcomes (some recipients receive funds while others do not, or the sender's balance is partially debited without completing the full intended distribution).

## Likelihood Explanation

Any user who imports a CSV file with a missing amount column — whether due to a formatting mistake, a spreadsheet export artifact, or a crafted file — will trigger this. The CSV import feature is a documented, normal workflow. The generic error message provides no guidance to diagnose or recover from the partial state, making it plausible that a user will not realize the group is incomplete before submitting.

## Recommendation

1. **Add a bounds check before accessing `rowInfo[1]`** in the `default` branch:
   ```ts
   if (rowInfo.length < 2) {
     toastManager.error(`Row ${rowIndex + 1} is malformed: missing amount column.`);
     transactionGroup.clearGroup();
     return null;
   }
   ```
2. **Call `transactionGroup.clearGroup()`** in the outer `catch` block, mirroring the receiver-account-lookup failure path at line 131.
3. **Include the row number and field name** in all error messages to give the user actionable information.
4. **Apply the same bounds check to all header-row `rowInfo[1]` accesses** (lines 65, 68, 80, 92, 97, 100, 103) to prevent silent `undefined` propagation into transaction construction.

## Proof of Concept

Create a CSV file with a valid header section followed by a data row that has only one column (no amount):

```
Transaction Description,Test Group
Sender Account,0.0.1234
Fee Payer Account,0.0.1234
Sending Time,12:00:00
Transaction Fee,200000000
Transaction Valid Duration,180
Memo,test
AccountId,Amount,Date,Memo
0.0.5678
```

Import this file via the **Import CSV** button in the Create Transaction Group page. The import will throw a `TypeError` at line 144, the catch block will display `"Failed to import CSV file"`, and any rows processed before `0.0.5678` will remain in `transactionGroup.groupItems`. The user can then click **Sign & Submit** to execute the partial group. [2](#0-1) [9](#0-8)
