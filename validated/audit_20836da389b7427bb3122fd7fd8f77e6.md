Audit Report

## Title
Authorization Bypass via Missing Key-Ownership Check in `importSignatures` — Parallel Signature-Upload Path Omits the Enforcement Present in the Primary Path

## Summary
Two HTTP endpoints both write signatures into stored transaction bytes, but only the primary path (`POST /transactions/:id/signers`) enforces that every submitted public key belongs to the authenticated user. The secondary path (`POST /transactions/signatures/import`) performs only a cryptographic validity check and accepts any key whose signature verifies over the transaction body, regardless of whether the caller owns that key.

## Finding Description

**Primary path — ownership enforced**

`SignersController.uploadSignatureMap` → `SignersService.uploadSignatureMaps` → `processTransactionSignatures`: [1](#0-0) 

`userKeyMap` is built exclusively from `user.keys` (the authenticated user's registered keys). Any public key in the submitted signature map that is not in that map causes an immediate `ErrorCodes.PNY` throw, rejecting the entire request.

**Secondary path — ownership absent**

`TransactionsController.importSignatures` → `TransactionsService.importSignatures`: [2](#0-1) 

`validateSignature` (defined in `back-end/libs/common/src/utils/sdk/transaction.ts`) only verifies that the supplied bytes are a valid ECDSA/EdDSA signature over the transaction body for the given public key: [3](#0-2) 

There is no lookup against `user.keys` and no ownership assertion anywhere in `importSignatures`. Any cryptographically valid signature for any key is accepted and written into `transactionBytes`.

**No audit trail created**

The controller's own API description confirms the design: *"No signature entities will be created."* [4](#0-3) 

The `importSignatures` service method updates only `transactionBytes`; it never inserts a `TransactionSigner` row, so the injected signature is invisible in the signing audit trail. [5](#0-4) 

**Access gate is broader than ownership**

The only pre-condition before signature injection is `verifyAccess`, which is confirmed by the test suite to be a separate, broader check (creator / signer / observer / approver) that is entirely independent of key ownership: [6](#0-5) [7](#0-6) 

## Impact Explanation
An attacker who holds any role that passes `verifyAccess` (e.g., observer) on a target transaction can:

1. Obtain raw signature bytes for a foreign key out-of-band (e.g., from a `.tx2` export file, which is a first-class application feature that embeds raw signatures).
2. Call `POST /transactions/signatures/import` with that signature map.
3. The backend writes the foreign signature into `transactionBytes` without creating a `TransactionSigner` record, making the injection invisible in the signing audit trail.
4. If the injected signature satisfies the remaining threshold, the transaction advances toward `WAITING_FOR_EXECUTION` and can be submitted to the Hedera network without the key owner having explicitly approved through the normal workflow.

This constitutes unauthorized state advancement of a multi-party transaction and circumvention of the signing-approval workflow the platform is designed to enforce.

## Likelihood Explanation
- The attacker must be an authenticated, verified user with `verifyAccess` on the target transaction; observer role qualifies.
- Valid signature bytes for a foreign key are obtainable without breaking cryptography: the `.tx2` export/import workflow is a first-class feature of the application, and exported files contain raw signatures. [8](#0-7) 

- The endpoint requires no elevated privilege and is reachable from the standard frontend client or any HTTP client.
- No rate-limiting or additional confirmation step is applied to `importSignatures`.

## Recommendation
Add a key-ownership check inside `importSignatures` analogous to the one in `processTransactionSignatures`. After `validateSignature` returns the list of `publicKeys`, verify that each key is present in `user.keys` (by raw or DER string) before calling `sdkTransaction.addSignature`. If any key is not owned by the caller, reject the entire request (or at minimum skip that key and return an error for it). This mirrors the guard already present at: [1](#0-0) 

## Proof of Concept
```
# Attacker is an observer on transaction ID 42.
# Alice (key owner) previously exported a .tx2 file; attacker has the file.

# 1. Parse the .tx2 file to extract the SignatureMap bytes (raw signatures).
# 2. Format as UploadSignatureMapDto:
POST /api/transactions/signatures/import
Authorization: Bearer <attacker_jwt>
Content-Type: application/json

[{
  "id": 42,
  "signatureMap": { /* Alice's signature map extracted from .tx2 */ }
}]

# 3. Backend calls validateSignature → signature is cryptographically valid → accepted.
# 4. Alice's signature is written into transactionBytes with no TransactionSigner row.
# 5. If threshold is now met, transaction advances to WAITING_FOR_EXECUTION.
```

export const validateSignature = (transaction: SDKTransaction, signatureMap: SignatureMap) => {
  const signerPublicKeys: PublicKey[] = [];

  const { rowLength, nodeAccountIdRow, transactionIdCol } =
    getSignedTransactionsDimensions(transaction);

  for (const [nodeAccountId, transactionIds] of signatureMap._map) {
    for (const [transactionId, publicKeys] of transactionIds._map) {
      for (const [publicKeyDer, signature] of publicKeys._map) {
        const publicKey = PublicKey.fromString(publicKeyDer);
        const publicKeyHex = publicKey.toStringRaw();

        const alreadySigned =
          transaction._signerPublicKeys.has(publicKeyHex) ||
          transaction._signerPublicKeys.has(publicKeyDer);

        if (!alreadySigned) {
          const row = nodeAccountIdRow[nodeAccountId];
          const col = transactionIdCol[transactionId];

          const bodyBytes = transaction._signedTransactions.get(col * rowLength + row).bodyBytes;

          const signatureValid = publicKey.verify(bodyBytes, signature);

          if (signatureValid) {
            signerPublicKeys.push(publicKey);
          } else {
            throw new Error('Invalid signature');
          }
        }
      }
    }
  }

  return signerPublicKeys;
};
