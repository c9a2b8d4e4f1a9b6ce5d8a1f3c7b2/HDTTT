### Title
Division by Zero in Exchange Rate Calculation Produces `Infinity`/`NaN` Price Display

### Summary

In `front-end/src/renderer/stores/storeNetwork.ts`, the `currentRate` computed property divides `cent_equivalent` by `hbar_equivalent` without validating that `hbar_equivalent` is greater than zero. If the mirror node returns `hbar_equivalent: 0` (e.g., uninitialized local node, partial/malformed API response, or a custom mirror node endpoint), JavaScript silently produces `Infinity` or `NaN`. This corrupted rate value propagates into `getDollarAmount`, causing all USD price displays in the UI to show `$Infinity` or `$NaN`, misleading users about transaction costs.

### Finding Description

**Root cause — `storeNetwork.ts` lines 46 and 51:**

```typescript
let rate = networkCurrRate.cent_equivalent / networkCurrRate.hbar_equivalent / 100;
// ...
rate = networkNextRate.cent_equivalent / networkNextRate.hbar_equivalent / 100;
``` [1](#0-0) 

Neither `hbar_equivalent` value is checked for zero before being used as a divisor. The `ExchangeRate` interface types it as a plain `number` with no lower-bound constraint: [2](#0-1) 

The `getExchangeRateSet` function fetches the raw API response and returns it with no field-level validation: [3](#0-2) 

The corrupted `currentRate` (`Infinity` / `NaN`) is then consumed by `getDollarAmount`:

```typescript
if (hbarPrice !== null) {          // passes for Infinity and NaN
    let usdAmount = hbarAmount.times(hbarPrice);   // → Infinity or NaN
    result = `$${usdAmount.decimalPlaces(fractionDigits)}`;  // → "$Infinity" or "$NaN"
}
``` [4](#0-3) 

The null-guard `hbarPrice !== null` does **not** catch `Infinity` or `NaN`, so the corrupted value flows through unchecked.

### Impact Explanation

Every USD price label rendered in the UI (transaction cost previews, balance displays) will show `$Infinity` or `$NaN` for the duration of the session. Users cannot distinguish real costs from corrupted display values, which can cause them to approve or reject transactions based on false price information. The integrity of the price display — the only USD reference point in the application — is completely broken.

### Likelihood Explanation

The application explicitly supports custom and local mirror node endpoints (`CommonNetwork.LOCAL_NODE`, and arbitrary URLs via `getClientFromMirrorNode`). A local Hedera node that has not yet initialized its exchange rate file (0.0.112) will return `hbar_equivalent: 0`. Any network error that causes a partial JSON response, or a misconfigured custom mirror node, can also produce this value. No privileged access is required; any user who points the tool at a non-production mirror node triggers the condition. [5](#0-4) 

### Recommendation

Add a zero-guard before computing the rate, mirroring the external report's `assert old > 0` pattern:

```typescript
const currentRate = computed(() => {
  if (!exchangeRateSet.value) return null;

  const timestamp = parseInt(exchangeRateSet.value.timestamp);
  const networkCurrRate = exchangeRateSet.value.current_rate;

  if (!networkCurrRate.hbar_equivalent) return null;   // guard: zero or falsy
  let rate = networkCurrRate.cent_equivalent / networkCurrRate.hbar_equivalent / 100;

  const networkNextRate = exchangeRateSet.value.next_rate;
  if (timestamp > networkCurrRate.expiration_time) {
    if (!networkNextRate.hbar_equivalent) return null; // guard: zero or falsy
    rate = networkNextRate.cent_equivalent / networkNextRate.hbar_equivalent / 100;
  }

  return rate;
});
```

Returning `null` causes `getDollarAmount` to return `''` (empty string) rather than `$Infinity`/`$NaN`, which is the safe fallback already handled by the existing `else` branch. [6](#0-5) 

### Proof of Concept

1. Start a local Hedera node that has not yet loaded the exchange rate system file (`0.0.112`), so its mirror node returns `{ "current_rate": { "hbar_equivalent": 0, "cent_equivalent": 0, ... }, ... }`.
2. Open the Hedera Transaction Tool and set the network to `LOCAL_NODE` (or any custom mirror node URL pointing to the above node).
3. `setNetwork` calls `getExchangeRateSet`, which returns the response with `hbar_equivalent: 0`.
4. The `currentRate` computed property evaluates `0 / 0 / 100` → `NaN`.
5. Any component that calls `getDollarAmount(currentRate, amount)` receives `$NaN` as the displayed price.
6. All USD price labels in the UI now show `$NaN` for the remainder of the session.

export const getDollarAmount = (hbarPrice: number, hbarAmount: BigNumber) => {
  const fractionDigits = 5;

  let result: string;

  if (hbarPrice !== null) {
    const resolution = Math.pow(10, -fractionDigits);
    let usdAmount = hbarAmount.times(hbarPrice);
    if (usdAmount.isGreaterThan(0) && usdAmount.isLessThan(+resolution)) {
      usdAmount = new BigNumber(resolution);
    } else if (usdAmount.isGreaterThan(-resolution) && usdAmount.isLessThan(0)) {
      usdAmount = new BigNumber(-resolution);
    }
    result = `$${usdAmount.decimalPlaces(fractionDigits)}`;
  } else {
    result = '';
  }
  return result;
};
