Audit Report

## Title
Recovery Phrase Validation Inconsistency Allows Silent Import of Non-Restorable 12-Word Mnemonics

## Summary
`validateMnemonic` delegates entirely to the Hedera SDK's `Mnemonic.fromWords()` with no word-count guard, while `restorePrivateKey` hard-codes a `words.length !== 24` check. A user who pastes a valid 12-word BIP-39 mnemonic receives no error — the phrase is stored — but every subsequent attempt to derive a key from it throws `"Invalid Recovery Phrase"`. The user believes their keys are backed up, but they are not recoverable.

## Finding Description

**Root cause — split validation logic:**

`validateMnemonic` in `front-end/src/renderer/services/keyPairService.ts` delegates entirely to the SDK with no word-count check: [1](#0-0) 

`restorePrivateKey` in the same file enforces exactly 24 words, but only at key-derivation time: [2](#0-1) 

**Trigger path 1 — paste in `Import.vue`:**

The paste handler splits clipboard text, filters empty tokens, and slices to `WORD_COUNT` (24). If the user pastes a 12-word phrase, `mnemonic` is a 12-element array: [3](#0-2) 

The watcher then calls `validateMnemonic(normalizedWords)` — which returns `true` for a valid 12-word BIP-39 phrase — and proceeds to call `user.setRecoveryPhrase(normalizedWords)`: [4](#0-3) 

`setRecoveryPhrase` calls `createRecoveryPhrase`, which again calls `Mnemonic.fromWords` with no word-count guard — succeeds, stores the 12-word phrase: [5](#0-4) [6](#0-5) 

**Trigger path 2 — data migration:**

`decryptMigrationMnemonic` returns `words.split(' ')` with no length validation. A legacy tool that stored a 12-word mnemonic produces a 12-element array, which is passed directly to `createRecoveryPhrase`: [7](#0-6) [8](#0-7) 

**Failure at key restoration:**

Every code path that derives a private key — `RestoreStep.vue`, `RestoreMissingKeys.vue`, `BeginKeysImport.vue`, `SetupPersonal.vue` — calls `restorePrivateKey`, which throws immediately for any non-24-word phrase: [9](#0-8) [10](#0-9) [11](#0-10) 

## Impact Explanation
A user who imports a valid 12-word BIP-39 mnemonic sees no error during import. The phrase is stored and displayed as valid. When the user later attempts to restore or derive any key pair, every call to `restorePrivateKey` throws `"Invalid Recovery Phrase"`. The user has no working key backup. If the original private key is lost, the funds/account are permanently unrecoverable. This constitutes a permanent lock/freeze of user state caused by a reachable user action.

## Likelihood Explanation
The paste handler in `Import.vue` is an explicitly supported workflow. BIP-39 12-word mnemonics are the default for many wallets (MetaMask, Trust Wallet, etc.). A user migrating from such a wallet, or from the legacy Hedera Transaction Tools desktop app (which the migration feature explicitly targets), will naturally attempt to paste their existing 12-word phrase. No privilege is required; the user acts entirely within normal product flows.

## Recommendation
Add a word-count check to `validateMnemonic` so it rejects any phrase that is not exactly 24 words:

```ts
export const validateMnemonic = async (words: string[]) => {
  if (words.length !== 24) return false;
  try {
    words = words.map(w => w.toLocaleLowerCase());
    await Mnemonic.fromWords(words);
    return true;
  } catch {
    return false;
  }
};
```

Additionally, add a length guard in `decryptMigrationMnemonic` (or in `DecryptRecoveryPhrase.vue` before calling `createRecoveryPhrase`) to reject any decrypted mnemonic that is not 24 words, surfacing a clear error to the user rather than silently storing an unusable phrase.

## Proof of Concept

1. Open the app and navigate to the Import Recovery Phrase screen.
2. Copy any valid 12-word BIP-39 mnemonic (e.g., from MetaMask) to the clipboard.
3. Paste into any word input field in `Import.vue`.
4. Observe: `handlePaste` produces a 12-element array; `validateMnemonic` returns `true`; the watcher calls `user.setRecoveryPhrase` with 12 words; no error is shown and the phrase appears accepted.
5. Proceed to restore or derive a key pair (e.g., via `RestoreStep.vue`).
6. Observe: `restorePrivateKey` immediately throws `"Invalid Recovery Phrase"` because `words.length` is 12, not 24.
7. The stored phrase cannot be used to derive any key, yet the UI gave no warning during import.

export async function decryptMigrationMnemonic(password: string): Promise<string[] | null> {
  const content = await fs.promises.readFile(getPropertiesPath(), { encoding: 'utf-8', flag: 'r' });
  const parsedContent = parseUserProperties(content);

  const token = parsedContent.hash;
  if (!token) throw Error('No hash found at location');

  const isLegacy = parsedContent.legacy;

  let words: string | null;
  if (isLegacy) {
    words = await decryptLegacyMnemonic(getMnemonicPath(), password);
  } else {
    words = await decryptMnemonic(getMnemonicPath(), token, password);

  return null;
}
