Audit Report

## Title
Missing Cryptographic Signature Validity Check in `uploadSignatureMaps` Allows Malicious Signer to Corrupt Transaction State

## Summary
The `POST /transactions/:transactionId?/signers` endpoint (`SignersService.uploadSignatureMaps`) accepts signature maps and writes them into stored Hedera transaction bytes without performing any cryptographic verification of the signature data. The only input guard is a non-zero byte-length check in the `IsSignatureMap` decorator. A legitimate signer can submit a signature map containing their own registered public key paired with arbitrary garbage bytes; the SDK's `addSignature` method blindly embeds those bytes, the changed transaction bytes are persisted to the database, and a `TransactionSigner` record is created marking the key as having signed. When the transaction is later submitted to the Hedera network it will fail with `INVALID_SIGNATURE`.

## Finding Description

**Two signature-upload paths exist with inconsistent validation.**

**Path 1 — `POST /transactions/signatures/import` (`TransactionsService.importSignatures`):**

Before calling `addSignature`, this path calls `validateSignature`:

```
const { data: publicKeys, error } = safe<PublicKey[]>(
  validateSignature.bind(this, sdkTransaction, map),
);
if (error) throw new BadRequestException(ErrorCodes.ISNMPN);
``` [1](#0-0) 

`validateSignature` calls `publicKey.verify(bodyBytes, signature)` for every entry in the map and throws `'Invalid signature'` on failure — a proper cryptographic check. [2](#0-1) 

**Path 2 — `POST /transactions/:transactionId?/signers` (`SignersService.uploadSignatureMaps`):**

This path goes through `validateAndProcessSignatures` → `processTransactionSignatures`. There is **no call to `validateSignature` anywhere in `signers.service.ts`** (confirmed by grep). The only check performed is that the public key in the map belongs to the authenticated user; then `addSignature` is called unconditionally:

```typescript
sdkTransaction = sdkTransaction.addSignature(publicKey, map);
``` [3](#0-2) 

The upstream `IsSignatureMap` decorator only rejects a signature if its decoded byte length is exactly zero:

```typescript
if (decodedSignature.length === 0) {
  throw new BadRequestException(ErrorCodes.ISNMP);
}
``` [4](#0-3) 

Any non-empty byte sequence — `0xdeadbeef`, 1 byte of `0x00`, 63 bytes of random data — passes this check and is accepted into the `SignatureMap`.

After `addSignature` embeds the invalid bytes, `isSameBytes` is `false` (the transaction bytes changed), so `persistSignatureChanges` writes the corrupted bytes to the database and inserts a `TransactionSigner` record: [5](#0-4) 

## Impact Explanation
A malicious signer submits an invalid signature via `POST /transactions/:id/signers`. The transaction's stored bytes are overwritten with the corrupted signature. If the signing threshold is subsequently met by other legitimate signers, the transaction advances to `WAITING_FOR_EXECUTION`. The chain service submits it to Hedera, which rejects it with `INVALID_SIGNATURE`. If the transaction's `validStart` window expires before anyone detects and corrects the state, the transaction is permanently unexecutable. For high-value or time-sensitive transactions (e.g., `NodeCreateTransaction`, `FileUpdateTransaction` for network governance files), this constitutes permanent, unrecoverable loss of the transaction's intended effect.

## Likelihood Explanation
The attacker only needs to be an authenticated user with a registered key that is listed as a signer on the target transaction — a normal, non-privileged role. No admin access, no leaked credentials, and no special tooling are required. The attack is a single crafted HTTP POST request. The `IsSignatureMap` decorator's `length === 0` guard is trivially bypassed by supplying any non-empty hex string (e.g., `"0xdeadbeef"`).

## Recommendation
Add a call to `validateSignature` inside `processTransactionSignatures` in `signers.service.ts`, mirroring the pattern already used in `importSignatures`:

```typescript
// In processTransactionSignatures, before calling addSignature:
const { data: validKeys, error } = safe<PublicKey[]>(
  validateSignature.bind(null, sdkTransaction, map),
);
if (error) throw new Error(ErrorCodes.ISNMPN);
```

This ensures `publicKey.verify(bodyBytes, signature)` is called for every entry in the map before any bytes are written to the transaction or the database. [6](#0-5) 

## Proof of Concept

1. Authenticate as a user whose key is listed as a required signer on transaction ID `42`.
2. Obtain the DER-encoded public key string for that registered key (available from the user's profile).
3. Craft a valid-format signature map with that public key but with garbage signature bytes:

```json
POST /transactions/42/signers
{
  "signatureMap": {
    "0.0.3": {
      "0.0.1234@1700000000.000000000": {
        "<DER_PUBLIC_KEY>": "0xdeadbeefdeadbeef"
      }
    }
  }
}
```

4. The request returns HTTP 201. The `TransactionSigner` record is created and `transaction.transactionBytes` in the database now contains the invalid signature.
5. Once the signing threshold is met by other legitimate signers, the transaction is submitted to Hedera and rejected with `INVALID_SIGNATURE`.

export const validateSignature = (transaction: SDKTransaction, signatureMap: SignatureMap) => {
  const signerPublicKeys: PublicKey[] = [];

  const { rowLength, nodeAccountIdRow, transactionIdCol } =
    getSignedTransactionsDimensions(transaction);

  for (const [nodeAccountId, transactionIds] of signatureMap._map) {
    for (const [transactionId, publicKeys] of transactionIds._map) {
      for (const [publicKeyDer, signature] of publicKeys._map) {
        const publicKey = PublicKey.fromString(publicKeyDer);
        const publicKeyHex = publicKey.toStringRaw();

        const alreadySigned =
          transaction._signerPublicKeys.has(publicKeyHex) ||
          transaction._signerPublicKeys.has(publicKeyDer);

        if (!alreadySigned) {
          const row = nodeAccountIdRow[nodeAccountId];
          const col = transactionIdCol[transactionId];

          const bodyBytes = transaction._signedTransactions.get(col * rowLength + row).bodyBytes;

          const signatureValid = publicKey.verify(bodyBytes, signature);

          if (signatureValid) {
            signerPublicKeys.push(publicKey);
          } else {
            throw new Error('Invalid signature');
          }
        }
      }
    }
  }

  return signerPublicKeys;
};
