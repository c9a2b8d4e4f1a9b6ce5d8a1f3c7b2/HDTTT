### Title
Missing Cryptographic Signature Verification in `uploadSignatureMaps` Allows Malicious Signer to Corrupt Transaction Bytes

### Summary
The `processTransactionSignatures` method in `signers.service.ts`, called by the `POST /transactions/:transactionId/signers` endpoint, accepts and persists signature maps without verifying that the submitted signature bytes are cryptographically valid for the transaction body. A malicious authenticated signer can submit a well-formed but cryptographically invalid signature for their own registered public key. The backend accepts it, overwrites the stored transaction bytes with the corrupted data, and may advance the transaction status to `WAITING_FOR_EXECUTION`, causing the transaction to fail irrecoverably when submitted to the Hedera network.

### Finding Description

**Root cause:** `processTransactionSignatures` in `signers.service.ts` only checks that the public key in the submitted `SignatureMap` belongs to the authenticated user (via `userKeyMap` lookup), then calls `sdkTransaction.addSignature(publicKey, map)` directly — without ever calling `validateSignature`, which performs the actual cryptographic check (`publicKey.verify(bodyBytes, signature)`). [1](#0-0) 

Compare this to the `importSignatures` path in `transactions.service.ts`, which explicitly calls `validateSignature` before calling `addSignature`: [2](#0-1) 

The `validateSignature` utility function performs the actual cryptographic check: [3](#0-2) 

The `IsSignatureMap` decorator only validates the structural format (valid account ID, transaction ID, non-empty bytes) — it does not verify the signature cryptographically: [4](#0-3) 

**Exploit flow:**

1. Attacker is an authenticated user whose key is a required signer of a target transaction.
2. Attacker calls `POST /transactions/:transactionId/signers` with a `SignatureMap` containing their valid public key but garbage/invalid signature bytes.
3. `validateAndProcessSignatures` → `processTransactionSignatures` looks up the public key in `userKeyMap` (passes, key belongs to attacker), then calls `sdkTransaction.addSignature(publicKey, map)` with the invalid signature.
4. The Hedera SDK adds the invalid signature to the transaction's internal state; `_signerPublicKeys` now includes the attacker's key.
5. `isSameBytes` is `false` (bytes changed), so `persistSignatureChanges` overwrites `transactionBytes` in the database with the corrupted bytes. [5](#0-4) 

6. `processTransactionStatus` checks `hasValidSignatureKey([...sdkTransaction._signerPublicKeys], signatureKey)` — the attacker's key is present in `_signerPublicKeys`, so the check passes and the status may advance to `WAITING_FOR_EXECUTION`. [6](#0-5) 

7. The chain service's `getValidatedSDKTransaction` also only checks key presence in `_signerPublicKeys`, not cryptographic validity of the signature bytes: [7](#0-6) 

8. When the transaction is submitted to the Hedera network, it is rejected due to the invalid signature. The transaction bytes in the database are permanently corrupted and cannot be recovered.

### Impact Explanation
A malicious signer can permanently corrupt the stored transaction bytes of any transaction they are a required signer of. This causes:
- **Irrecoverable transaction failure**: The corrupted bytes are stored permanently; the transaction will always fail at the Hedera network level.
- **Wasted co-signers' effort**: All other signers who legitimately signed the transaction have their work destroyed.
- **Potential fee loss**: If the transaction is submitted to the Hedera network before the failure is detected, the transaction fee is consumed.
- **Status integrity violation**: The transaction may be advanced to `WAITING_FOR_EXECUTION` under false pretenses, misleading all participants.

This maps to "Permanent lock, freeze, or unrecoverable corruption of user/project state" from the RESEARCHER.md goals.

### Likelihood Explanation
The attacker preconditions are minimal: any authenticated organization user whose key appears as a required signer on a transaction can execute this attack. No admin access, no leaked credentials, and no privileged role is required. The `POST /transactions/:transactionId/signers` endpoint is a standard workflow endpoint reachable by any verified user. The attack requires only crafting a `SignatureMap` with valid format (passes `IsSignatureMap` decorator) but invalid signature bytes — trivially achievable by any API client.

### Recommendation
Add cryptographic signature verification in `processTransactionSignatures` before calling `addSignature`, mirroring the pattern already used in `importSignatures`:

```typescript
// In processTransactionSignatures, before sdkTransaction.addSignature(publicKey, map):
const { data: validKeys, error } = safe<PublicKey[]>(
  validateSignature.bind(null, sdkTransaction, map),
);
if (error) throw new Error(ErrorCodes.ISNMPN);
```

The `validateSignature` function already exists and is correct — it just needs to be called in the signer upload path as well.

### Proof of Concept

**Preconditions:** Attacker is an authenticated user with key `K` registered in the system. Key `K` is a required signer of transaction with ID `T`.

**Steps:**
1. Construct a valid `SignatureMap` structure (passes `IsSignatureMap` format validation) with the attacker's real public key but with `64` bytes of `0x00` as the signature value.
2. Send `POST /transactions/T/signers` with this payload using the attacker's JWT.
3. Observe: the server returns HTTP 201 with a signer record created.
4. Query the transaction: `transactionBytes` in the database now contains the invalid signature.
5. If `K` was the last required signer, the transaction status advances to `WAITING_FOR_EXECUTION`.
6. When the chain service submits the transaction to Hedera, it receives `INVALID_SIGNATURE` and the transaction is marked `FAILED`.
7. The transaction cannot be re-signed or recovered because the stored bytes are permanently corrupted. [8](#0-7) [9](#0-8)

  private async processTransactionSignatures(
    transaction: Transaction,
    map: SignatureMap,
    userKeyMap: Map<string, UserKey>,
    existingSignerIds: Set<number>
  ) {
    let sdkTransaction = SDKTransaction.fromBytes(transaction.transactionBytes);

    const userKeys: UserKey[] = [];
    const processedRawKeys = new Set<string>();

    // To explain what is going on here, we need to understand how sdkTransaction.addSignature works.
    // The addSignature method will go through each inner transaction, then go through the map
    // and pull the signatures for the supplied public key belonging to that inner transaction
    // (denoted by the node and transaction id), add the signatures to the inner transactions.
    // So we need to go through the map and get each unique publicKey and call addSignature one time
    // per key.
    for (const nodeMap of map.values()) {
      for (const txMap of nodeMap.values()) {
        for (const publicKey of txMap.keys()) {
          const raw = publicKey.toStringRaw();

          // Skip duplicates across node/tx maps, and already-processed keys
          if (processedRawKeys.has(raw)) continue;
          processedRawKeys.add(raw);

          // Look up key (raw first, then DER)
          let userKey = userKeyMap.get(raw);
          if (!userKey) {
            userKey = userKeyMap.get(publicKey.toStringDer());
          }
          if (!userKey) throw new Error(ErrorCodes.PNY);

          // Only add the signature once per unique key
          sdkTransaction = sdkTransaction.addSignature(publicKey, map);

          // Only return "new" signers (not already persisted)
          if (!existingSignerIds.has(userKey.id)) {
            userKeys.push(userKey);
          }
        }
      }
    }

    // Finally, compare the resulting transaction bytes to see if any signatures were actually added
    const isSameBytes = Buffer.from(sdkTransaction.toBytes()).equals(
      transaction.transactionBytes
    );

    return { sdkTransaction, userKeys, isSameBytes };

export const validateSignature = (transaction: SDKTransaction, signatureMap: SignatureMap) => {
  const signerPublicKeys: PublicKey[] = [];

  const { rowLength, nodeAccountIdRow, transactionIdCol } =
    getSignedTransactionsDimensions(transaction);

  for (const [nodeAccountId, transactionIds] of signatureMap._map) {
    for (const [transactionId, publicKeys] of transactionIds._map) {
      for (const [publicKeyDer, signature] of publicKeys._map) {
        const publicKey = PublicKey.fromString(publicKeyDer);
        const publicKeyHex = publicKey.toStringRaw();

        const alreadySigned =
          transaction._signerPublicKeys.has(publicKeyHex) ||
          transaction._signerPublicKeys.has(publicKeyDer);

        if (!alreadySigned) {
          const row = nodeAccountIdRow[nodeAccountId];
          const col = transactionIdCol[transactionId];

          const bodyBytes = transaction._signedTransactions.get(col * rowLength + row).bodyBytes;

          const signatureValid = publicKey.verify(bodyBytes, signature);

          if (signatureValid) {
            signerPublicKeys.push(publicKey);
          } else {
            throw new Error('Invalid signature');
          }
        }
      }
    }
  }

  return signerPublicKeys;
};

export function IsSignatureMap() {
  const isObject = child => child && typeof child === 'object';

  const assertNodeAccountIdValid = (nodeAccountId: string, transactionIds) => {
    if (!isAccountId(nodeAccountId) || !isObject(transactionIds)) {
      throw new BadRequestException(ErrorCodes.ISNMP);
    }
  };

  const assertTransactionIdValid = (transactionId: string, publicKeys) => {
    if (!isTransactionId(transactionId) || !isObject(publicKeys)) {
      throw new BadRequestException(ErrorCodes.ISNMP);
    }
  };

  return function (target: any, propertyKey: string) {
    // ONLY apply Transform - remove Type() completely
    Transform(({ value, obj }) => {
      // If already transformed, return as-is
      if (value instanceof SignatureMap) {
        return value;
      }

      const signatureMap = new SignatureMap();

      if (!isObject(value)) {
        throw new BadRequestException(ErrorCodes.ISNMP);
      }

      for (const nodeAccountId in value) {
        const transactionIds = value[nodeAccountId];

        assertNodeAccountIdValid(nodeAccountId, transactionIds);

        for (const transactionId in transactionIds) {
          const publicKeys = transactionIds[transactionId];
          assertTransactionIdValid(transactionId, publicKeys);

          for (const publicKey in publicKeys) {
            const signature = publicKeys[publicKey];
            const decodedSignature = new Uint8Array(decode(signature));

            if (decodedSignature.length === 0) {
              throw new BadRequestException(ErrorCodes.ISNMP);
            }

            signatureMap.addSignature(
              AccountId.fromString(nodeAccountId),
              TransactionId.fromString(transactionId),
              PublicKey.fromString(publicKey),
              decodedSignature,
            );
          }
        }
      }

      return signatureMap;
    }, { toClassOnly: true })(target, propertyKey);
