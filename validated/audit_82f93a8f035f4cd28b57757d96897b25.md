Audit Report

## Title
Missing Cryptographic Signature Verification in `uploadSignatureMap` Allows False Signing Records and Transaction Corruption

## Summary
The `POST /transactions/:transactionId/signers` endpoint (`SignersService.uploadSignatureMaps`) accepts a `SignatureMap`, verifies only that the submitted public key belongs to the authenticated user, and calls `sdkTransaction.addSignature(publicKey, map)` without ever cryptographically verifying that the signature bytes are valid over the transaction body. A registered user who is a required signer can submit their legitimate public key paired with arbitrary/garbage signature bytes, causing the server to record them as a signer, persist corrupted transaction bytes, and potentially advance the transaction to `WAITING_FOR_EXECUTION` — all without possessing the corresponding private key.

## Finding Description

**Root cause:** `SignersService.processTransactionSignatures` performs only a key-ownership check before calling `addSignature`. No call to `publicKey.verify(bodyBytes, signature)` exists in this code path.

In `back-end/apps/api/src/transactions/signers/signers.service.ts`:

```
for (const publicKey of txMap.keys()) {
  let userKey = userKeyMap.get(raw);          // ownership check only
  if (!userKey) throw new Error(ErrorCodes.PNY);
  sdkTransaction = sdkTransaction.addSignature(publicKey, map);  // NO crypto verify
  userKeys.push(userKey);
}
``` [1](#0-0) 

**Contrast with `importSignatures`** in `back-end/apps/api/src/transactions/transactions.service.ts`, which explicitly calls `validateSignature(sdkTransaction, map)` before `addSignature`:

```typescript
const { data: publicKeys, error } = safe<PublicKey[]>(
  validateSignature.bind(this, sdkTransaction, map),
);
if (error) throw new BadRequestException(ErrorCodes.ISNMPN);
``` [2](#0-1) 

The `validateSignature` utility (in `back-end/libs/common/src/utils/sdk/transaction.ts`) calls `publicKey.verify(bodyBytes, signature)` and throws `'Invalid signature'` on failure:

```typescript
const signatureValid = publicKey.verify(bodyBytes, signature);
if (signatureValid) {
  signerPublicKeys.push(publicKey);
} else {
  throw new Error('Invalid signature');
}
``` [3](#0-2) 

The `signers` path never calls this function. The two code paths are inconsistent.

## Impact Explanation

1. **False signing record**: A `TransactionSigner` row is inserted for the attacker's key without cryptographic proof of possession. This permanently corrupts the audit trail of the multi-signature workflow. [4](#0-3) 

2. **Transaction byte corruption**: Because `addSignature` appends whatever bytes are in the map, `isSameBytes` evaluates to `false`, and the corrupted transaction bytes are persisted to the database. [5](#0-4) 

3. **Transaction execution failure / DoS**: If `processTransactionStatus` advances the transaction to `WAITING_FOR_EXECUTION`, the chain service will submit the corrupted bytes to Hedera and receive `INVALID_SIGNATURE`, permanently moving the transaction to `FAILED` status. High-value operations (transfers, account updates, node operations) are irreversibly blocked.

4. **Notification suppression**: `bulkUpdateNotificationReceivers` marks the attacker's signing notification as read, suppressing reminders to legitimate signers. [6](#0-5) 

## Likelihood Explanation

The attacker requires only:
- A valid registered account on the backend.
- Their public key to be one of the required signers for the target transaction (a normal condition in multi-sig workflows).
- The ability to craft a `SignatureMap` with their public key and arbitrary bytes — trivially done with the Hedera SDK or by hand-crafting the request body.

No privileged access, no leaked credentials, and no cryptographic break are required. The attack is a single authenticated HTTP POST.

## Recommendation

Apply the same `validateSignature` guard used in `importSignatures` to `processTransactionSignatures` in `signers.service.ts`. Before calling `sdkTransaction.addSignature(publicKey, map)`, call:

```typescript
const { data: publicKeys, error } = safe<PublicKey[]>(
  validateSignature.bind(this, sdkTransaction, map),
);
if (error) throw new Error(ErrorCodes.ISNMPN);
```

This ensures `publicKey.verify(bodyBytes, signature)` is called for every signature in the map, and any invalid signature bytes are rejected before being appended to the transaction or recorded in the database. [7](#0-6) 

## Proof of Concept

1. Attacker is an authenticated user with registered public key `PK` that is a required signer for transaction `T`.
2. Attacker constructs a `SignatureMap` containing `PK` mapped to 64 bytes of random garbage as the signature value.
3. Attacker submits `POST /transactions/T/signers` with this map.
4. Server: `userKeyMap.get(raw)` finds `PK` → ownership check passes.
5. `sdkTransaction.addSignature(PK, map)` appends the garbage bytes → `isSameBytes` is `false`.
6. A `TransactionSigner` row is inserted for the attacker's key; the corrupted `transactionBytes` are persisted.
7. `processTransactionStatus` may advance the transaction to `WAITING_FOR_EXECUTION`.
8. The chain service submits the transaction to Hedera → `INVALID_SIGNATURE` → transaction moves to `FAILED`. [8](#0-7)

  private async processTransactionSignatures(
    transaction: Transaction,
    map: SignatureMap,
    userKeyMap: Map<string, UserKey>,
    existingSignerIds: Set<number>
  ) {
    let sdkTransaction = SDKTransaction.fromBytes(transaction.transactionBytes);

    const userKeys: UserKey[] = [];
    const processedRawKeys = new Set<string>();

    // To explain what is going on here, we need to understand how sdkTransaction.addSignature works.
    // The addSignature method will go through each inner transaction, then go through the map
    // and pull the signatures for the supplied public key belonging to that inner transaction
    // (denoted by the node and transaction id), add the signatures to the inner transactions.
    // So we need to go through the map and get each unique publicKey and call addSignature one time
    // per key.
    for (const nodeMap of map.values()) {
      for (const txMap of nodeMap.values()) {
        for (const publicKey of txMap.keys()) {
          const raw = publicKey.toStringRaw();

          // Skip duplicates across node/tx maps, and already-processed keys
          if (processedRawKeys.has(raw)) continue;
          processedRawKeys.add(raw);

          // Look up key (raw first, then DER)
          let userKey = userKeyMap.get(raw);
          if (!userKey) {
            userKey = userKeyMap.get(publicKey.toStringDer());
          }
          if (!userKey) throw new Error(ErrorCodes.PNY);

          // Only add the signature once per unique key
          sdkTransaction = sdkTransaction.addSignature(publicKey, map);

          // Only return "new" signers (not already persisted)
          if (!existingSignerIds.has(userKey.id)) {
            userKeys.push(userKey);
          }
        }
      }
    }

    // Finally, compare the resulting transaction bytes to see if any signatures were actually added
    const isSameBytes = Buffer.from(sdkTransaction.toBytes()).equals(
      transaction.transactionBytes
    );

    return { sdkTransaction, userKeys, isSameBytes };
  }

export const validateSignature = (transaction: SDKTransaction, signatureMap: SignatureMap) => {
  const signerPublicKeys: PublicKey[] = [];

  const { rowLength, nodeAccountIdRow, transactionIdCol } =
    getSignedTransactionsDimensions(transaction);

  for (const [nodeAccountId, transactionIds] of signatureMap._map) {
    for (const [transactionId, publicKeys] of transactionIds._map) {
      for (const [publicKeyDer, signature] of publicKeys._map) {
        const publicKey = PublicKey.fromString(publicKeyDer);
        const publicKeyHex = publicKey.toStringRaw();

        const alreadySigned =
          transaction._signerPublicKeys.has(publicKeyHex) ||
          transaction._signerPublicKeys.has(publicKeyDer);

        if (!alreadySigned) {
          const row = nodeAccountIdRow[nodeAccountId];
          const col = transactionIdCol[transactionId];

          const bodyBytes = transaction._signedTransactions.get(col * rowLength + row).bodyBytes;

          const signatureValid = publicKey.verify(bodyBytes, signature);

          if (signatureValid) {
            signerPublicKeys.push(publicKey);
          } else {
            throw new Error('Invalid signature');
          }
        }
      }
    }
  }

  return signerPublicKeys;
};
