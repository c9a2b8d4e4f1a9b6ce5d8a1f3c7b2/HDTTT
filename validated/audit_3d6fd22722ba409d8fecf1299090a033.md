Audit Report

## Title
Critically Low PBKDF2 Iteration Count Enables Rapid Brute-Force of Encrypted Private Keys

## Summary
The `deriveKey` function in `front-end/src/main/utils/crypto.ts` uses only **2,560 PBKDF2-SHA512 iterations** to derive the AES-256-GCM key that protects all locally stored Hedera private keys. This is approximately 82× below the OWASP minimum recommendation of 210,000 for PBKDF2-SHA512. An attacker who obtains the SQLite database file can brute-force the user's password at extremely high speed and recover all plaintext private keys.

## Finding Description

The `deriveKey` function is confirmed at: [1](#0-0) 

It is called by both `encrypt()` and `decrypt()` in the same file: [2](#0-1) 

`storeKeyPair` in `keyPairs.ts` calls `encrypt()` to protect private keys before writing to SQLite: [3](#0-2) 

`changeDecryptionPassword` decrypts then re-encrypts all key pairs using the same weak KDF: [4](#0-3) 

The same 2,560-iteration `deriveKey` is duplicated verbatim in the automation utility: [5](#0-4) 

By contrast, the data migration path (`generateArgon2id`) correctly uses Argon2id with 256 MB memory cost and 3 time iterations — a strong KDF — but this is absent from the primary production encryption path: [6](#0-5) 

The password strength check enforced at registration requires only a minimum of 10 characters with no complexity requirements: [7](#0-6) 

## Impact Explanation

OWASP currently recommends 210,000 iterations for PBKDF2-SHA512. The application uses 2,560 — approximately 82× fewer. On a modern GPU (e.g., RTX 4090), PBKDF2-SHA512 at 2,560 iterations can be computed at hundreds of millions to billions of attempts per second using tools like Hashcat (mode `-m 12100`). An attacker who obtains the SQLite database file can extract the salt, IV, auth tag, and ciphertext from the base64-encoded `private_key` column (format confirmed: `salt[64] | iv[16] | tag[16] | ciphertext`), then exhaust a large portion of the realistic 10-character password space in minutes to hours, recovering all plaintext Hedera private keys. Since private keys control Hedera accounts directly, this results in complete and irreversible loss of all user funds. [8](#0-7) 

## Likelihood Explanation

The SQLite database is a local file on the user's machine. It is reachable via:
- Malware or RAT with filesystem access (no physical access required — not excluded by SECURITY.md)
- Cloud backup services (e.g., iCloud, OneDrive) that sync the application data directory (no physical access required)
- Theft of a laptop or unencrypted disk backup

No network access to the Hedera backend is required. The attacker only needs the database file. The 2,560-iteration KDF provides negligible resistance to offline brute-force on any modern hardware. The minimum password policy (10 characters, no complexity) further reduces the search space. [9](#0-8) 

## Recommendation

Replace the `deriveKey` function in both `front-end/src/main/utils/crypto.ts` and `automation/utils/crypto/crypto.ts` with Argon2id (already a dependency in both files via the `argon2` package), matching the parameters already used in the migration path: `memoryCost: 262144` (256 MB), `timeCost: 3`, `parallelism: 1`. At minimum, if PBKDF2 must be retained, raise the iteration count to at least 210,000 per current OWASP guidance. All existing encrypted key pairs will need to be re-encrypted on next login (similar to the existing `changeDecryptionPassword` flow). [10](#0-9) [11](#0-10) 

## Proof of Concept

1. Locate the SQLite database file in the application data directory.
2. Extract a `private_key` value from the `KeyPair` table for any user.
3. Base64-decode the value; the first 64 bytes are the salt, bytes 64–79 are the IV, bytes 80–95 are the GCM auth tag, and the remainder is the ciphertext.
4. Construct a Hashcat attack using mode `-m 12100` (PBKDF2-HMAC-SHA512) with the extracted salt and 2,560 iterations, targeting the AES-256-GCM key derivation.
5. Run a dictionary or brute-force attack against the 10-character minimum password space.
6. On recovery of the correct password, call `decrypt(private_key, recovered_password)` using the application's own `decrypt()` function to obtain the plaintext Hedera private key.
7. Use the private key to sign transactions and drain the associated Hedera account. [12](#0-11) [13](#0-12)

import crypto from 'crypto';
import * as argon2 from 'argon2';
import * as bcrypt from 'bcrypt';

export function deriveKey(password: string, salt: Buffer) {
  const iterations = 2560;
  const keyLength = 32;

  return crypto.pbkdf2Sync(password, salt, iterations, keyLength, 'sha512');
}

export function encrypt(data: string, password: string) {
  const iv = crypto.randomBytes(16);
  const salt = crypto.randomBytes(64);

  const key = deriveKey(password, salt);

  const cipher = crypto.createCipheriv('aes-256-gcm', key, iv);

  const encrypted = Buffer.concat([cipher.update(data, 'utf8'), cipher.final()]);

  const tag = cipher.getAuthTag();

  return Buffer.concat([salt, iv, tag, encrypted]).toString('base64');
}

export function decrypt(data: string, password: string) {
  const bData = Buffer.from(data, 'base64');

  const salt = bData.subarray(0, 64);
  const iv = bData.subarray(64, 80);
  const tag = bData.subarray(80, 96);
  const text = bData.subarray(96).toString('base64');

  const key = deriveKey(password, salt);

  const decipher = crypto.createDecipheriv('aes-256-gcm', key, iv);
  decipher.setAuthTag(tag);

  const decrypted = decipher.update(text, 'base64', 'utf8') + decipher.final('utf8');

  return decrypted;
}

export const storeKeyPair = async (

  password: string | null,

      const useKeychain = await getUseKeychainClaim();

        throw new Error('Password is required to store unencrypted key pair');

      data: keyPair,
    });
  } catch (error: unknown) {
    logger.error('Failed to store key pair', { error });
    throw new Error(error instanceof Error ? error.message : 'Failed to store key pair');
  }
};

export const changeDecryptionPassword = async (
  userId: string,
  oldPassword: string,
  newPassword: string,
) => {
  const prisma = getPrismaClient();

  const keyPairs = await getKeyPairs(userId);

  for (let i = 0; i < keyPairs.length; i++) {
    const keyPair = keyPairs[i];
    const decryptedPrivateKey = decrypt(keyPair.private_key, oldPassword);
    const encryptedPrivateKey = encrypt(decryptedPrivateKey, newPassword);

    await prisma.keyPair.update({
      where: {
        id: keyPair.id,
        public_key: keyPair.public_key,
      },
      data: {
        private_key: encryptedPrivateKey,
      },
    });
  }

export async function generateArgon2id(password: string, salt: Buffer) {
  const options = {
    type: argon2.argon2id,
    memoryCost: 262144, // 256MB
    timeCost: 3, // iterations
    parallelism: 1, // threads
    salt,
    raw: true, // get the raw bytes
  };
  return argon2.hash(password, options);
}
