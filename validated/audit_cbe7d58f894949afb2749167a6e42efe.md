### Title
Unbounded Array Input in `uploadSignatureMap` Enables Server-Side Resource Exhaustion DoS

### Summary
The `POST /transactions/signers` endpoint accepts an unbounded array of `UploadSignatureMapDto` objects with no maximum size constraint. Any authenticated user can submit a single request containing thousands of entries, triggering concurrent CPU-intensive SDK deserialization, nested signature-map iteration, and bulk database operations — exhausting server resources and degrading or denying service to all other users.

### Finding Description

**Root cause:** `UploadSignatureMapDto` has no `@ArrayMaxSize` decorator, and the controller passes the raw array directly to the service without any length check.

`back-end/apps/api/src/transactions/dto/upload-signature.dto.ts` — the DTO carries only field-level validators (`@IsDefined`, `@IsNumber`, `@IsSignatureMap`), with no array-level size cap: [1](#0-0) 

`back-end/apps/api/src/transactions/signers/signers.controller.ts` — the controller accepts `UploadSignatureMapDto | UploadSignatureMapDto[]` and forwards the entire array to the service: [2](#0-1) 

**Exploit path:**

1. `loadTransactionData` issues a single `WHERE id IN (...)` query with all attacker-supplied IDs — a potentially enormous `IN` clause: [3](#0-2) 

2. `validateAndProcessSignatures` then fans out with `Promise.all(dto.map(...))`, processing every entry **concurrently**. For each valid entry this includes `SDKTransaction.fromBytes()` (protobuf deserialization), nested triple-loop over the `SignatureMap`, and key lookups: [4](#0-3) 

3. `persistSignatureChanges` then executes a database transaction that bulk-updates transaction bytes, notification receivers, and inserts signers — all proportional to the input array size: [5](#0-4) 

A secondary surface with the same class exists in `createTransactionObservers`, whose DTO enforces only `@ArrayMinSize(1)` with no upper bound: [6](#0-5) 

The developers themselves acknowledged the missing limit in `importSignatures`: [7](#0-6) 

### Impact Explanation
A single crafted POST request with thousands of entries causes concurrent CPU exhaustion (protobuf deserialization × N), memory pressure (all results held in memory simultaneously via `Promise.all`), and heavy database load (large `IN` queries + bulk writes). This degrades or denies service to all other users of the API without requiring any special privilege beyond a valid JWT.

### Likelihood Explanation
Any registered, verified user can reach this endpoint. No admin role, no leaked credentials, and no special transaction ownership is required — the attacker only needs a valid session token. The attack requires a single HTTP request and is trivially scriptable.

### Recommendation
Add `@ArrayMaxSize(N)` to `UploadSignatureMapDto` (and `CreateTransactionObserversDto`) at the DTO level, where `N` is a reasonable operational limit (e.g., 50–100). Additionally, enforce the limit in the service layer as a defense-in-depth measure, consistent with the developer note already present in `importSignatures`.

### Proof of Concept

```
POST /transactions/signers
Authorization: Bearer <valid_jwt>
Content-Type: application/json

[
  { "id": 1, "signatureMap": { ... } },
  { "id": 2, "signatureMap": { ... } },
  ... (repeat 10,000 times with valid or invalid IDs)
]
```

Expected outcome: the server spawns 10,000 concurrent async tasks via `Promise.all`, each deserializing transaction bytes and iterating the signature map, saturating the Node.js event loop and the PostgreSQL connection pool. Legitimate requests time out or are rejected for the duration of processing.

  private async loadTransactionData(dto: UploadSignatureMapDto[]) {
    const transactionIds = dto.map(item => item.id);

    // Batch load all transactions
    const transactions = await this.dataSource.manager.find(Transaction, {
      where: { id: In(transactionIds) },
    });

    const transactionMap = new Map(transactions.map(t => [t.id, t]));

    // Batch load all existing signers
    const existingSigners = await this.dataSource.manager.find(TransactionSigner, {
      where: { transactionId: In(transactionIds) },
      select: ['transactionId', 'userKeyId'],
    });

    // Group by transaction ID
    const signersByTransaction = new Map<number, Set<number>>();
    for (const signer of existingSigners) {
      if (!signersByTransaction.has(signer.transactionId)) {
        signersByTransaction.set(signer.transactionId, new Set());
      }
      signersByTransaction.get(signer.transactionId).add(signer.userKeyId);
    }

    return { transactionMap, signersByTransaction };
  }

  private async persistSignatureChanges(
    validationResults: any[],
    user: User,
  ) {
    const signers = new Set<TransactionSigner>();
    let notificationsToDismiss: number[] = [];

    // Prepare batched operations
    const transactionsToUpdate: { id: number; transactionBytes: Buffer }[] = [];
    const notificationsToUpdate: { userId: number; transactionId: number }[] = [];
    const signersToInsert: { userId: number; transactionId: number; userKeyId: number }[] = [];
    const transactionsToProcess: { id: number; transaction: Transaction }[] = [];

    for (const result of validationResults) {
      if (result.error) {
        console.error(`[TX ${result.id}] Validation failed: ${result.error}`);
        continue;
      }

      const { id, transaction, sdkTransaction, userKeys, isSameBytes } = result;

      // Skip if nothing to do - no signatures were added to the transaction
      // AND no new signers were inserted (the signature can be present on the transaction
      // if collated by an outside or 'offline' method)
      if (isSameBytes && userKeys.length === 0) continue;

      // Collect updates
      if (!isSameBytes) {
        transaction.transactionBytes = Buffer.from(sdkTransaction.toBytes());
        transactionsToUpdate.push({ id, transactionBytes: transaction.transactionBytes });
      }

      // Collect inserts
      if (userKeys.length > 0) {
        const newSigners = userKeys.map(userKey => ({
          userId: user.id,
          transactionId: id,
          userKeyId: userKey.id,
        }));
        signersToInsert.push(...newSigners);
      }

      transactionsToProcess.push({ id, transaction });
      notificationsToUpdate.push({ userId: user.id, transactionId: transaction.id });
    }

    // Execute in single transaction
    try {
      await this.dataSource.transaction(async manager => {
        // Bulk update transactions
        if (transactionsToUpdate.length > 0) {
          await this.bulkUpdateTransactions(manager, transactionsToUpdate);
        }

        // Bulk update notifications
        if (notificationsToUpdate.length > 0) {
          const updatedNotificationReceivers = await this.bulkUpdateNotificationReceivers(manager, notificationsToUpdate);

          // To maintain backwards compatibility and multi-machine support, we send off a dismiss event.
          emitDismissedNotifications(
            this.notificationsPublisher,
            updatedNotificationReceivers,
          );

          notificationsToDismiss = updatedNotificationReceivers.map(nr => nr.id);
        }

        // Bulk insert signers
        if (signersToInsert.length > 0) {
          const results = await this.bulkInsertSigners(manager, signersToInsert);
          results.forEach(signer => signers.add(signer));
        }
      });
    } catch (err) {
      console.error('Database transaction failed:', err);
      throw new BadRequestException(ErrorCodes.FST);
    }

    return {
      transactionsToProcess,
      signers,
      notificationsToDismiss,
    };
  }
