### Title
`uploadSignatureMaps` Accepts Unverified Signatures, Allowing a Malicious Signer to Permanently Corrupt Transaction Bytes and Block Execution

### Summary
The `processTransactionSignatures` method in `signers.service.ts` calls `sdkTransaction.addSignature(publicKey, map)` without first cryptographically verifying that the submitted signature was produced over the correct transaction body bytes. A malicious registered signer can submit a signature made over a different (modified) transaction body; the server accepts it, records the user as a signer, and persists the corrupted bytes. When the transaction is later executed, the Hedera network rejects it with `INVALID_SIGNATURE`, permanently blocking execution with no recovery path.

### Finding Description

**Root cause — missing `validateSignature` call in `uploadSignatureMaps` path**

`importSignatures` in `transactions.service.ts` correctly calls `validateSignature` before accepting any signature: [1](#0-0) 

`validateSignature` explicitly calls `publicKey.verify(bodyBytes, signature)` for every entry in the map and throws on mismatch: [2](#0-1) 

The `uploadSignatureMaps` path (`POST /transactions/:id/signers`) goes through a completely different code path — `validateAndProcessSignatures` → `processTransactionSignatures` — which **never calls `validateSignature`**. It calls `addSignature` directly: [3](#0-2) 

The Hedera SDK's `addSignature` does not verify the signature cryptographically; it only inserts the raw bytes into the transaction's signature map. This is confirmed by the developer comment explaining what `addSignature` does (iterate and insert), with no mention of verification: [4](#0-3) 

**Exploit flow**

1. Attacker is a legitimate registered signer of transaction `T` (e.g., added by the creator in organization mode).
2. Attacker fetches `T`'s `transactionBytes` from the API.
3. Attacker locally constructs a **different** transaction body `T'` (e.g., different amount, different recipient, or simply random bytes) and signs it with their private key, producing `sigMap'`.
4. Attacker posts `sigMap'` to `POST /transactions/{id}/signers`.
5. `processTransactionSignatures` loads the original transaction from DB, calls `sdkTransaction.addSignature(publicKey, sigMap')` — no verification occurs.
6. `isSameBytes` is `false` (bytes changed), so the corrupted bytes are persisted: [5](#0-4) 

7. The attacker is recorded as a signer. The transaction now carries an invalid signature in the DB.
8. When the chain service executes the transaction, the Hedera network returns `INVALID_SIGNATURE` and the transaction is marked `FAILED` permanently. [6](#0-5) 

### Impact Explanation

A malicious signer can permanently block execution of any transaction they are assigned to sign. The transaction is marked `FAILED` in the database with no mechanism to strip the bad signature and retry. In organization workflows where transactions are matched to specific signers (analogous to the quadruple-to-signing-request deterministic matching in the external report), a single insider can silently kill any targeted transaction — including high-value operations such as `FileUpdate`, `AccountUpdate`, `NodeCreate`, or HBAR transfers — without any visible error at submission time.

### Likelihood Explanation

The attacker must be a registered signer of the target transaction, which is a normal user role in organization mode. No admin or privileged access is required. The attack requires only a standard API call with a crafted `signatureMap` payload — trivially achievable by any user with API access. The absence of any test covering invalid-signature submission in `processTransactionSignatures` confirms the gap is undetected. [7](#0-6) 

### Recommendation

Add a call to `validateSignature` inside `processTransactionSignatures`, immediately before `addSignature`, mirroring the pattern already used in `importSignatures`:

```typescript
// In processTransactionSignatures, before addSignature:
const { data: validKeys, error } = safe<PublicKey[]>(
  validateSignature.bind(null, sdkTransaction, map),
);
if (error) throw new Error(ErrorCodes.ISNMPN);

sdkTransaction = sdkTransaction.addSignature(publicKey, map);
```

This ensures every submitted signature is cryptographically verified against the stored transaction body bytes before being accepted or persisted.

### Proof of Concept

```typescript
// 1. Attacker fetches transaction bytes for transaction ID 42
const txBytes = await api.get('/transactions/42').transactionBytes;

// 2. Attacker constructs a DIFFERENT transaction and signs it

  .setTransactionId(TransactionId.generate('0.0.999'))
  .setNodeAccountIds([AccountId.fromString('0.0.3')])
  .freeze();
const attackerKey = PrivateKey.generateED25519();
await differentTx.sign(attackerKey);
const maliciousMap = differentTx.getSignatures(); // sig over wrong body

// 3. Attacker registers their public key as a user key on the server
// (normal account setup — no privilege required)

// 4. Attacker submits the malicious signature map
await api.post('/transactions/42/signers', [{
  id: 42,
  signatureMap: maliciousMap,  // signed over wrong body, never verified
}]);
// → HTTP 201, attacker recorded as signer, corrupted bytes persisted

// 5. Chain service later executes the transaction
// → Hedera network returns INVALID_SIGNATURE
// → Transaction permanently marked FAILED
```

The server returns HTTP 201 and records the attacker as a signer. The transaction is permanently unexecutable. [8](#0-7) [9](#0-8)

```typescript
export const validateSignature = (transaction: SDKTransaction, signatureMap: SignatureMap) => {
  const signerPublicKeys: PublicKey[] = [];

  const { rowLength, nodeAccountIdRow, transactionIdCol } =
    getSignedTransactionsDimensions(transaction);

  for (const [nodeAccountId, transactionIds] of signatureMap._map) {
    for (const [transactionId, publicKeys] of transactionIds._map) {
      for (const [publicKeyDer, signature] of publicKeys._map) {
        const publicKey = PublicKey.fromString(publicKeyDer);
        const publicKeyHex = publicKey.toStringRaw();

        const alreadySigned =
          transaction._signerPublicKeys.has(publicKeyHex) ||
          transaction._signerPublicKeys.has(publicKeyDer);

        if (!alreadySigned) {
          const row = nodeAccountIdRow[nodeAccountId];
          const col = transactionIdCol[transactionId];

          const bodyBytes = transaction._signedTransactions.get(col * rowLength + row).bodyBytes;

          const signatureValid = publicKey.verify(bodyBytes, signature);

          if (signatureValid) {
            signerPublicKeys.push(publicKey);
          } else {
            throw new Error('Invalid signature');
          }
        }
      }
    }
  }

  return signerPublicKeys;
};
```

```typescript
  private async processTransactionSignatures(
    transaction: Transaction,
    map: SignatureMap,
    userKeyMap: Map<string, UserKey>,
    existingSignerIds: Set<number>
  ) {
    let sdkTransaction = SDKTransaction.fromBytes(transaction.transactionBytes);

    const userKeys: UserKey[] = [];
    const processedRawKeys = new Set<string>();

    // To explain what is going on here, we need to understand how sdkTransaction.addSignature works.
    // The addSignature method will go through each inner transaction, then go through the map
    // and pull the signatures for the supplied public key belonging to that inner transaction
    // (denoted by the node and transaction id), add the signatures to the inner transactions.
    // So we need to go through the map and get each unique publicKey and call addSignature one time
    // per key.
    for (const nodeMap of map.values()) {
      for (const txMap of nodeMap.values()) {
        for (const publicKey of txMap.keys()) {
          const raw = publicKey.toStringRaw();

          // Skip duplicates across node/tx maps, and already-processed keys
          if (processedRawKeys.has(raw)) continue;
          processedRawKeys.add(raw);

          // Look up key (raw first, then DER)
          let userKey = userKeyMap.get(raw);
          if (!userKey) {
            userKey = userKeyMap.get(publicKey.toStringDer());
          }
          if (!userKey) throw new Error(ErrorCodes.PNY);

          // Only add the signature once per unique key
          sdkTransaction = sdkTransaction.addSignature(publicKey, map);

          // Only return "new" signers (not already persisted)
          if (!existingSignerIds.has(userKey.id)) {
            userKeys.push(userKey);
          }
        }
      }
    }

    // Finally, compare the resulting transaction bytes to see if any signatures were actually added
    const isSameBytes = Buffer.from(sdkTransaction.toBytes()).equals(
      transaction.transactionBytes
    );

    return { sdkTransaction, userKeys, isSameBytes };
```
