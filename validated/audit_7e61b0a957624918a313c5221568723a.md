Audit Report

## Title
Any Verified User Can Enumerate All Organization Users' Public Keys via Unguarded `GET /user-keys` Endpoint

## Summary
`UserKeysAllController` exposes `GET /user-keys` protected only by authentication/verification guards, with no `AdminGuard`. The backing service method performs an unrestricted database query returning every user's key records — including `mnemonicHash` and `index` — to any verified organization member.

## Finding Description

**Root cause:** The `getUserKeys` handler in `UserKeysAllController` applies no `AdminGuard` and passes only pagination parameters to the service. [1](#0-0) 

The service method `getUserKeys` issues a `findAndCount` with no `where` clause and no field-level `select` restriction, returning all rows from the `user_key` table: [2](#0-1) 

The serialization DTO exposes every sensitive field — `id`, `userId`, `publicKey`, `mnemonicHash`, `index`, `deletedAt`: [3](#0-2) 

**Contrast with the restricted sibling:** `UserKeysController.getUserKeys` correctly injects `@GetUser()` and delegates to `getUserKeysRestricted`, which filters by `userId` and suppresses `mnemonicHash`/`index` for non-owners: [4](#0-3) 

The `AdminGuard` exists and is used elsewhere (e.g., `PATCH /auth/elevate-admin`) but was simply never applied to this controller: [5](#0-4) 

## Impact Explanation
Any verified user can retrieve the complete paginated key inventory of the entire organization. The `UserKeyDto` exposes `publicKey` (Hedera account identifier), `mnemonicHash` (hash of the mnemonic seed phrase), `index` (key derivation index), and `userId` — all fields that `getUserKeysRestricted` deliberately withholds from non-owners. This breaks the ownership invariant enforced everywhere else in the key management surface.

## Likelihood Explanation
- Precondition is only a valid, verified JWT — the lowest privilege level.
- The endpoint is Swagger-documented under `@ApiTags('User Keys All')`, making it trivially discoverable.
- No special tooling is required beyond a standard HTTP client.

## Recommendation
Add `AdminGuard` to `UserKeysAllController` so only administrators can enumerate all keys:

```typescript
// user-keys-all.controller.ts
import { AdminGuard, JwtAuthGuard, JwtBlackListAuthGuard, VerifiedUserGuard } from '../guards';

@UseGuards(JwtBlackListAuthGuard, JwtAuthGuard, VerifiedUserGuard, AdminGuard)
export class UserKeysAllController { ... }
```

If the endpoint is intended to serve non-admin users, the service must be changed to filter by the requesting user's ID and apply the same field-level restrictions as `getUserKeysRestricted`.
