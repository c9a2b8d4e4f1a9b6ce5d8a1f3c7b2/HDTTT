Audit Report

## Title
Recovery Phrase and Private Key Written to System Clipboard Without Clearing — Persistent Sensitive Credential Exposure

## Summary
The Hedera Transaction Tool Electron desktop application writes 24-word BIP-39 recovery phrases and decrypted raw private keys to the system clipboard via `navigator.clipboard.writeText()` across three separate UI flows. The clipboard is never cleared after the write. Any OS-level process running under the same user account can silently read these credentials at any time after the user clicks "Copy."

## Finding Description

All three code paths are confirmed in the repository:

**Path 1 — `handleCopyRecoveryPhrase()` in `Generate.vue`:** [1](#0-0) 

The full 24-word mnemonic (`words.value.join(', ')`) is written to the clipboard with no subsequent clearing.

**Path 2 — `copyRecoveryPhrase()` in `Summary.vue`:** [2](#0-1) 

`user.recoveryPhrase?.words.join(', ')` is written to the clipboard during the migration summary flow with no subsequent clearing.

**Path 3 — `handleCopy()` in `KeysTab.vue`:** [3](#0-2) 

This generic clipboard writer is invoked with the decrypted raw private key value sourced directly from the `decryptedKeys` reactive array: [4](#0-3) 

No `setTimeout`-based clearing, no IPC call to clear the native clipboard, and no other clearing mechanism exists anywhere in the codebase for any of these three paths.

## Impact Explanation

- **Recovery phrase (Paths 1 & 2):** A 24-word BIP-39 mnemonic is the root secret from which all Hedera key pairs are derived. Theft gives an attacker permanent, full control over every account protected by any key derived from it — including the ability to sign and submit arbitrary Hedera transactions (transfers, account updates, deletions).
- **Private key (Path 3):** A raw private key gives an attacker direct signing authority over the specific Hedera account(s) associated with that key, enabling unauthorized fund transfers and account mutations.
- The OS clipboard persists across application sessions on all major platforms. The secret is accessible to every process running under the same OS user account indefinitely until manually overwritten or the system is rebooted.

## Likelihood Explanation

The attacker preconditions are minimal:

1. **Local process (primary vector):** Any background process — malware, a compromised npm dependency loaded by the Electron main process, or a script — running as the same OS user can call the OS clipboard API without elevated privileges and read the contents.
2. **Malicious browser tab (secondary vector):** A browser tab that has been granted clipboard read permission can call `navigator.clipboard.readText()` and exfiltrate the secret remotely.
3. **No user interaction required after the copy:** The attacker only needs to poll the clipboard at any point after the user clicks "Copy." The exposure window is unbounded.

The "Copy" button is a prominently exposed UI feature used during normal onboarding (`Generate.vue`), migration (`Summary.vue`), and key management (`KeysTab.vue`) flows, making exploitation highly likely in practice.

## Recommendation

After `writeText()` resolves, schedule a clipboard clear using a short timeout (e.g., 30–60 seconds), which is standard practice in password managers and key management tools:

```ts
navigator.clipboard.writeText(secret).then(() => {
  toastManager.success('Copied — clipboard will be cleared in 30 seconds');
  setTimeout(() => navigator.clipboard.writeText(''), 30_000);
});
```

Apply this pattern to all three locations:
- `handleCopyRecoveryPhrase()` in `Generate.vue` [1](#0-0) 
- `copyRecoveryPhrase()` in `Summary.vue` [2](#0-1) 
- `handleCopy()` in `KeysTab.vue` (when called with private key data) [3](#0-2) 

Additionally, consider using Electron's native `clipboard` IPC module instead of `navigator.clipboard` for more reliable cross-platform clearing behavior in the main process.

## Proof of Concept

1. Launch the Hedera Transaction Tool and complete account setup to the `Generate.vue` step.
2. Click the "Copy" button — `handleCopyRecoveryPhrase()` writes the 24-word mnemonic to the clipboard.
3. From any process running as the same OS user (e.g., a Node.js script, Python script, or browser tab with clipboard permission), read the clipboard contents — the full recovery phrase is present.
4. Close the application entirely and repeat step 3 — the recovery phrase is still present in the clipboard.
5. Repeat steps 1–4 for the private key copy button in `KeysTab.vue` (Settings → Keys → reveal a key → click the copy icon).
